## 🐛 問題描述

### 用戶報告的問題

**症狀**:
- 客戶端 Flutter 應用在支付確認後卡住
- 最終顯示連接超時錯誤
- 不管「即時同步開關」是否開啟都會發生

**錯誤訊息**:
```
I/flutter (27097): [BookingService] 創建預約失敗: ClientException with SocketException: Connection timed out (OS Error: Connection timed out, errno = 110), address = 10.0.2.2, port = 60650, uri=http://10.0.2.2:3000/api/bookings
```

**關鍵信息**:
- 客戶端嘗試連接：`http://10.0.2.2:3000/api/bookings`
- 連接超時錯誤（errno = 110）
- Android 模擬器環境（10.0.2.2 是模擬器訪問主機的特殊 IP）
- 應用程式在支付確認後進入後台

## ✅ 根本原因

**問題**: 後端 Node.js Express 服務器沒有運行

**影響**:
- 客戶端無法連接到 `http://10.0.2.2:3000/api/bookings`
- 所有 API 請求都會超時
- 訂單創建失敗

**與即時同步的關係**:
- ❌ 與 Supabase 即時同步**無關**
- ❌ 與「即時同步開關」**無關**
- ✅ 純粹是後端服務器未啟動的問題

## 📝 注意事項

### Android 模擬器網路配置

**10.0.2.2 的含義**:
- Android 模擬器使用 `10.0.2.2` 訪問主機的 `localhost`
- 這是 Android 模擬器的特殊 IP 地址
- 等同於主機的 `127.0.0.1` 或 `localhost`

**確保防火牆允許連接**:
```bash
# Windows
netsh advfirewall firewall add rule name="Node.js Server" dir=in action=allow protocol=TCP localport=3000

# macOS/Linux
sudo ufw allow 3000/tcp
```

---
