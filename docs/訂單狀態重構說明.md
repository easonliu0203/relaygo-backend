# 訂單狀態重構說明

## 📋 重構目標

重新設計訂單狀態映射邏輯，新增 `awaitingDriver`（待司機確認）和 `awaitingBalance`（待付尾款）兩個狀態，以更明確區分訂單流程中的不同階段。

---

## ✅ 已完成的修改

### 1. Flutter 模型（`mobile/lib/core/models/booking_order.dart`）

#### 新增枚舉值

```dart
enum BookingStatus {
  @JsonValue('pending')
  pending,        // 待配對（待付訂金或待派單）
  
  @JsonValue('awaitingDriver')
  awaitingDriver, // 待司機確認 ⭐ 新增
  
  @JsonValue('matched')
  matched,        // 已配對
  
  @JsonValue('inProgress')
  inProgress,     // 進行中
  
  @JsonValue('awaitingBalance')
  awaitingBalance, // 待付尾款 ⭐ 新增
  
  @JsonValue('completed')
  completed,      // 已完成
  
  @JsonValue('cancelled')
  cancelled,      // 已取消
}
```

#### 更新狀態擴展方法

- **displayName**：
  - `awaitingDriver` → "待司機確認"
  - `awaitingBalance` → "待付尾款"

- **color**：
  - `awaitingDriver` → 淺橙色 (`0xFFFFA726`)
  - `awaitingBalance` → 金色 (`0xFFFFB74D`)

#### 計算屬性（無需修改）

- `balancePaid`：只有 `status == BookingStatus.completed` 時才返回 `true`
- `balanceAmount`：計算尾款金額
- `totalPaid`：計算已支付總額

## 🎯 預期效果

### 修改前的問題

1. **`matched` → `pending`**：無法區分「等待派單」和「等待司機確認」
2. **`trip_ended` → `completed`**：誤導用戶以為訂單已完成，實際上還需支付尾款

### 修改後的改進

1. **`matched` → `awaitingDriver`**：明確顯示「待司機確認」，用戶知道正在等待司機接單
2. **`trip_ended` → `awaitingBalance`**：明確顯示「待付尾款」，用戶知道需要支付尾款才能完成訂單

## 📝 注意事項

1. **Freezed 程式碼生成**：修改枚舉後必須重新生成 Freezed 程式碼
2. **Edge Function 部署**：修改狀態映射後必須重新部署 Edge Function
3. **APP 重啟**：修改枚舉後必須完全重啟 APP（熱重載不夠）
4. **向後兼容性**：舊的訂單資料可能仍然使用舊的狀態值，需要確保 Edge Function 能正確處理
5. **測試覆蓋**：需要測試所有狀態轉換，確保 UI 顯示正確
