## 🚀 快速開始

### 方法 1: 使用啟動腳本（推薦）

```bash
# 在項目根目錄執行
scripts\start-all-services.bat
```

這個腳本會自動：
- ✅ 檢查 Backend 依賴
- ✅ 啟動 Backend 服務（端口 3000）
- ✅ 顯示服務地址和測試命令

### 方法 2: 手動啟動

```bash
# 1. 啟動 Backend
cd backend
npm run dev

# 2. 在新的 Terminal 中啟動 Flutter 應用
cd mobile
flutter run
```

## ❓ 常見問題

### Q1: Backend 啟動失敗，提示端口被佔用

**錯誤訊息**:
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解決方案**:

**方法 1: 停止佔用端口的進程**
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Linux/Mac
lsof -i :3000
kill -9 <PID>
```

**方法 2: 修改 Backend 端口**
```bash
# 在 backend/.env 中修改
PORT=3001
```

### Q2: Flutter 應用無法連接到 Backend

**錯誤訊息**:
```
Exception: 創建預約失敗: ClientException with SocketException: Connection timed out
```

**解決方案**:

1. **確認 Backend 已啟動**
   ```bash
   curl http://localhost:3000/health
   ```

2. **確認 URL 配置正確**
   - Android 模擬器：`http://10.0.2.2:3000`
   - 真機：`http://<局域網IP>:3000`

3. **檢查防火牆設置**
   - 確保防火牆未阻止端口 3000

4. **使用健康檢查功能**
   - 在應用中使用 `HealthCheckService` 檢查連接

### Q3: Android 模擬器無法訪問 10.0.2.2

**可能原因**:
- 模擬器網絡配置問題
- 主機防火牆阻止

**解決方案**:

1. **重啟模擬器**
   ```bash
   # 關閉模擬器
   adb emu kill
   
   # 重新啟動
   flutter emulators --launch <emulator_id>
   ```

2. **檢查模擬器網絡設置**
   - 在模擬器中打開設置 → 網絡
   - 確認網絡已連接

3. **使用真機測試**
   - 修改 `.env` 文件中的 `API_BASE_URL`
   - 使用局域網 IP（如 `192.168.1.100`）

### Q4: Flutter 熱重載後無法連接 Backend

**可能原因**:
- Backend 服務被意外停止
- 網絡連接中斷

**解決方案**:

1. **檢查 Backend 狀態**
   ```bash
   curl http://localhost:3000/health
   ```

2. **重啟 Backend**
   ```bash
   # 在 Backend Terminal 中按 Ctrl+C 停止
   # 然後重新啟動
   npm run dev
   ```

3. **重啟 Flutter 應用**
   ```bash
   # 在 Flutter Terminal 中按 R 熱重啟
   R
   ```

## 🔧 故障排除

### 診斷步驟

#### 1. 檢查 Backend 服務

```bash
# 檢查 Backend 是否運行
curl http://localhost:3000/health

# 檢查端口是否被監聽
netstat -ano | findstr :3000  # Windows
lsof -i :3000                 # Linux/Mac
```

#### 2. 檢查 Flutter 配置

```bash
# 檢查 Flutter 環境
flutter doctor

# 檢查設備連接
flutter devices

# 清理並重建
flutter clean
flutter pub get
flutter run
```

#### 3. 檢查網絡連接

**在 Android 模擬器中**:
1. 打開瀏覽器
2. 訪問 `http://10.0.2.2:3000/health`
3. 確認可以訪問

**在真機中**:
1. 確認手機和電腦在同一局域網
2. 獲取電腦的局域網 IP
   ```bash
   ipconfig  # Windows
   ifconfig  # Linux/Mac
   ```
3. 修改 `.env` 文件中的 `API_BASE_URL`
4. 重新編譯應用

### 日誌查看

#### Backend 日誌

Backend 日誌會直接輸出到 Terminal，包括：
- API 請求日誌
- 錯誤日誌
- Firebase 操作日誌

#### Flutter 日誌

```bash
# 查看實時日誌
flutter logs

# 查看特定設備的日誌
flutter logs -d <device_id>
```

#### Android 日誌

```bash
# 查看 Android 系統日誌
adb logcat

# 過濾 Flutter 日誌
adb logcat | findstr flutter
```

### 重置環境

如果遇到無法解決的問題，可以嘗試重置環境：

```bash
# 1. 停止所有服務
# 關閉所有 Terminal 窗口

# 2. 清理 Backend
cd backend
rm -rf node_modules
npm install

# 3. 清理 Flutter
cd mobile
flutter clean
flutter pub get

# 4. 重新啟動
scripts\start-all-services.bat
```

## 💡 最佳實踐

### 開發流程

1. **每次開發前**
   - 啟動 Backend 服務
   - 驗證 Backend 健康狀態
   - 啟動 Flutter 應用

2. **開發過程中**
   - 使用熱重載（`r`）快速測試
   - 使用熱重啟（`R`）重置狀態
   - 定期檢查 Backend 日誌

3. **開發結束後**
   - 停止 Backend 服務（Ctrl+C）
   - 停止 Flutter 應用（`q`）
   - 關閉 Android 模擬器

### 調試技巧

1. **使用 debugPrint**
   ```dart
   debugPrint('[MyService] 開始執行操作...');
   ```

2. **使用 Flutter DevTools**
   ```bash
   flutter pub global activate devtools
   flutter pub global run devtools
   ```

3. **使用 Postman 測試 API**
   - 創建 API 測試集合
   - 測試各個端點
   - 驗證響應格式
