# 訂單詳情頁面返回按鈕修復

## 🐛 問題描述

### 問題：訂單完成後的返回按鈕無法正常工作

**重現步驟**：
1. 完成整個訂單流程（創建訂單 → 支付訂金 → 派單 → 司機確認 → 開始行程 → 結束行程 → 支付尾款）
2. 訂單狀態變為「已完成」（`BookingStatus.completed`）
3. 在訂單詳情頁面（`order_detail_page.dart`）最下方看到一個「返回」按鈕
4. **問題發生**：點擊「返回」按鈕後，完全沒有反應
5. **結果**：無法返回到首頁或「我的訂單」頁面，用戶被困在訂單詳情頁面

### 影響範圍
- **檔案**：`mobile/lib/apps/customer/presentation/pages/order_detail_page.dart`
- **影響用戶**：所有完成訂單的客戶（特別是從「訂單完成」頁面進入訂單詳情的用戶）
- **嚴重程度**：🟡 中（影響用戶體驗，但可以使用系統返回鍵）

---

## 🔍 問題診斷

### 根本原因

**導航方式不一致導致的導航堆疊問題**：

1. **從訂單列表頁面進入訂單詳情**（正常情況）：
   ```dart
   // order_list_page.dart (第 130 行)
   onTap: () => context.push('/order-detail/${order.id}'),
   ```
   - 使用 `context.push()`，將訂單詳情頁面推入導航堆疊
   - 點擊「返回」按鈕時，`context.pop()` 可以正常工作 ✅

2. **從訂單完成頁面進入訂單詳情**（問題情況）：
   ```dart
   // booking_complete_page.dart (第 76 行)
   onPressed: () {
     context.go('/order-detail/$bookingId');  // ❌ 使用 context.go()
   }
   ```
   - 使用 `context.go()`，**替換整個導航堆疊**
   - 訂單詳情頁面成為導航堆疊中的唯一頁面
   - 點擊「返回」按鈕時，`context.pop()` 無法工作，因為沒有上一個頁面 ❌

### `context.go()` vs `context.push()` 的區別

| 方法 | 行為 | 導航堆疊 | 返回按鈕 |
|------|------|---------|---------|
| `context.push()` | 推入新頁面 | 保留之前的頁面 | `context.pop()` 可以返回 ✅ |
| `context.go()` | 替換導航堆疊 | 清除之前的頁面 | `context.pop()` 無法返回 ❌ |

---

## ✅ 修復方案

### 智能返回邏輯

修改「返回」按鈕的 `onPressed` 回調，使其能夠適應不同的導航情況：

```dart
onPressed: () {
  // 檢查是否可以返回上一頁
  if (Navigator.of(context).canPop()) {
    // 如果可以（例如：從訂單列表頁面使用 context.push() 進入）
    context.pop();
  } else {
    // 如果不可以（例如：從訂單完成頁面使用 context.go() 進入）
    context.go('/orders');
  }
}
```

**邏輯說明**：
1. 使用 `Navigator.of(context).canPop()` 檢查導航堆疊中是否有上一個頁面
2. 如果有，使用 `context.pop()` 返回上一頁（保持原有行為）
3. 如果沒有，使用 `context.go('/orders')` 導航到訂單列表頁面（修復問題）

## 💡 技術細節

### `Navigator.of(context).canPop()` 方法

**功能**：檢查導航堆疊中是否有上一個頁面可以返回

**返回值**：
- `true`：導航堆疊中有上一個頁面，可以使用 `pop()` 返回
- `false`：導航堆疊中沒有上一個頁面，`pop()` 無法工作

**使用場景**：
- 在不確定導航堆疊狀態時，使用此方法檢查是否可以返回
- 避免在無法返回時調用 `pop()`，導致沒有反應或錯誤

### 為什麼不直接修改 `booking_complete_page.dart`？

**選項 1：修改 `booking_complete_page.dart`**
```dart
// 將 context.go() 改為 context.push()
context.push('/order-detail/$bookingId');
```

**缺點**：
- 用戶從訂單完成頁面進入訂單詳情後，點擊返回會回到訂單完成頁面
- 用戶還需要再點擊一次才能回到首頁或訂單列表
- 用戶體驗不佳

**選項 2：修改 `order_detail_page.dart`（我們選擇的方案）**
```dart
// 智能返回邏輯
if (Navigator.of(context).canPop()) {
  context.pop();
} else {
  context.go('/orders');
}
```

**優點**：
- 無論用戶如何進入訂單詳情頁面，都能正確返回到合適的頁面
- 用戶體驗更好，一次點擊即可返回
- 更靈活，適應所有導航情況
