## 📋 需求描述

### 問題背景

目前在客戶端應用（Customer App）和司機端應用（Driver App）中，個人檔案頁面的「編輯個人資料」功能尚未實作，導致用戶無法填寫和更新個人資訊。

### 影響範圍

1. **訂單顯示**
   - 訂單中應該顯示客戶/司機的姓名、電話等資訊
   - 目前因為資料未填寫，訂單中可能顯示「未設定」或空白

2. **公司端管理**
   - 公司端的客戶管理頁面和司機管理頁面
   - 應該能看到完整的客戶和司機資料
   - 目前因為用戶無法編輯資料，管理頁面顯示不完整

### 期望功能

1. **客戶端個人資料編輯頁面**
   - 允許客戶編輯姓名、電話、地址等資訊
   - 資料保存到 Supabase 的 `user_profiles` 表

2. **司機端個人資料編輯頁面**
   - 允許司機編輯姓名、電話、地址等資訊
   - 資料保存到 Supabase 的 `user_profiles` 表

3. **資料驗證和同步**
   - 確保資料格式正確（電話號碼格式、必填欄位等）
   - 更新後立即在訂單和公司端管理頁面中顯示

### 技術棧

- **Flutter**: 移動應用框架
- **Riverpod**: 狀態管理
- **Supabase Flutter SDK**: 資料庫操作
- **Freezed**: 不可變數據模型
- **Firebase Auth**: 用戶認證

### 資料流程

```
用戶編輯資料
    ↓
EditProfilePage (UI)
    ↓
UserProfileNotifier (State Management)
    ↓
SupabaseService (Data Layer)
    ↓
Supabase user_profiles 表
    ↓
自動更新 UI (Riverpod)
```

### 資料庫表結構

**user_profiles 表**:
```sql
CREATE TABLE user_profiles (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    phone VARCHAR(20),
    avatar_url TEXT,
    date_of_birth DATE,
    gender VARCHAR(10) CHECK (gender IN ('male', 'female', 'other')),
    address TEXT,
    emergency_contact_name VARCHAR(100),
    emergency_contact_phone VARCHAR(20),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(user_id)
);
```

### 3. 創建 Supabase 服務

**文件**: `mobile/lib/core/services/supabase_service.dart`

**功能**:
- 封裝 Supabase 資料庫操作
- 提供 CRUD 方法
- 處理錯誤和異常

**主要方法**:
1. `getUserProfile(String userId)` - 獲取用戶資料
2. `createUserProfile(...)` - 創建用戶資料
3. `updateUserProfile(...)` - 更新用戶資料
4. `upsertUserProfile(...)` - 創建或更新用戶資料
5. `deleteUserProfile(String userId)` - 刪除用戶資料

**關鍵代碼**:
```dart
class SupabaseService {
  final SupabaseClient _client;

  SupabaseService() : _client = Supabase.instance.client;

  Future<UserProfile?> getUserProfile(String userId) async {
    try {
      final response = await _client
          .from('user_profiles')
          .select()
          .eq('user_id', userId)
          .maybeSingle();

      if (response == null) {
        return null;
      }

      return UserProfile.fromJson({
        'id': response['id'],
        'userId': response['user_id'],
        'firstName': response['first_name'],
        // ... 其他欄位
      });
    } catch (e) {
      throw SupabaseServiceException('獲取用戶資料失敗', e);
    }
  }

  Future<UserProfile> upsertUserProfile({
    required String userId,
    // ... 其他參數
  }) async {
    // 先檢查是否已存在
    final existing = await getUserProfile(userId);

    if (existing == null) {
      // 不存在，創建新記錄
      return await createUserProfile(...);
    } else {
      // 已存在，更新記錄
      return await updateUserProfile(...);
    }
  }
}
```

### 4. 創建用戶資料 Provider

**文件**: `mobile/lib/shared/providers/user_profile_provider.dart`

**功能**:
- 使用 Riverpod 管理用戶資料狀態
- 自動監聽認證狀態變化
- 提供重新載入和更新方法

**關鍵代碼**:
```dart
/// 用戶資料狀態通知器
class UserProfileNotifier extends StateNotifier<AsyncValue<UserProfile?>> {
  final SupabaseService _supabaseService;
  final User? _currentUser;

  UserProfileNotifier(this._supabaseService, this._currentUser)
      : super(const AsyncValue.loading()) {
    _loadUserProfile();
  }

  Future<void> updateProfile({
    String? firstName,
    String? lastName,
    // ... 其他參數
  }) async {
    if (_currentUser == null) {
      state = AsyncValue.error(
        Exception('用戶未登入'),
        StackTrace.current,
      );
      return;
    }

    state = const AsyncValue.loading();

    try {
      final updatedProfile = await _supabaseService.upsertUserProfile(
        userId: _currentUser.uid,
        firstName: firstName,
        lastName: lastName,
        // ... 其他參數
      );

      state = AsyncValue.data(updatedProfile);
    } catch (e, stack) {
      state = AsyncValue.error(e, stack);
      rethrow;
    }
  }
}
```
