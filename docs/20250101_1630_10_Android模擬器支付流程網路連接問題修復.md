## 📋 任務概述

用戶在 Android 模擬器上測試包車平台的支付流程時遇到網路連接錯誤，導致無法完成預約和支付操作。需要診斷問題根本原因並提供修復方案。

### 核心問題
- Android 模擬器無法連接到開發機器上的後端服務
- 錯誤訊息：`Connection refused (OS Error: Connection refused, errno = 111)`
- 影響範圍：預約創建、支付處理等所有需要後端 API 的功能

## 🔍 問題診斷過程

### 1. 錯誤訊息分析

**用戶提供的錯誤截圖顯示**：
```
支付失敗

Exception: 創建預約失敗: ClientException with SocketException: 
Connection refused (OS Error: Connection refused, errno = 111), 
address = localhost, port = 50156, 
uri=http://localhost:3001/api/bookings

確認
```

**關鍵資訊提取**：
1. ❌ 錯誤類型：`SocketException: Connection refused`
2. ❌ 目標地址：`localhost:3001`
3. ❌ 端點：`/api/bookings`
4. ❌ 錯誤碼：`errno = 111`（連接被拒絕）

### 2. 根本原因分析

**Android 模擬器網路特性**：

Android 模擬器運行在虛擬機中，有自己獨立的網路堆疊。在模擬器中：

| 地址 | 實際指向 | 說明 |
|------|---------|------|
| `localhost` / `127.0.0.1` | 模擬器本身 | ❌ 無法訪問開發機器 |
| `10.0.2.2` | 開發機器的 localhost | ✅ 正確配置 |
| `10.0.2.15` | 模擬器自己的 IP | - |

**問題根源**：
- 代碼中使用了 `localhost:3001`
- 在 Android 模擬器中，`localhost` 指向模擬器本身
- 模擬器本身沒有運行後端服務，因此連接被拒絕

- `pricing_service.dart` 仍在使用 `localhost`
- 錯誤訊息中的 `localhost` 來自 `pricing_service.dart`

```dart
class ApiConfig {
  static late String _baseUrl;
  static bool _initialized = false;
  
  static Future<void> initialize() async {
    if (_initialized) return;
    
    // 從環境變數讀取
    await dotenv.load(fileName: ".env");
    _baseUrl = dotenv.env['API_BASE_URL'] ?? _getDefaultBaseUrl();
    
    // 驗證配置
    if (!_isValidUrl(_baseUrl)) {
      throw Exception('Invalid API base URL: $_baseUrl');
    }
    
    _initialized = true;
  }
  
  static String get baseUrl {
    if (!_initialized) {
      throw Exception('ApiConfig not initialized');
    }
    return _baseUrl;
  }
  
  static String _getDefaultBaseUrl() {
    if (Platform.isAndroid) {
      return 'http://10.0.2.2:3001/api';
    } else {
      return 'http://localhost:3001/api';
    }
  }
  
  static bool _isValidUrl(String url) {
    try {
      Uri.parse(url);
      return true;
    } catch (e) {
      return false;
    }
  }
}
```
