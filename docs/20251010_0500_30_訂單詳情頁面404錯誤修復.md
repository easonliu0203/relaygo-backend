## 📋 問題描述

### 當前狀況

1. 在「所有訂單」頁面（`web-admin/src/app/orders/page.tsx`）點擊「查看詳情」按鈕後，出現 404 錯誤
2. 在「待處理訂單」頁面（`web-admin/src/app/orders/pending/page.tsx`）點擊「查看詳情」按鈕後，出現 404 錯誤

### 錯誤訊息

```
404
This page could not be found.
```

### 影響

- 無法查看訂單的詳細資料
- 無法看到客戶 UID
- 無法看到司機 UID（成功配對後）
- 無法看到其他訂單詳細資訊

### 1. 創建訂單詳情 API

**文件**: `web-admin/src/app/api/admin/bookings/[id]/route.ts`

**功能**:
- 獲取訂單詳情
- 包含客戶資料（從 users 和 user_profiles 表）
- 包含司機資料（從 users、user_profiles 和 drivers 表）
- 格式化資料結構

**關鍵代碼**:
```typescript
export async function GET(
  request: NextRequest,
  { params }: { params: { id: string } }
) {
  const bookingId = params.id;
  const db = new DatabaseService();

  // 獲取訂單資訊，包含客戶和司機資料
  const { data: booking, error: bookingError } = await db.supabase
    .from('bookings')
    .select(`
      *,
      customer:users!customer_id (
        id,
        email,
        user_profiles!user_id (
          full_name,
          phone
        )
      ),
      driver:users!driver_id (
        id,
        email,
        user_profiles!user_id (
          full_name,
          phone
        ),
        drivers!user_id (
          vehicle_type,
          vehicle_plate,
          rating
        )
      )
    `)
    .eq('id', bookingId)
    .single();

  // 格式化客戶資料
  const customerProfile = booking.customer?.user_profiles?.[0] || booking.customer?.user_profiles;
  const formattedCustomer = booking.customer ? {
    id: booking.customer.id,
    email: booking.customer.email,
    name: customerProfile?.full_name || '未知客戶',
    phone: customerProfile?.phone || '無電話',
  } : null;

  // 格式化司機資料
  let formattedDriver = null;
  if (booking.driver) {
    const driverProfile = booking.driver.user_profiles?.[0] || booking.driver.user_profiles;
    const driverInfo = booking.driver.drivers?.[0] || booking.driver.drivers;
    
    formattedDriver = {
      id: booking.driver.id,
      email: booking.driver.email,
      name: driverProfile?.full_name || '未知司機',
      phone: driverProfile?.phone || '無電話',
      vehicleType: driverInfo?.vehicle_type || '-',
      vehiclePlate: driverInfo?.vehicle_plate || '-',
      rating: driverInfo?.rating || 0,
    };
  }

  return NextResponse.json({
    success: true,
    data: formattedBooking,
    message: '成功獲取訂單詳情',
  });
}
```

**資料格式化**:
- 將資料庫欄位名稱（snake_case）轉換為前端欄位名稱（camelCase）
- 處理嵌套的關聯資料（customer、driver）
- 提供預設值避免顯示 undefined

## 🐛 遇到的困難和解決方法

### 困難 1: 處理嵌套的關聯資料

**問題**: Supabase 返回的關聯資料可能是陣列或物件

**解決**:
```typescript
// 使用可選鏈和陣列索引處理
const customerProfile = booking.customer?.user_profiles?.[0] || booking.customer?.user_profiles;
```

### 困難 2: 資料庫欄位名稱轉換

**問題**: 資料庫使用 snake_case，前端使用 camelCase

**解決**:
```typescript
// 手動轉換欄位名稱
const formattedBooking = {
  bookingNumber: booking.booking_number,
  customerId: booking.customer_id,
  driverId: booking.driver_id,
  // ...
};
```

### 困難 3: 動態路由參數獲取

**問題**: 需要從 URL 中獲取訂單 ID

**解決**:
```typescript
import { useParams } from 'next/navigation';

const params = useParams();
const orderId = params.id as string;
```

## 📝 犯過的錯誤和教訓

### 錯誤 1: 忘記創建 API 路由

**錯誤**:
- 只創建了頁面，沒有創建對應的 API 路由
- 導致 API 調用失敗

**教訓**:
- 創建頁面時，同時檢查 API 端點是否存在
- 確保前後端路由一致

### 錯誤 2: 沒有處理空狀態

**錯誤**:
- 沒有處理訂單不存在的情況
- 沒有處理司機未分配的情況

**教訓**:
- 使用 `Empty` 組件顯示空狀態
- 使用條件渲染處理不同情況

### 錯誤 3: 沒有提供預設值

**錯誤**:
- 直接顯示資料，可能顯示 undefined

**教訓**:
- 使用 `||` 運算符提供預設值
- 使用 `'-'` 表示無資料
