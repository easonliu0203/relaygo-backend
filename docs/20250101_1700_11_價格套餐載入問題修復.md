# 價格套餐載入問題修復

**日期**: 2025-01-01 17:00  
**編號**: 11  
**主旨**: Android 模擬器價格套餐持續載入問題診斷與修復  
**開發者**: AI Assistant  

## 📋 任務概述

用戶在 Android 模擬器上測試包車平台客戶端應用時，「選擇方案」頁面（車型套餐選擇頁面）持續顯示「載入中」狀態，車型套餐列表無法正常展示。

### 核心問題
- 車型套餐列表一直處於 loading 狀態
- 頁面沒有顯示任何錯誤訊息
- 用戶無法選擇車型套餐繼續預約流程

### 前置條件
- 已完成 Android 模擬器網路配置修復（編號 10）
- `pricing_service.dart` 的 API URL 已修改為 `10.0.2.2:3001`
- 應用已重新建置並運行

## 🛠️ 具體修復方案

### 修復 1：添加請求超時處理

**目的**：即使後端服務未啟動，也能快速降級到模擬資料

**修改文件**：`mobile/lib/core/services/pricing_service.dart`

**添加 import**：
```dart
import 'dart:async';  // 用於 TimeoutException
import 'dart:io';     // 用於 SocketException
```

**修改 getAvailablePackages() 方法**：

```dart
Future<List<VehiclePackage>> getAvailablePackages() async {
  try {
    debugPrint('[PricingService] 開始獲取價格配置');
    debugPrint('[PricingService] API URL: $_baseUrl/pricing/packages');
    
    // 添加 5 秒超時
    final response = await http.get(
      Uri.parse('$_baseUrl/pricing/packages'),
      headers: {'Content-Type': 'application/json'},
    ).timeout(
      const Duration(seconds: 5),
      onTimeout: () {
        debugPrint('[PricingService] API 請求超時，使用模擬資料');
        throw TimeoutException('API 請求超時');
      },
    );

    debugPrint('[PricingService] API 回應狀態: ${response.statusCode}');

    if (response.statusCode == 200) {
      final data = json.decode(response.body);
      debugPrint('[PricingService] API 回應資料: $data');
      
      if (data['success'] == true && data['data'] != null) {
        final List<dynamic> packagesJson = data['data'];
        final packages = packagesJson
            .map((json) => VehiclePackage.fromJson(json))
            .toList();
        debugPrint('[PricingService] 成功獲取 ${packages.length} 個套餐');
        return packages;
      } else {
        debugPrint('[PricingService] API 回應格式錯誤，使用模擬資料');
      }
    } else {
      debugPrint('[PricingService] API 請求失敗: ${response.statusCode}');
    }
  } on TimeoutException catch (e) {
    debugPrint('[PricingService] 請求超時: $e');
  } on SocketException catch (e) {
    debugPrint('[PricingService] 網路連接失敗: $e');
  } catch (e) {
    debugPrint('[PricingService] 獲取價格配置失敗: $e');
  }

  // 降級到模擬資料
  debugPrint('[PricingService] 使用模擬資料');
  return _getMockPackages();
}
```

**改進點**：
1. ✅ 添加 5 秒超時限制
2. ✅ 詳細的日誌記錄
3. ✅ 區分不同類型的錯誤（超時、網路連接、其他）
4. ✅ 快速降級到模擬資料

## 🚧 遇到的困難和解決方法

### 困難 1：診斷問題花費時間

**問題描述**：
- 用戶只報告「持續載入」
- 沒有錯誤訊息
- 需要逐步排查可能的原因

**解決方法**：
- 系統性地檢查每個環節
- 使用 curl 測試後端服務
- 添加詳細的日誌記錄

**經驗**：
- 建立標準的診斷流程
- 使用工具輔助診斷
- 不要假設，要驗證

### 困難 2：平衡超時時間

**問題描述**：
- 超時太短：正常請求可能被中斷
- 超時太長：用戶等待時間過長
- 需要找到合適的平衡點

**解決方法**：
- 設定 5 秒超時（合理的等待時間）
- 本地網路請求通常在 1-2 秒內完成
- 5 秒足夠處理網路延遲

**經驗**：
- 根據實際場景調整超時時間
- 本地開發：5-10 秒
- 生產環境：10-30 秒
- 可以通過配置文件調整

### 困難 3：確保降級機制可靠

**問題描述**：
- 降級機制必須在任何情況下都能工作
- 模擬資料必須與真實資料格式一致
- 需要測試各種錯誤場景

**解決方法**：
- 測試多種錯誤場景（超時、網路錯誤、服務錯誤）
- 確保模擬資料格式正確
- 添加詳細的日誌記錄

**經驗**：
- 降級機制是容錯設計的關鍵
- 必須經過充分測試
- 模擬資料要定期更新

## ❌ 犯過的錯誤和教訓

### 錯誤 1：假設後端服務已啟動

**錯誤描述**：
- 在前一個任務中修復了網路配置
- 假設後端服務一直在運行
- 沒有驗證後端服務狀態

**教訓**：
- 不要假設，要驗證
- 每次診斷問題都要檢查基礎設施
- 使用工具（如 curl）快速驗證

**改進措施**：
- 創建後端服務啟動指南
- 添加服務狀態檢查腳本
- 在文檔中強調前置條件

### 錯誤 2：沒有設定超時時間

**錯誤描述**：
- 原始代碼沒有超時設定
- 依賴 HTTP 客戶端的默認超時（可能很長）
- 導致用戶長時間等待

**教訓**：
- 所有網路請求都應該設定超時
- 超時時間要根據場景調整
- 超時後要有明確的處理策略

**改進措施**：
```dart
// 總是添加超時處理
final response = await http.get(url).timeout(
  const Duration(seconds: 5),
  onTimeout: () => throw TimeoutException('請求超時'),
);
```

### 錯誤 3：日誌記錄不夠詳細

**錯誤描述**：
- 原始代碼只有簡單的錯誤日誌
- 無法快速定位問題
- 診斷問題需要添加臨時日誌

**教訓**：
- 關鍵步驟都要記錄日誌
- 日誌要包含足夠的上下文資訊
- 使用統一的日誌格式

**改進措施**：
```dart
debugPrint('[ServiceName] 操作描述');
debugPrint('[ServiceName] 關鍵資訊: $value');
```
