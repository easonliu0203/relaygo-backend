# 封測版本快速啟動指南

**目標**: 快速部署封測版本，跳過複雜的金流整合，專注於核心業務邏輯驗證

## 🚀 快速啟動 (5 分鐘)

### 1. 環境設定
```bash
# 1. 複製環境變數
cp .env.example .env

# 2. 設定封測模式
echo "PAYMENT_PROVIDER=mock" >> .env
echo "PAYMENT_TEST_MODE=true" >> .env
echo "NODE_ENV=development" >> .env
```

### 2. 一鍵啟動
```bash
# 執行快速設定腳本
./scripts/setup.sh

# 啟動封測環境
./scripts/dev.sh --beta
```

### 3. 驗證啟動
- 🌐 後端 API: http://localhost:3000/health
- 💻 Web 後台: http://localhost:3001
- 📱 Flutter 應用: 在模擬器中運行

```typescript

  provider: 'mock',
```

## 🐛 常見問題

### Q1: 模擬支付總是失敗
```bash
# 檢查配置
echo $MOCK_PAYMENT_SUCCESS_RATE
# 應該顯示 0.9 (90% 成功率)

# 重設配置
export MOCK_PAYMENT_SUCCESS_RATE=1.0  # 100% 成功
```

### Q2: 線下支付無法確認
```bash
# 檢查管理員權限
curl -X GET http://localhost:3000/api/admin/payments/pending \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 手動確認支付
curl -X POST http://localhost:3000/api/admin/payments/confirm \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"transactionId": "offline_xxx", "adminId": "admin-uuid"}'
```

### Q3: 司機無法接收派單
```bash
# 檢查司機狀態
curl -X GET http://localhost:3000/api/drivers/profile \
  -H "Authorization: Bearer $DRIVER_TOKEN"

# 更新可用狀態
curl -X PUT http://localhost:3000/api/drivers/availability \
  -H "Authorization: Bearer $DRIVER_TOKEN" \
  -d '{"isAvailable": true}'
```
