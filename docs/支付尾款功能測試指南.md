## 🐛 常見問題排查

### 問題 1: 支付尾款頁面沒有自動跳轉

**可能原因**:
- 路由配置錯誤
- `context.push()` 調用失敗

**排查步驟**:
1. 檢查 `order_detail_page.dart` 的 `endTrip()` 成功回調
2. 確認有 `context.push('/payment-balance/${order.id}');`
3. 檢查路由配置是否正確

**解決方案**:
```dart
// 在 endTrip() 成功後
if (context.mounted) {
  context.push('/payment-balance/${order.id}');
}
```

### 問題 2: 尾款金額計算錯誤

**可能原因**:
- Backend 計算邏輯錯誤
- 訂金金額未正確保存

**排查步驟**:
1. 查看 Supabase `bookings` 表的 `total_amount` 和 `deposit_amount`
2. 檢查 Backend API 的計算邏輯
3. 查看 Backend 終端機日誌

**預期計算**:
```
尾款 = 總金額 - 訂金
45 = 65 - 20
```

### 問題 3: 支付記錄未創建

**可能原因**:
- Backend API 錯誤
- 資料庫連接問題

**排查步驟**:
1. 查看 Backend 終端機日誌
2. 檢查 Supabase `payments` 表
3. 確認 API 響應狀態碼

**解決方案**:
- 檢查 Backend 日誌中的錯誤訊息
- 確認資料庫連接正常
- 重新啟動 Backend 服務

### 問題 4: Firestore 狀態未更新

**可能原因**:
- Edge Function 未執行
- Outbox 事件未處理

**排查步驟**:
1. 查看 Supabase `outbox` 表
2. 確認有 `trip_ended` 和 `completed` 事件
3. 檢查 Edge Function 日誌

**解決方案**:
- 等待 1-2 分鐘（Cron Job 每分鐘執行一次）
- 查看 Edge Function 日誌確認執行
- 手動觸發 Edge Function（如果需要）
