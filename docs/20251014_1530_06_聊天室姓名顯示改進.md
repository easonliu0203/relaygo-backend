## 📋 需求描述

### 當前問題

聊天頁面顯示的用戶姓名是從 Email 截取的預設值：
- `customerName`: "customer.test" (從 `customer.test@relaygo.com` 截取)
- `driverName`: "driver.test" (從 `driver.test@relaygo.com` 截取)

### 改進需求

希望聊天室顯示用戶在「個人檔案 > 編輯個人資料」中填寫的真實姓名。

### 具體要求

1. **優先使用真實姓名**：
   - 從 Supabase `user_profiles` 表查詢用戶的 `first_name` 和 `last_name`
   - 組合成完整姓名（格式：`last_name + first_name`）

2. **降級策略（Fallback）**：
   - 如果用戶尚未填寫個人資料（`first_name` 或 `last_name` 為空）
   - 則使用當前邏輯：從 Email 截取用戶名（`email.split('@')[0]`）

3. **預期結果**：
   - 如果客戶已填寫姓名：顯示 "張三" 或 "San Zhang"（根據資料庫內容）
   - 如果客戶未填寫姓名：顯示 "customer.test"（從 Email 截取）

## 📚 開發心得和經驗總結

### 1. 降級策略的重要性

**教訓**: 永遠不要假設資料完整性

**最佳實踐**:
```typescript
// ✅ 好的實作：多層降級策略
let customerName = '客戶';
if (customerProfile?.first_name && customerProfile?.last_name) {
  customerName = `${customerProfile.last_name}${customerProfile.first_name}`;
} else if (customerProfile?.first_name) {
  customerName = customerProfile.first_name;
} else if (customerProfile?.last_name) {
  customerName = customerProfile.last_name;
} else if (customer.email) {
  customerName = customer.email.split('@')[0];
}

// ❌ 壞的實作：沒有降級策略
const customerName = `${customerProfile.last_name}${customerProfile.first_name}`;
// 如果 customerProfile 為 null，會拋出錯誤
```

### 2. 姓名格式的文化差異

**中文姓名**: `姓 + 名`（例如：張三）  
**英文姓名**: `名 + 姓`（例如：San Zhang）

**當前實作**: 使用 `last_name + first_name` 格式（適合中文）

**未來改進**: 可以根據語言設置調整順序

### 3. 查詢優化

**當前實作**: 兩次獨立查詢
```typescript
const { data: customerProfile } = await supabase
  .from('user_profiles')
  .select('first_name, last_name')
  .eq('user_id', booking.customer_id)
  .single();

const { data: driverProfile } = await supabase
  .from('user_profiles')
  .select('first_name, last_name')
  .eq('user_id', driver.id)
  .single();
```

**未來優化**: 可以使用 JOIN 或批量查詢減少請求次數

## 🚨 遇到的困難和解決方法

### 困難 1: 確定查詢鍵

**問題**: 不確定應該使用 Firebase UID 還是 Supabase UUID 查詢 `user_profiles`

**解決方法**:
1. 檢查 `user_profiles` 表結構
2. 確認 `user_id` 欄位是外鍵，指向 `users.id` (Supabase UUID)
3. 使用 Supabase UUID 查詢

### 困難 2: 姓名組合邏輯

**問題**: 如何處理只填寫名字或只填寫姓氏的情況

**解決方法**:
1. 使用多層 if-else 判斷
2. 優先級：完整姓名 > 名字 > 姓氏 > Email 截取
3. 確保每種情況都有合理的顯示

### 困難 3: 錯誤處理

**問題**: 如果查詢 `user_profiles` 失敗，是否應該中斷聊天室創建

**解決方法**:
1. 查詢失敗不拋出錯誤
2. 使用降級策略（Email 截取）
3. 確保聊天室創建流程不受影響

## 💡 犯過的錯誤和教訓

### 錯誤 1: 忘記處理 null 值

**錯誤代碼**:
```typescript
const customerName = `${customerProfile.last_name}${customerProfile.first_name}`;
```

**問題**: 如果 `customerProfile` 為 null，會拋出錯誤

**修正**:
```typescript
if (customerProfile?.first_name && customerProfile?.last_name) {
  customerName = `${customerProfile.last_name}${customerProfile.first_name}`;
}
```

### 錯誤 2: 使用錯誤的查詢鍵

**錯誤代碼**:
```typescript
.eq('user_id', customer.firebase_uid)  // ❌ 錯誤：firebase_uid
```

**問題**: `user_profiles.user_id` 是 Supabase UUID，不是 Firebase UID

**修正**:
```typescript
.eq('user_id', booking.customer_id)  // ✅ 正確：Supabase UUID
```
