# 訂單列表過濾邏輯修復

## 🐛 問題描述

### 問題：「我的訂單 > 進行中」頁面缺少新狀態的訂單

- **預期行為**：當訂單狀態為 `awaitingDriver`（待司機確認）或 `awaitingBalance`（待付尾款）時，應該在「我的訂單 > 進行中」頁面顯示
- **實際行為**：訂單不顯示在「進行中」頁面，用戶找不到訂單
- **影響範圍**：客戶端和司機端都有這個問題

---

## 🔍 根本原因

### Firestore 查詢過濾邏輯缺少新狀態

在 `mobile/lib/core/services/booking_service.dart` 中，`getActiveBookings()` 和 `getDriverActiveBookings()` 方法的 Firestore 查詢條件只包含了舊的三個狀態：

```dart
// 修改前（缺少新狀態）
.where('status', whereIn: [
  BookingStatus.pending.name,
  BookingStatus.matched.name,
  BookingStatus.inProgress.name,
])
```

**問題**：
- ❌ 缺少 `awaitingDriver`（待司機確認）
- ❌ 缺少 `awaitingBalance`（待付尾款）

**結果**：
- 當訂單狀態為 `awaitingDriver` 時，Firestore 查詢不會返回這個訂單
- 當訂單狀態為 `awaitingBalance` 時，Firestore 查詢不會返回這個訂單
- 用戶在「我的訂單 > 進行中」頁面看不到這些訂單

## ✅ 修復方案

### 修改檔案：`mobile/lib/core/services/booking_service.dart`

#### 1. 客戶端進行中訂單查詢（第 343-373 行）

**修改前**：
```dart
Stream<List<BookingOrder>> getActiveBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  return _firestore
      .collection('orders_rt')
      .where('customerId', isEqualTo: currentUserId)
      .where('status', whereIn: [
        BookingStatus.pending.name,
        BookingStatus.matched.name,
        BookingStatus.inProgress.name,
      ])
      .orderBy('createdAt', descending: true)
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());
}
```

**修改後**：
```dart
/// 獲取進行中的訂單（從 Firestore 鏡像讀取）
/// 用於即時畫面展示，資料來自 Supabase 的單向鏡像
/// 
/// 進行中訂單包含以下狀態：
/// - pending: 待配對（待付訂金或待派單）
/// - awaitingDriver: 待司機確認（已分配司機，等待司機確認接單）
/// - matched: 已配對（司機已確認接單）
/// - inProgress: 行程進行中
/// - awaitingBalance: 待付尾款（行程已結束，等待支付尾款）
Stream<List<BookingOrder>> getActiveBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  return _firestore
      .collection('orders_rt')
      .where('customerId', isEqualTo: currentUserId)
      .where('status', whereIn: [
        BookingStatus.pending.name,
        BookingStatus.awaitingDriver.name,      // ⭐ 新增：待司機確認
        BookingStatus.matched.name,
        BookingStatus.inProgress.name,
        BookingStatus.awaitingBalance.name,     // ⭐ 新增：待付尾款
      ])
      .orderBy('createdAt', descending: true)
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());
}
```

```dart
/// 獲取司機的進行中訂單（從 Firestore 鏡像讀取）
/// 用於司機端即時畫面展示，資料來自 Supabase 的單向鏡像
///
/// 進行中訂單包含以下狀態：
/// - pending: 待配對（司機可以看到待接單的訂單）
/// - awaitingDriver: 待司機確認（已分配司機，等待司機確認接單）
/// - matched: 已配對（司機已確認接單）
/// - inProgress: 行程進行中
/// - awaitingBalance: 待付尾款（行程已結束，等待客戶支付尾款）
///
/// 不包含：
/// - completed: 已完成
/// - cancelled: 已取消
Stream<List<BookingOrder>> getDriverActiveBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  return _firestore
      .collection('orders_rt')
      .where('driverId', isEqualTo: currentUserId)
      .where('status', whereIn: [
        BookingStatus.pending.name,
        BookingStatus.awaitingDriver.name,      // ⭐ 新增：待司機確認
        BookingStatus.matched.name,
        BookingStatus.inProgress.name,
        BookingStatus.awaitingBalance.name,     // ⭐ 新增：待付尾款
      ])
      .orderBy('createdAt', descending: true)
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());
}
```

## 📝 技術細節

### Firestore 查詢限制

Firestore 的 `whereIn` 查詢最多支持 10 個值。目前我們使用了 5 個狀態值，仍在限制範圍內：

```dart
.where('status', whereIn: [
  BookingStatus.pending.name,           // 1
  BookingStatus.awaitingDriver.name,    // 2
  BookingStatus.matched.name,           // 3
  BookingStatus.inProgress.name,        // 4
  BookingStatus.awaitingBalance.name,   // 5
])
```

如果未來需要添加更多狀態，可能需要考慮其他查詢策略（例如：使用 `whereNotIn` 排除已完成和已取消的訂單）。

### 為什麼不包含 `completed` 和 `cancelled`？

這兩個狀態的訂單應該顯示在「歷史訂單」頁面，而不是「進行中」頁面。相關方法：
- 客戶端：`getUserCompletedBookings()`（第 452-471 行）
- 司機端：`getDriverCompletedBookings()`（第 432-451 行）
