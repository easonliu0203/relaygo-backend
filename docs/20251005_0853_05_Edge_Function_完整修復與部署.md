## 📋 問題描述

### 初始問題
用戶報告手機 App 顯示「訂單不存在」錯誤，但在 Supabase Dashboard 中確認：
- ✅ `bookings` 表中有新訂單記錄
- ✅ `outbox` 表中有對應的事件記錄

這表示訂單創建成功，Trigger 正常工作，但同步到 Firestore 失敗。

### 用戶提供的代碼
用戶提供了當前部署的 Edge Function 代碼，發現是**舊版本**（未修復）：

```typescript
// ❌ 錯誤的代碼（舊版本）
if (event.aggregate_type === 'order') {
  await syncOrderToFirestore(event);
}
```

**問題**：
- Trigger 創建的是 `'booking'` 類型
- Edge Function 檢查的是 `'order'` 類型
- 結果：Edge Function 永遠不會處理任何事件

---

## 🔍 診斷過程

### 步驟 1：確認問題根源

**檢查 Trigger 代碼**（`supabase/migrations/20250101_create_outbox_table.sql`）：
```sql
INSERT INTO outbox (
  aggregate_type,
  aggregate_id,
  event_type,
  payload
) VALUES (
  'booking',  -- ✅ Trigger 創建 'booking' 類型
  NEW.id::TEXT,
  ...
);
```

**檢查 Edge Function 代碼**（用戶提供的舊版本）：
```typescript
if (event.aggregate_type === 'order') {  // ❌ 檢查 'order' 類型
  await syncOrderToFirestore(event);
}
```

**結論**：類型不匹配導致事件永遠不會被處理。

### 步驟 2：確認 payload 欄位不匹配

**Trigger 提供的欄位**：
```typescript
{
  bookingNumber, destination, startDate, startTime,
  durationHours, vehicleType, ...
}
```

**舊版 Edge Function 期望的欄位**：
```typescript
{
  dropoffAddress, bookingTime, passengerCount, ...
}
```

**結論**：即使類型匹配，欄位不匹配也會導致同步失敗。

---

## 🛠️ 修復方案

### 修復 1：aggregate_type 檢查

**修改前**：
```typescript
if (event.aggregate_type === 'order') {
  await syncOrderToFirestore(event);
}
```

**修改後**：
```typescript
if (event.aggregate_type === 'booking') {
  await syncBookingToFirestore(event);
}
```

### 修復 2：函數重命名

**修改前**：
```typescript
async function syncOrderToFirestore(event: OutboxEvent): Promise<void> {
  const orderId = event.aggregate_id;
  const orderData = event.payload;
  ...
}
```

**修改後**：
```typescript
async function syncBookingToFirestore(event: OutboxEvent): Promise<void> {
  const bookingId = event.aggregate_id;
  const bookingData = event.payload;
  ...
}
```

### 修復 3：payload 欄位映射

**修改前**：
```typescript
const firestoreData = {
  dropoffAddress: orderData.dropoffAddress,  // ❌ 欄位不存在
  bookingTime: orderData.bookingTime,        // ❌ 欄位不存在
  passengerCount: orderData.passengerCount,  // ❌ 欄位不存在
  ...
};
```

**修改後**：
```typescript
const firestoreData = {
  id: bookingData.id,
  bookingNumber: bookingData.bookingNumber,      // ✅ 匹配 Trigger
  destination: bookingData.destination,          // ✅ 匹配 Trigger
  startDate: bookingData.startDate,              // ✅ 匹配 Trigger
  startTime: bookingData.startTime,              // ✅ 匹配 Trigger
  durationHours: bookingData.durationHours,      // ✅ 匹配 Trigger
  vehicleType: bookingData.vehicleType,          // ✅ 匹配 Trigger
  specialRequirements: bookingData.specialRequirements || '',
  requiresForeignLanguage: bookingData.requiresForeignLanguage || false,
  basePrice: bookingData.basePrice,
  foreignLanguageSurcharge: bookingData.foreignLanguageSurcharge || 0,
  overtimeFee: bookingData.overtimeFee || 0,
  tipAmount: bookingData.tipAmount || 0,
  totalAmount: bookingData.totalAmount,
  depositAmount: bookingData.depositAmount,
  pickupLocation: bookingData.pickupLocation || { latitude: 0, longitude: 0 },
  createdAt: bookingData.createdAt,
  updatedAt: bookingData.updatedAt,
  driverId: bookingData.driverId || null,
  actualStartTime: bookingData.actualStartTime || null,
  actualEndTime: bookingData.actualEndTime || null,
};
```

### 驗證查詢

創建了快速驗證查詢（`supabase/quick-verify.sql`）：
```sql
SELECT 
  b.id as booking_id,
  b.booking_number,
  b.status,
  o.processed_at,
  CASE 
    WHEN o.processed_at IS NOT NULL THEN '✅ 已同步'
    WHEN o.processed_at IS NULL AND EXTRACT(EPOCH FROM (NOW() - o.created_at)) < 60 THEN '⏳ 等待中'
    WHEN o.processed_at IS NULL THEN '❌ 未同步'
  END as sync_status,
  o.error_message
FROM bookings b
LEFT JOIN outbox o ON o.payload->>'id' = b.id::TEXT
WHERE b.created_at >= NOW() - INTERVAL '1 hour'
ORDER BY b.created_at DESC
LIMIT 10;
```
