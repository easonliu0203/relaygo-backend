## 🔍 診斷過程

### 步驟 1: 分析查詢代碼

**文件**: `mobile/lib/core/services/booking_service.dart`

**查詢 1 - getDriverBookings()** (第 367-382 行):
```dart
Stream<List<BookingOrder>> getDriverBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  return _firestore
      .collection('orders_rt')
      .where('driverId', isEqualTo: currentUserId)  // 條件 1
      .orderBy('createdAt', descending: true)       // 排序
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());
}
```

**需要的索引**:
- `driverId` (ASCENDING) + `createdAt` (DESCENDING)

**查詢 2 - getDriverActiveBookings()** (第 386-405 行):
```dart
Stream<List<BookingOrder>> getDriverActiveBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  return _firestore

      ])
      .orderBy('createdAt', descending: true)       // 排序
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());
}
```

**需要的索引**:
- `driverId` (ASCENDING) + `status` (ASCENDING) + `createdAt` (DESCENDING)

### 步驟 2: 檢查現有索引配置

**文件**: `firebase/firestore.indexes.json`

**發現**:
- ✅ 已有 `customerId` + `createdAt` 索引（第 46-58 行）
- ✅ 已有 `customerId` + `status` + `createdAt` 索引（第 73-90 行）
- ❌ **缺少** `driverId` + `createdAt` 索引
- ❌ **缺少** `driverId` + `status` + `createdAt` 索引

**問題分析**:
- 之前只為客戶端（customerId）創建了索引
- 沒有為司機端（driverId）創建對應的索引
- 導致司機端查詢失敗

## 🔧 修復方案

### 添加司機端查詢所需的索引

**文件**: `firebase/firestore.indexes.json`

**添加的索引 1** - 司機訂單列表:
```json
{
  "collectionGroup": "orders_rt",
  "queryScope": "COLLECTION",
  "fields": [
    {
      "fieldPath": "driverId",
      "order": "ASCENDING"
    },
    {
      "fieldPath": "createdAt",
      "order": "DESCENDING"
    }
  ]
}
```

**用途**: 支持 `getDriverBookings()` 查詢

## 🔍 索引配置詳解

### 完整的 orders_rt 索引列表

| 索引 | 欄位 | 用途 |
|------|------|------|
| 1 | `customerId` + `createdAt` | 客戶訂單列表 |
| 2 | `status` + `createdAt` | 按狀態查詢訂單 |
| 3 | `customerId` + `status` + `createdAt` | 客戶特定狀態訂單 |
| 4 | `driverId` + `createdAt` | **司機訂單列表** ⭐ 新增 |
| 5 | `driverId` + `status` + `createdAt` | **司機進行中訂單** ⭐ 新增 |

### 索引設計原則

**複合索引規則**:
1. **等值查詢欄位在前**: `where('field', '==', value)`
2. **範圍查詢欄位在中**: `where('field', 'in', [...])`
3. **排序欄位在後**: `orderBy('field')`

**範例**:
```dart
// 查詢
.where('driverId', isEqualTo: uid)     // 等值 → 第一個欄位
.where('status', whereIn: [...])       // 範圍 → 第二個欄位
.orderBy('createdAt', descending: true) // 排序 → 第三個欄位

// 對應索引
{
  "fields": [
    { "fieldPath": "driverId", "order": "ASCENDING" },
    { "fieldPath": "status", "order": "ASCENDING" },
    { "fieldPath": "createdAt", "order": "DESCENDING" }
  ]
}
```

## 💡 開發心得

### 1. Firestore 複合查詢需要索引

**重點**:
- 任何包含多個欄位的查詢都需要複合索引
- `where` + `orderBy` 組合需要索引
- 多個 `where` 條件需要索引

**最佳實踐**:
- 在開發階段就規劃好索引
- 使用 Firebase Emulator 測試查詢
- 提前創建所需的索引

### 2. 索引建立需要時間

**注意事項**:
- 索引部署後不會立即生效
- 需要等待 2-5 分鐘（或更長）
- 在索引建立完成前，查詢會失敗

**建議**:
- 在非高峰時段部署索引
- 提前通知用戶可能的服務中斷
- 監控索引建立進度

### 3. 對稱設計索引

**問題**:
- 客戶端和司機端有相似的查詢需求
- 但索引配置不對稱

**解決方案**:
- 為客戶端和司機端創建對稱的索引
- `customerId` 索引 → `driverId` 索引
- 確保兩端都能正常查詢

**範例**:
```json
// 客戶端索引
{ "fieldPath": "customerId", ... }

// 司機端索引（對稱）
{ "fieldPath": "driverId", ... }
```
