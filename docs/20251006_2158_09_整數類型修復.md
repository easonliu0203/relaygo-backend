## 🔍 問題診斷過程

### 1. 分析錯誤訊息

**錯誤**：`type 'double' is not a subtype of type 'int'`

**含義**：
- Dart 嘗試將 double 類型的值賦值給 int 類型的變數
- Dart 是強類型語言，不會自動將 double 轉換為 int
- 必須明確轉換：`value.toInt()`

**推論**：
- 某個欄位在 Firestore 中是 double 類型
- 但客戶端期望 int 類型
- 導致類型轉換失敗

---

### 2. 檢查客戶端期望

**檔案**：`mobile/lib/core/models/booking_order.dart`

**客戶端模型定義**：
```dart
const factory BookingOrder({
  ...
  required int passengerCount,    // ✅ 期望 int
  int? luggageCount,              // ✅ 期望 int
  required double estimatedFare,  // ✅ 期望 double
  required double depositAmount,  // ✅ 期望 double
  ...
}) = _BookingOrder;
```

**客戶端解析代碼**：
```dart
factory BookingOrder.fromFirestore(DocumentSnapshot doc) {
  final data = doc.data() as Map<String, dynamic>;
  
  return BookingOrder(
    ...
    passengerCount: data['passengerCount'] ?? 1,  // ❌ 直接使用，沒有類型轉換
    luggageCount: data['luggageCount'],           // ❌ 直接使用，沒有類型轉換
    estimatedFare: (data['estimatedFare'] ?? 0.0).toDouble(),  // ✅ 明確轉換為 double
    depositAmount: (data['depositAmount'] ?? 0.0).toDouble(),  // ✅ 明確轉換為 double
    ...
  );
}
```

**發現**：
- `passengerCount` 和 `luggageCount` 期望 **int** 類型
- 但解析代碼沒有明確轉換類型
- 如果 Firestore 存儲為 double，會導致類型錯誤

---

### 3. 檢查 Edge Function 的資料類型處理

**檔案**：`supabase/functions/sync-to-firestore/index.ts`

**原始代碼**（第 409-410 行）：
```typescript
} else if (typeof value === 'number') {
  fields[key] = { doubleValue: value }  // ❌ 所有數值都是 double

```

**問題**：
- 所有數值（包括整數）都被轉換為 `doubleValue`
- Firestore 存儲為 double 類型
- 客戶端期望 int 類型的欄位會解析失敗

### 4. 檢查 Firestore 資料

**Edge Function 日誌**：
```json
{
  "passengerCount": 1,        // JavaScript 中是整數
  "estimatedFare": 0,         // JavaScript 中是整數
  "depositAmount": 0          // JavaScript 中是整數
}
```

**轉換為 Firestore REST API 格式**（修復前）：
```json
{
  "passengerCount": { "doubleValue": 1 },     // ❌ 轉換為 double
  "estimatedFare": { "doubleValue": 0 },      // ✅ 應該是 double（正確）
  "depositAmount": { "doubleValue": 0 }       // ✅ 應該是 double（正確）
}
```

**Firestore 存儲**（修復前）：
```
passengerCount: 1  ← number (double) 類型 ❌
estimatedFare: 0   ← number (double) 類型 ✅
depositAmount: 0   ← number (double) 類型 ✅
```

**客戶端解析**：
```dart
passengerCount: data['passengerCount'] ?? 1
// data['passengerCount'] 是 double (1.0)
// 賦值給 int 類型的變數
// ❌ 錯誤：type 'double' is not a subtype of type 'int'
```

---

### 5. 根本原因

**問題**：
- Edge Function 將所有數值都轉換為 `doubleValue`
- Firestore 存儲為 double 類型
- 客戶端期望某些欄位是 int 類型
- Dart 無法自動將 double 轉換為 int
- 導致類型轉換失敗

**需要區分的欄位**：
- **整數欄位**：`passengerCount`, `luggageCount` → 應該使用 `integerValue`
- **浮點數欄位**：`estimatedFare`, `depositAmount` → 應該使用 `doubleValue`

## 🛠️ 修復實施

### 修改 1：標記整數欄位

**目的**：在 `syncBookingToFirestore` 函數中標記哪些欄位應該是整數

#### 修改前（錯誤）

```typescript
const firestoreData = {
  ...
  // 乘客資訊
  passengerCount: 1,  // ❌ 普通數值，會被轉換為 double
  luggageCount: null,
  notes: bookingData.specialRequirements || null,
  ...
}
```

---

#### 修改後（正確）✅

```typescript
const firestoreData = {
  ...
  // 乘客資訊（使用 _integer 標記，convertToFirestoreFields 會轉換為整數）
  passengerCount: {
    _integer: 1,  // ✅ 使用 _integer 標記
  },
  luggageCount: null,
  notes: bookingData.specialRequirements || null,
  ...
}
```

**關鍵變更**：
- ✅ 使用 `{ _integer: 1 }` 格式包裝整數值
- ✅ `convertToFirestoreFields` 會檢測 `_integer` 並轉換為 Firestore 整數

---

### 修改 2：更新 `convertToFirestoreFields` 函數

**目的**：
1. 檢測 `_integer` 標記並轉換為 `integerValue`
2. 自動檢測 JavaScript 整數並轉換為 `integerValue`

#### 修改前（錯誤）

```typescript
function convertToFirestoreFields(data: any): any {
  const fields: any = {}

  for (const [key, value] of Object.entries(data)) {
    ...
    } else if (typeof value === 'number') {
      fields[key] = { doubleValue: value }  // ❌ 所有數值都是 double
    }
    ...
  }

  return fields
}
```

**問題**：
- 所有數值都使用 `doubleValue`
- 沒有區分整數和浮點數

---

#### 修改後（正確）✅

```typescript
function convertToFirestoreFields(data: any): any {
  const fields: any = {}

  for (const [key, value] of Object.entries(data)) {
    ...
    } else if (typeof value === 'number') {
      // ✅ 新增：檢查是否是整數
      if (Number.isInteger(value)) {
        fields[key] = { integerValue: value.toString() }
      } else {
        fields[key] = { doubleValue: value }
      }
    }
    ...
    } else if (typeof value === 'object') {
      ...
      // ✅ 新增：檢查是否是整數格式（包含 _integer）
      else if ('_integer' in value) {
        const intValue = value._integer as number
        fields[key] = {
          integerValue: intValue.toString()
        }
      }
      ...
    }
  }

  return fields
}
```

**關鍵變更**：
1. ✅ 檢測 JavaScript 整數：`Number.isInteger(value)`
2. ✅ 整數使用 `integerValue`：`{ integerValue: value.toString() }`
3. ✅ 浮點數使用 `doubleValue`：`{ doubleValue: value }`
4. ✅ 檢測 `_integer` 標記並轉換為 `integerValue`

**注意**：
- Firestore REST API 的 `integerValue` 必須是字串格式
- 需要使用 `value.toString()` 轉換

## 💡 開發心得和經驗總結

### 1. Firestore 資料類型的細節很重要

**教訓**：
- Firestore 區分 integer 和 double
- 客戶端（特別是 Dart）是強類型語言
- 必須使用正確的資料類型

**Firestore 數值類型**：
- `integerValue`：整數（例如：1, 2, 3）
- `doubleValue`：浮點數（例如：1.5, 2.0, 3.14）

---

### 2. 強類型語言的類型轉換

**Dart 的類型系統**：
- 不會自動將 double 轉換為 int
- 必須明確轉換：`value.toInt()`
- 或確保資料類型正確

**對比其他語言**：
- JavaScript：自動轉換（`1.0 === 1` 為 true）
- Python：自動轉換（`int(1.0)` 為 1）
- Dart：不自動轉換（必須明確）

---

### 3. 錯誤訊息的價值（再次強調）

**有效的診斷**：
- 「訂單不存在」→ 「載入訂單失敗」→ `type 'double' is not a subtype of type 'int'`
- 每次錯誤訊息都更具體
- 逐步縮小問題範圍

**改進建議**：
- 在客戶端添加更詳細的錯誤訊息
- 記錄具體是哪個欄位解析失敗
- 記錄期望的類型和實際的類型

---

### 4. 遇到的困難

**困難 1：不熟悉 Firestore REST API 的整數格式**
- 最初以為 `integerValue` 可以直接使用數值
- 實際上必須是字串格式：`{ integerValue: "1" }`
- 解決：查閱 Firestore REST API 文檔

**困難 2：如何在 JavaScript 中檢測整數**
- JavaScript 的 `typeof` 無法區分整數和浮點數
- 需要使用 `Number.isInteger()` 方法
- 解決：使用 `Number.isInteger(value)` 檢測

---

### 5. 犯過的錯誤

**錯誤 1：假設所有數值都可以是 double**
- 最初將所有數值都轉換為 `doubleValue`
- 導致整數欄位也是 double 類型
- 教訓：必須區分整數和浮點數

**錯誤 2：沒有考慮客戶端的類型系統**
- 沒有考慮 Dart 的強類型特性
- 假設客戶端會自動轉換類型
- 教訓：必須了解客戶端的類型系統
