# Supabase 初始化錯誤修復

**日期**: 2025-10-10 10:30  
**問題**: 客戶端和司機端應用中 Supabase 未初始化錯誤  
**狀態**: ✅ 已修復

---

## 📋 問題描述

### 錯誤現象

在客戶端和司機端應用中，訪問個人檔案相關頁面時出現以下錯誤：

```
'package:supabase_flutter/src/supabase.dart': Failed assertion: line 45 pos 7: 
'_instance._isInitialized': You must initialize the supabase instance before calling Supabase.instance
See also: https://docs.flutter.dev/testing/errors
```

### 發生位置

1. **客戶端個人檔案頁面**
   - 文件: `mobile/lib/apps/customer/presentation/pages/customer_profile_page.dart`
   - 觸發: 訪問個人資料標籤頁時

2. **司機端編輯個人資料頁面**
   - 文件: `mobile/lib/apps/driver/presentation/pages/edit_profile_page.dart`
   - 觸發: 點擊「編輯個人資料」時

### 影響範圍

- ❌ 無法訪問個人資料頁面
- ❌ 無法編輯個人資料
- ❌ 無法保存用戶資料到 Supabase
- ❌ 應用崩潰或顯示錯誤訊息

---

## 🔍 問題診斷

### 診斷過程

#### 1. 檢查錯誤堆疊

錯誤堆疊顯示問題出在 `Supabase.instance.client` 調用時：

```dart
// mobile/lib/core/services/supabase_service.dart
class SupabaseService {
  final SupabaseClient _client;

  SupabaseService() : _client = Supabase.instance.client; // ❌ 這裡拋出錯誤
}
```

#### 2. 檢查初始化代碼

查看 `mobile/lib/main.dart`，發現已經添加了 Supabase 初始化：

```dart
void main() async {
  // ...
  
  // 初始化 Supabase
  await Supabase.initialize(
    url: dotenv.env['SUPABASE_URL'] ?? '',
    anonKey: dotenv.env['SUPABASE_ANON_KEY'] ?? '',
  );
  
  // ...
}
```

#### 3. 檢查應用入口文件

**關鍵發現**：客戶端和司機端使用的是**不同的入口文件**！

- 客戶端: `mobile/lib/apps/customer/main_customer.dart`
- 司機端: `mobile/lib/apps/driver/main_driver.dart`
- 共用文件: `mobile/lib/main.dart` （未被使用）

查看 `main_customer.dart` 和 `main_driver.dart`，發現：

```dart
void main() async {
  // ...
  
  // 初始化 Firebase Core
  await Firebase.initializeApp(
    name: 'customer', // 或 'driver'
    options: DefaultFirebaseOptions.currentPlatform,
  );
  
  // ❌ 沒有初始化 Supabase！
  
  // ...
}
```

### 根本原因

**客戶端和司機端的入口文件中沒有初始化 Supabase**，導致在調用 `Supabase.instance.client` 時拋出未初始化錯誤。

---

## 🔧 修復方案

### 修復 1: 客戶端入口文件

**文件**: `mobile/lib/apps/customer/main_customer.dart`

**修改內容**:

1. **添加導入**:
```dart
import 'package:supabase_flutter/supabase_flutter.dart';
```

2. **添加初始化代碼**:
```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  try {
    // 載入環境變數
    await dotenv.load(fileName: ".env");

    // 初始化 Supabase
    final supabaseUrl = dotenv.env['SUPABASE_URL'] ?? '';
    final supabaseAnonKey = dotenv.env['SUPABASE_ANON_KEY'] ?? '';
    
    if (supabaseUrl.isNotEmpty && supabaseAnonKey.isNotEmpty) {
      await Supabase.initialize(
        url: supabaseUrl,
        anonKey: supabaseAnonKey,
      );
      debugPrint('✅ Supabase 初始化成功');
    } else {
      debugPrint('⚠️ Supabase 配置不完整，跳過初始化');
    }

    // 初始化 Firebase Core
    await Firebase.initializeApp(
      name: 'customer',
      options: DefaultFirebaseOptions.currentPlatform,
    );

    // ... 其他初始化代碼
  } catch (e, stackTrace) {
    debugPrint('❌ 初始化過程中發生錯誤: $e');
    debugPrint('Stack trace: $stackTrace');
  }

  runApp(
    const ProviderScope(
      child: CustomerApp(),
    ),
  );
}
```

**關鍵改進**:
- ✅ 在 Firebase 之前初始化 Supabase
- ✅ 檢查配置是否完整
- ✅ 添加調試日誌輸出
- ✅ 改進錯誤處理（包含堆疊追蹤）

### 修復 2: 司機端入口文件

**文件**: `mobile/lib/apps/driver/main_driver.dart`

**修改內容**: 與客戶端相同，只是 Firebase 初始化名稱為 `'driver'`

### 修復 3: 改進 SupabaseService

**文件**: `mobile/lib/core/services/supabase_service.dart`

**修改前**:
```dart
class SupabaseService {
  final SupabaseClient _client;

  SupabaseService() : _client = Supabase.instance.client;
}
```

**修改後**:
```dart
class SupabaseService {
  late final SupabaseClient _client;

  SupabaseService() {
    try {
      _client = Supabase.instance.client;
    } catch (e) {
      throw SupabaseServiceException(
        'Supabase 未初始化。請確保在應用啟動時調用 Supabase.initialize()',
        e,
      );
    }
  }
}
```

**改進點**:
- ✅ 使用 `late` 關鍵字延遲初始化
- ✅ 添加 try-catch 錯誤處理
- ✅ 提供清晰的錯誤訊息

### 修復 4: 創建配置檢查腳本

**文件**: `mobile/scripts/check-supabase-config.bat`

**功能**:
- 檢查 `.env` 文件是否存在
- 檢查 `SUPABASE_URL` 是否配置
- 檢查 `SUPABASE_ANON_KEY` 是否配置
- 提供清晰的錯誤訊息

## 💡 開發心得

### 1. 多入口文件的陷阱

**教訓**: 在多 flavor 的 Flutter 專案中，每個 flavor 可能有自己的入口文件。修改共用的 `main.dart` 不會影響實際運行的應用。

**解決方案**:
- 檢查 `flutter run` 命令中的 `--target` 參數
- 確認實際使用的入口文件
- 在所有入口文件中添加必要的初始化代碼

### 2. 初始化順序很重要

**最佳實踐**:
```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  
  // 1. 載入環境變數
  await dotenv.load(fileName: ".env");
  
  // 2. 初始化 Supabase（需要環境變數）
  await Supabase.initialize(...);
  
  // 3. 初始化 Firebase
  await Firebase.initializeApp(...);
  
  // 4. 其他初始化
  // ...
  
  // 5. 啟動應用
  runApp(...);
}
```

### 3. 錯誤處理的重要性

**改進前**:
```dart
SupabaseService() : _client = Supabase.instance.client;
// 錯誤訊息不清楚，難以診斷
```

**改進後**:
```dart
SupabaseService() {

    _client = Supabase.instance.client;
  } catch (e) {
    throw SupabaseServiceException(
      'Supabase 未初始化。請確保在應用啟動時調用 Supabase.initialize()',
      e,
    );
  }
}
// 提供清晰的錯誤訊息和解決方案
```

### 4. 調試日誌的價值

添加調試日誌可以快速定位問題：

```dart
debugPrint('✅ Supabase 初始化成功');
debugPrint('⚠️ Supabase 配置不完整，跳過初始化');
debugPrint('❌ 初始化過程中發生錯誤: $e');
```
