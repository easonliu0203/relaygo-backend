# 客戶端行程控制按鈕功能使用指南

## 📖 功能概述

客戶端行程控制按鈕功能允許客戶在訂單詳情頁面控制行程的開始和結束，並自動同步狀態到所有端（公司端、客戶端、司機端）。

### 核心功能

1. **開始行程** - 客戶與司機見面後，點擊開始計時
2. **結束行程** - 到達目的地後，點擊結束行程並進入付款流程

## 🚀 快速開始

### 前置條件

- ✅ Backend API 正在運行（http://localhost:3000）
- ✅ Supabase 已連接
- ✅ Firebase 已配置
- ✅ Supabase Edge Function (sync-to-firestore) 正在運行
- ✅ Flutter 應用已編譯

### 部署步驟

#### 1. 重啟 Backend

```bash
cd backend
npm run dev
```

確認 Backend 正在運行：
```bash
curl http://localhost:3000/api/booking-flow/test
```

應該返回：
```json
{
  "success": true,
  "message": "Booking Flow API is working",
  "timestamp": "2025-10-15T..."
}
```

#### 2. 重新部署 Edge Function（可選）

如果修改了 Firestore 同步邏輯：
```bash
cd supabase
supabase functions deploy sync-to-firestore
```

#### 3. 編譯 Flutter 應用

```bash
cd mobile
flutter pub get
flutter run
```

## 📱 使用流程

### 完整訂單流程

```
1. 客戶下單 → 訂單狀態: pending
2. 公司派單 → 訂單狀態: assigned
3. 司機確認接單 → 訂單狀態: driver_confirmed (Firestore: matched)
4. 司機出發 → 訂單狀態: driver_departed (Firestore: inProgress)
5. 司機到達 → 訂單狀態: driver_arrived (Firestore: inProgress)
6. 【客戶開始行程】→ 訂單狀態: trip_started (Firestore: inProgress) ⭐ 新功能
7. 【客戶結束行程】→ 訂單狀態: trip_ended (Firestore: completed) ⭐ 新功能
8. 客戶支付尾款 → 訂單狀態: completed
```

### 客戶端操作步驟

#### 步驟 1: 開始行程

**顯示條件**:
- 訂單狀態為「已配對」（matched）或「進行中」（inProgress）
- 司機已確認接單或已到達上車地點

**操作流程**:
1. 打開訂單詳情頁面
2. 看到「開始行程 🚀」按鈕（綠色）
3. 點擊按鈕
4. 確認對話框：「確定要開始行程嗎？請確認您已與司機見面。」
5. 點擊「確認開始」
6. 等待載入（顯示「正在開始行程...」）
7. 成功提示：「✅ 行程已開始！」
8. 訂單自動刷新，按鈕變更為「結束行程」

**注意事項**:
- ⚠️ 請務必在與司機見面時按下，避免金額計算錯誤
- 開始行程後會記錄開始時間，用於計費

#### 步驟 2: 結束行程

**顯示條件**:
- 訂單狀態為「進行中」（inProgress）
- 行程已開始（trip_started）

**操作流程**:
1. 到達目的地後，打開訂單詳情頁面
2. 看到「結束行程 🏁」按鈕（橙色）
3. 點擊按鈕
4. 確認對話框：「確定要結束行程嗎？結束後將需要支付尾款。」
5. 點擊「確認結束」
6. 等待載入（顯示「正在結束行程...」）
7. 成功提示：「✅ 行程已結束！請支付尾款」
8. 訂單自動刷新，進入付款流程

**注意事項**:
- ⚠️ 請務必在確定結束行程時按下，避免金額計算錯誤
- 結束行程後會記錄結束時間，計算行程時長
- 可能需要支付加班費或額外費用

## 🧪 測試方法

### 方法 1: 使用 Flutter 應用測試

1. **準備測試訂單**
   - 使用客戶帳號登入
   - 找到一個狀態為「已配對」或「進行中」的訂單
   - 進入訂單詳情頁面

2. **測試開始行程**
   - 確認看到「開始行程 🚀」按鈕
   - 點擊按鈕 → 確認 → 等待成功提示
   - 檢查訂單狀態是否更新
   - 檢查聊天室是否收到系統訊息：「客戶已開始行程 🚀」

3. **測試結束行程**
   - 確認看到「結束行程 🏁」按鈕
   - 點擊按鈕 → 確認 → 等待成功提示
   - 檢查訂單狀態是否更新
   - 檢查聊天室是否收到系統訊息：「行程已結束，請支付尾款 💰」

### 方法 2: 使用 API 測試腳本

```bash
cd backend
chmod +x test-customer-trip-api.sh

# 編輯腳本，修改以下參數：
# BOOKING_ID="實際的訂單 ID"
# CUSTOMER_UID="實際的客戶 Firebase UID"

./test-customer-trip-api.sh
```

### 方法 3: 使用 curl 手動測試

**測試開始行程**:
```bash
curl -X POST http://localhost:3000/api/booking-flow/bookings/{BOOKING_ID}/start-trip \
  -H "Content-Type: application/json" \
  -d '{"customerUid": "{CUSTOMER_FIREBASE_UID}"}'
```

**測試結束行程**:
```bash
curl -X POST http://localhost:3000/api/booking-flow/bookings/{BOOKING_ID}/end-trip \
  -H "Content-Type: application/json" \
  -d '{"customerUid": "{CUSTOMER_FIREBASE_UID}"}'
```

## ❓ 常見問題

### Q1: 按鈕沒有顯示

**可能原因**:
- 訂單狀態不正確
- Firestore 同步延遲
- Flutter 應用未刷新

**解決方法**:
1. 檢查訂單狀態：
   ```dart
   debugPrint('Order status: ${order.status}');
   ```
2. 等待 5-30 秒（Firestore 同步延遲）
3. 手動刷新訂單頁面

### Q2: API 返回 403 錯誤（無權限）

**可能原因**:
- 客戶 Firebase UID 不正確
- 訂單不屬於該客戶

**解決方法**:
1. 確認客戶已登入
2. 確認訂單的 customer_id 與客戶匹配
3. 檢查 Backend 日誌

### Q3: API 返回 400 錯誤（狀態不正確）

**可能原因**:
- 訂單狀態不符合要求
- 重複點擊按鈕

**解決方法**:
1. 檢查訂單當前狀態
2. 確認狀態流程正確
3. 刷新訂單頁面

### Q4: 系統訊息沒有發送

**可能原因**:
- Firebase Admin SDK 配置錯誤
- Firestore 安全規則限制

**解決方法**:
1. 檢查 Backend 日誌
2. 確認 Firebase 憑證正確
3. 檢查 Firestore 安全規則

### Q5: Firestore 沒有同步

**可能原因**:
- Edge Function 未運行
- Outbox 表未寫入

**解決方法**:
1. 檢查 Edge Function 狀態
2. 查看 outbox 表：
   ```sql
   SELECT * FROM outbox WHERE processed_at IS NULL;
   ```
3. 等待 5-30 秒（同步延遲）

## 🎯 最佳實踐

### 客戶端使用建議

1. **開始行程時機**
   - 確認已與司機見面
   - 確認已上車
   - 避免提前點擊

2. **結束行程時機**
   - 確認已到達目的地
   - 確認已下車
   - 避免提前點擊

3. **錯誤處理**
   - 如果誤點，立即聯繫客服
   - 不要重複點擊按鈕
   - 注意查看錯誤提示

### 開發建議

1. **狀態管理**
   - 使用 Riverpod 管理訂單狀態
   - 自動刷新訂單資料
   - 避免狀態不一致

2. **錯誤處理**
   - 完整的 try-catch
   - 清晰的錯誤訊息
   - 用戶友好的提示

3. **用戶體驗**
   - 確認對話框避免誤觸
   - 載入指示器提供反饋
   - 成功/錯誤提示明確
