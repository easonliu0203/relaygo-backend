## 🔄 狀態轉換規則

### 允許的狀態轉換

```
pending_payment → paid_deposit (支付訂金)
paid_deposit → matched (手動派單)
paid_deposit → cancelled (取消訂單)

matched → driver_confirmed (司機確認)
matched → cancelled (取消訂單)

driver_confirmed → driver_departed (司機出發)
driver_confirmed → trip_started (直接開始行程，跳過出發和抵達)
driver_confirmed → cancelled (取消訂單)

driver_departed → driver_arrived (司機抵達)

driver_arrived → trip_started (開始行程)

trip_started → trip_ended (結束行程)

trip_ended → completed (支付尾款)
```

**注意**：`matched` 狀態在 Firestore 中映射為 `awaitingDriver`（待司機確認），`trip_ended` 狀態映射為 `awaitingBalance`（待付尾款）。

### 不允許的狀態轉換

- ❌ 已完成的訂單不能回退到其他狀態
- ❌ 已取消的訂單不能恢復
- ❌ 不能跳過必要的中間狀態（例如：不能從 `pending_payment` 直接到 `completed`）

## 🎯 各狀態的業務含義

### 1. pending_payment（待支付）
- **業務含義**：訂單已創建，等待客戶支付訂金
- **客戶端顯示**：「待配對」
- **可執行操作**：支付訂金、取消訂單
- **下一步**：支付訂金 → `paid_deposit`

### 2. paid_deposit（已付訂金）
- **業務含義**：訂金已支付，等待管理員派單
- **客戶端顯示**：「待配對」
- **可執行操作**：等待派單、取消訂單（可退款）
- **下一步**：管理員派單 → `matched`

### 3. matched（已分配司機）
- **業務含義**：系統已分配司機，等待司機確認接單
- **客戶端顯示**：「待司機確認」（Firestore: `awaitingDriver`）
- **可執行操作**：等待司機確認、取消訂單
- **下一步**：司機確認 → `driver_confirmed`

### 4. driver_confirmed（司機已確認）
- **業務含義**：司機已確認接單，準備出發
- **客戶端顯示**：「已配對」
- **可執行操作**：司機出發、查看司機資訊、聊天
- **下一步**：司機出發 → `driver_departed`
- **特殊**：可以直接開始行程（跳過出發和抵達）

### 5. driver_departed（司機已出發）
- **業務含義**：司機已出發前往載客地點
- **客戶端顯示**：「進行中」
- **可執行操作**：等待司機抵達、查看司機位置、聊天
- **下一步**：司機抵達 → `driver_arrived`

### 6. driver_arrived（司機已到達）
- **業務含義**：司機已到達載客地點，等待客戶上車
- **客戶端顯示**：「進行中」
- **可執行操作**：客戶開始行程、聊天
- **下一步**：客戶開始行程 → `trip_started`

### 7. trip_started（行程進行中）
- **業務含義**：客戶已上車，行程正在進行
- **客戶端顯示**：「進行中」
- **可執行操作**：客戶結束行程、聊天
- **下一步**：客戶結束行程 → `trip_ended`
- **記錄**：`actual_start_time` 記錄實際開始時間

### 8. trip_ended（行程已結束）
- **業務含義**：行程已結束，等待客戶支付尾款
- **客戶端顯示**：「待付尾款」（Firestore: `awaitingBalance`）
- **可執行操作**：支付尾款
- **下一步**：支付尾款 → `completed`
- **記錄**：`actual_end_time` 記錄實際結束時間
- **自動導航**：Flutter APP 自動導航到支付尾款頁面

### 9. completed（訂單完成）
- **業務含義**：尾款已支付，訂單完成
- **客戶端顯示**：「已完成」
- **可執行操作**：查看訂單詳情、評價司機
- **下一步**：無（終態）

### 10. cancelled（已取消）
- **業務含義**：訂單已取消
- **客戶端顯示**：「已取消」
- **可執行操作**：查看取消原因
- **下一步**：無（終態）

## 💰 費用資訊顯示邏輯

### 根據訂單狀態顯示不同的費用資訊

#### 狀態 1-8：訂單未完成（`balancePaid = false`）

顯示欄位：
- ✅ 預估總費用：`estimatedFare`
- ✅ 已付訂金：`depositAmount`（綠色 ✓ 或紅色 ✗）
- ✅ 剩餘費用：`balanceAmount`（橙色）

範例：
```
預估總費用：NT$ 65
已付訂金：  NT$ 20 ✓
剩餘費用：  NT$ 45
```

**注意**：當訂單狀態為 `awaitingBalance`（待付尾款）時，會自動導航到支付尾款頁面。

#### 狀態 9：訂單已完成（`balancePaid = true`）

顯示欄位：
- ✅ 預估總費用：`estimatedFare`
- ✅ 已付訂金：`depositAmount`（綠色 ✓）
- ✅ 已付尾款：`balanceAmount`（綠色 ✓）
- ✅ **已付總額**：`totalPaid`（綠色 ✓，粗體）

範例：
```

已付訂金：  NT$ 20 ✓
已付尾款：  NT$ 45 ✓

```

## 🔧 開發者注意事項

### 1. 狀態檢查

在執行狀態轉換前，**必須檢查當前狀態**：

```typescript
// ❌ 錯誤：沒有檢查當前狀態
await supabase
  .from('bookings')
  .update({ status: 'driver_departed' })
  .eq('id', bookingId);

// ✅ 正確：檢查當前狀態
if (booking.status !== 'driver_confirmed') {
  throw new Error('訂單狀態不正確');
}
await supabase
  .from('bookings')
  .update({ status: 'driver_departed' })
  .eq('id', bookingId);
```

### 2. 權限驗證

不同角色只能執行特定的狀態轉換：

- **客戶**：支付訂金、開始行程、結束行程、支付尾款、取消訂單
- **司機**：確認接單、出發、抵達
- **管理員**：派單、取消訂單

### 3. 時間記錄

某些狀態轉換需要記錄時間：

- `trip_started`：記錄 `actual_start_time`
- `trip_ended`：記錄 `actual_end_time`

### 4. 系統訊息

狀態轉換時應發送系統訊息到聊天室：

```typescript
await sendSystemMessage(bookingId, '司機已出發前往載客地點');
```

### 5. Edge Function 部署

修改狀態映射邏輯後，**必須重新部署 Edge Function**：

```bash
npx supabase functions deploy sync-to-firestore
```

## 📝 程式碼範例

### Backend API - 狀態轉換

```typescript
// 司機出發
router.post('/bookings/:bookingId/depart', async (req, res) => {
  const { bookingId } = req.params;
  
  // 1. 查詢訂單
  const { data: booking } = await supabase
    .from('bookings')
    .select('*')
    .eq('id', bookingId)
    .single();
  
  // 2. 檢查狀態
  if (booking.status !== 'driver_confirmed') {
    return res.status(400).json({ error: '訂單狀態不正確' });
  }
  
  // 3. 更新狀態
  await supabase
    .from('bookings')
    .update({ status: 'driver_departed' })
    .eq('id', bookingId);
  
  // 4. 發送系統訊息
  await sendSystemMessage(bookingId, '司機已出發前往載客地點');
  
  res.json({ success: true });
});
```

### Flutter - 狀態枚舉

```dart
enum BookingStatus {
  @JsonValue('pending')
  pending,        // 待配對（待付訂金或待派單）

  @JsonValue('awaitingDriver')
  awaitingDriver, // 待司機確認

  @JsonValue('matched')
  matched,        // 已配對

  @JsonValue('inProgress')
  inProgress,     // 進行中

  @JsonValue('awaitingBalance')
  awaitingBalance, // 待付尾款

  @JsonValue('completed')
  completed,      // 已完成

  @JsonValue('cancelled')
  cancelled,      // 已取消
}
```
