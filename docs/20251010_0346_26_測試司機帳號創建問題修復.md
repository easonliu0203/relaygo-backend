## 📋 問題描述

### 問題現象

在測試司機配對功能時，發現以下問題：

1. **手動派單功能無法使用**
   - 點擊「手動派單」按鈕
   - 彈出「選擇司機對話框」
   - 對話框中顯示「沒有可用司機」

2. **測試司機帳號不存在**
   - 系統中應該存在測試司機帳號：`driver.test@gelaygo.com`
   - 但實際查詢資料庫發現該帳號不存在

### 影響範圍

- ❌ 無法測試手動派單功能
- ❌ 無法測試自動派單功能
- ❌ 無法驗證司機配對機制是否正常運作

#### 3. 檢查 drivers 表結構

**文件**: `supabase/migrations/20250104_create_user_profiles_and_drivers.sql`

**表結構**:
```sql
CREATE TABLE IF NOT EXISTS drivers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    license_number VARCHAR(50) UNIQUE NOT NULL,
    license_expiry DATE NOT NULL,  -- ⚠️ 必填欄位
    vehicle_type VARCHAR(10) NOT NULL CHECK (vehicle_type IN ('A', 'B', 'C', 'D')),
    vehicle_model VARCHAR(100),
    vehicle_year INTEGER,
    vehicle_plate VARCHAR(20) UNIQUE NOT NULL,
    insurance_number VARCHAR(50),
    insurance_expiry DATE,
    background_check_status VARCHAR(20) DEFAULT 'pending',
    background_check_date DATE,
    rating DECIMAL(3,2) DEFAULT 0.00,
    total_trips INTEGER DEFAULT 0,
    is_available BOOLEAN DEFAULT false,
    languages TEXT[],
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(user_id)
);
```

**發現**: ⚠️ `license_expiry` 是必填欄位（NOT NULL）

### 根本原因

**問題 1**: 測試司機帳號不存在
- 資料庫中沒有 `driver.test@gelaygo.com` 用戶
- 沒有對應的 `user_profiles` 記錄
- 沒有對應的 `drivers` 記錄

**問題 2**: drivers 表有必填欄位
- `license_expiry` 必須提供
- 創建 driver 記錄時需要提供完整資料

## 💡 開發心得和經驗總結

### 1. 資料完整性的重要性

**學習要點**:
- 創建測試資料時，必須考慮所有必填欄位
- NOT NULL 約束會導致插入失敗
- 需要了解完整的表結構

**最佳實踐**:
```javascript
// ❌ 錯誤：缺少必填欄位
const { data, error } = await supabase
  .from('drivers')
  .insert({
    user_id: userId,
    vehicle_type: 'A',
    vehicle_plate: 'TEST-001',
    is_available: true
  });
// Error: null value in column "license_expiry" violates not-null constraint

// ✅ 正確：提供所有必填欄位
const licenseExpiry = new Date();
licenseExpiry.setFullYear(licenseExpiry.getFullYear() + 1);

const { data, error } = await supabase
  .from('drivers')
  .insert({
    user_id: userId,
    license_number: 'TEST-LICENSE-001',
    license_expiry: licenseExpiry.toISOString().split('T')[0], // 必填
    vehicle_type: 'A',
    vehicle_plate: 'TEST-001',
    is_available: true
  });
```

### 2. 自動化測試資料創建

**優點**:

### 3. 錯誤訊息的重要性

**學習要點**:
- 詳細的錯誤訊息有助於快速定位問題

```javascript
if (createError) {
```

### 4. 資料庫關聯的處理

**學習要點**:
- 一對一關聯需要使用 UNIQUE 約束
- 外鍵約束確保資料一致性
- 使用反向關聯語法查詢關聯資料

**Supabase 查詢語法**:
```javascript

    id,
    email,

    drivers!user_id (vehicle_type, is_available)
  `)
  .eq('role', 'driver');
```

## 🐛 遇到的困難和解決方法

### 困難 1: 用戶不存在

**問題**: 測試司機帳號不存在於資料庫中

**原因**: 沒有初始化測試資料

**解決**:
1. 創建自動化腳本
2. 檢查並創建完整的用戶記錄
3. 包含所有關聯表的資料

### 困難 2: 必填欄位錯誤

**問題**: 
```
null value in column "license_expiry" violates not-null constraint
```

**原因**: 創建 driver 記錄時缺少必填欄位

**解決**:
1. 檢查表結構，找出所有 NOT NULL 欄位
2. 為測試資料提供合理的預設值
3. 使用動態日期（如一年後）作為到期日

**代碼**:
```javascript
const licenseExpiry = new Date();
licenseExpiry.setFullYear(licenseExpiry.getFullYear() + 1);

const driverData = {
  license_expiry: licenseExpiry.toISOString().split('T')[0],
  insurance_expiry: licenseExpiry.toISOString().split('T')[0],
  background_check_date: new Date().toISOString().split('T')[0]
};
```

### 困難 3: 模組載入錯誤

**問題**:
```
Error: Cannot find module '@supabase/supabase-js'
```

**原因**: 腳本在錯誤的目錄執行

**解決**:
1. 使用相對路徑載入模組
2. 在正確的目錄執行腳本
