## 🔧 技術實作

### 1. Riverpod Providers 整合

**使用的 Providers**:
```dart
// 聊天室列表
final chatRoomsAsync = ref.watch(chatRoomsStreamProvider);

// 聊天訊息列表
final messagesAsync = ref.watch(
  chatMessagesStreamProvider(widget.chatRoom.bookingId),
);

// 發送訊息
final sendMessage = ref.read(sendMessageProvider);

// 標記已讀
final markAsRead = ref.read(markMessagesAsReadProvider);

// 當前用戶
final currentUser = ref.watch(firebaseServiceProvider).currentUser;
```

### 2. Firestore Real-time Listeners

**聊天室列表**:
```dart
chatRoomsAsync.when(
  data: (chatRooms) => /* 顯示列表 */,
  loading: () => /* 載入中 */,
  error: (error, stack) => /* 錯誤處理 */,
)
```

**聊天訊息列表**:
```dart
messagesAsync.when(
  data: (messages) => /* 顯示訊息 */,
  loading: () => /* 載入中 */,
  error: (error, stack) => /* 錯誤處理 */,
)
```

### 3. 自動滾動到底部

```dart
// 新訊息時自動滾動
WidgetsBinding.instance.addPostFrameCallback((_) {
  _scrollToBottom();
});

void _scrollToBottom() {
  if (_scrollController.hasClients) {
    _scrollController.animateTo(
      _scrollController.position.maxScrollExtent,
      duration: const Duration(milliseconds: 300),
      curve: Curves.easeOut,
    );
  }
}
```

### 4. 日期分隔線邏輯

```dart
// 檢查是否需要顯示日期分隔線
bool showDateDivider = false;
if (index == 0) {
  showDateDivider = true;
} else {
  final prevMessage = messages[index - 1];
  final currentDate = DateTime(
    message.createdAt.year,
    message.createdAt.month,
    message.createdAt.day,
  );
  final prevDate = DateTime(
    prevMessage.createdAt.year,
    prevMessage.createdAt.month,
    prevMessage.createdAt.day,
  );
  showDateDivider = !currentDate.isAtSameMomentAs(prevDate);
}
```

### 5. 發送訊息邏輯

```dart
Future<void> _sendMessage() async {
  final messageText = _messageController.text.trim();
  if (messageText.isEmpty || _isSending) return;

  setState(() {
    _isSending = true;
  });

  try {
    final currentUser = ref.read(firebaseServiceProvider).currentUser;
    final sendMessage = ref.read(sendMessageProvider);
    final receiverId = widget.chatRoom.getOtherUserId(currentUser.uid);

    await sendMessage(
      bookingId: widget.chatRoom.bookingId,
      receiverId: receiverId,
      messageText: messageText,
    );

    _messageController.clear();
    _scrollToBottom();
  } catch (e) {
    // 錯誤處理
  } finally {
    setState(() {
      _isSending = false;
    });
  }
}
```

## 💡 開發心得和經驗總結

### 成功的地方

1. **組件化設計**:
   - MessageBubble 組件可重用
   - 客戶端和司機端共用相同的組件
   - 代碼結構清晰，易於維護

2. **Riverpod 整合**:
   - 使用 StreamProvider 實現即時更新
   - 狀態管理簡潔明瞭
   - 自動處理載入和錯誤狀態

3. **用戶體驗**:
   - 自動滾動到最新訊息
   - 進入頁面時自動標記已讀
   - 發送訊息時顯示載入狀態
   - 錯誤處理和重試機制

4. **翻譯功能預留**:
   - UI 設計預留翻譯文字顯示位置
   - 資料模型已包含 translatedText 欄位
   - 之後串接 ChatGPT 4o mini 只需修改 API 邏輯

### 遇到的困難

1. **自動滾動時機**:
   - **問題**: 初始載入時無法正確滾動到底部
   - **原因**: ListView 尚未完成渲染
   - **解決**: 使用 `WidgetsBinding.instance.addPostFrameCallback` 延遲執行

2. **日期分隔線邏輯**:
   - **問題**: 需要比較相鄰訊息的日期
   - **解決**: 使用 DateTime 比較，只比較年月日部分

3. **未讀訊息計數**:
   - **問題**: 需要根據當前用戶角色顯示正確的未讀數量
   - **解決**: 使用 ChatRoom 模型的 `getUnreadCount()` 方法

### 待優化的部分

1. **頭像顯示**:
   - 目前使用預設圖標
   - 之後需要整合真實的用戶頭像

2. **訊息分頁載入**:
   - 目前一次載入所有訊息
   - 之後需要實作下拉載入更多（分頁）

3. **輸入中狀態**:
   - 目前只有 UI 組件
   - 之後需要實作即時同步輸入中狀態

4. **訊息撤回**:
   - 目前不支持撤回訊息
   - 之後可以考慮添加此功能
