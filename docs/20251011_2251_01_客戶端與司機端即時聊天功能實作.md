## 📋 功能需求概述

實作一個即時聊天系統，讓已配對的客戶和司機能夠在訂單開始前 24 小時內進行溝通。

### 核心功能

1. **聊天室列表**: 顯示所有已配對訂單的聊天室
2. **即時訊息**: 客戶和司機可以互相發送訊息
3. **未讀訊息**: 顯示未讀訊息數量和已讀狀態
4. **24 小時限制**: 只能在訂單開始前 24 小時內發送訊息
5. **翻譯功能預留**: 資料庫和 UI 預留翻譯文字欄位

## 🔧 已完成的工作

### 階段 1: 資料庫和 API 設計 ✅

#### 1.1 資料庫 Migration

**文件**: `supabase/migrations/20251011_create_chat_messages_table.sql`

**創建的表**:
```sql
CREATE TABLE chat_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  booking_id UUID NOT NULL,
  sender_id UUID NOT NULL,
  receiver_id UUID NOT NULL,
  message_text TEXT NOT NULL,
  translated_text TEXT,  -- 預留翻譯欄位
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  read_at TIMESTAMP WITH TIME ZONE,
  
  CONSTRAINT fk_booking FOREIGN KEY (booking_id) REFERENCES bookings(id) ON DELETE CASCADE,
  CONSTRAINT fk_sender FOREIGN KEY (sender_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_receiver FOREIGN KEY (receiver_id) REFERENCES users(id) ON DELETE CASCADE
);
```

**創建的索引**:
- `idx_chat_messages_booking_id`: 按訂單 ID 查詢
- `idx_chat_messages_created_at`: 按時間排序
- `idx_chat_messages_sender_id`: 按發送者查詢
- `idx_chat_messages_receiver_read`: 查詢未讀訊息
- `idx_chat_messages_booking_created`: 訂單 + 時間複合索引（分頁）

**RLS 策略**:
1. 用戶可以查看自己發送或接收的訊息
2. 用戶可以插入訊息（發送者必須是自己）
3. 用戶可以更新自己接收的訊息（標記為已讀）

**Trigger 函數**:
- `chat_messages_to_outbox()`: 將聊天訊息同步到 outbox 表
- 包含發送者、接收者、訂單資訊
- 自動同步到 Firestore

#### 1.2 API 端點

**1. 發送訊息 API** ✅

**文件**: `web-admin/src/app/api/chat/send-message/route.ts`

**端點**: `POST /api/chat/send-message`

**功能**:
- 驗證用戶身份（Firebase Auth）
- 驗證用戶是否為訂單的客戶或司機
- 驗證訂單是否已配對
- 驗證是否在開始時間前 24 小時內
- 寫入 Supabase chat_messages 表
- 通過 trigger 自動同步到 Firestore

**請求體**:
```json
{
  "bookingId": "uuid",
  "senderId": "firebase_uid",
  "receiverId": "firebase_uid",
  "messageText": "訊息內容"
}
```

**2. 獲取聊天室列表 API** ✅

**文件**: `web-admin/src/app/api/chat/rooms/route.ts`

**端點**: `GET /api/chat/rooms?userId={firebaseUid}`

**功能**:
- 獲取用戶的所有已配對訂單
- 為每個訂單創建聊天室資訊
- 包含最後一則訊息和未讀訊息數量
- 按最後訊息時間排序

**響應**:
```json
{
  "success": true,
  "data": [
    {
      "bookingId": "uuid",
      "otherUserId": "firebase_uid",
      "otherUserName": "姓名",
      "otherUserAvatar": "url",
      "pickupAddress": "地址",
      "bookingTime": "2025-10-12T14:00:00",
      "lastMessage": "最後訊息",
      "lastMessageTime": "2025-10-11T22:00:00",
      "unreadCount": 2,
      "status": "matched"
    }
  ],
  "total": 1
}
```

**3. 獲取聊天訊息 API** ✅

**文件**: `web-admin/src/app/api/chat/messages/[bookingId]/route.ts`

**端點**: `GET /api/chat/messages/:bookingId?userId={firebaseUid}&limit=50&before={timestamp}`

**功能**:
- 獲取指定訂單的聊天訊息
- 支持分頁載入（limit 和 before 參數）
- 驗證用戶權限
- 包含發送者資訊

**4. 標記訊息為已讀 API** ✅

**文件**: `web-admin/src/app/api/chat/mark-read/[bookingId]/route.ts`

**端點**: `PUT /api/chat/mark-read/:bookingId?userId={firebaseUid}`

**功能**:
- 標記指定訂單中用戶接收的所有未讀訊息為已讀
- 驗證用戶權限

#### 1.3 Firestore 同步服務

**文件**: `supabase/functions/sync-to-firestore/index.ts`

**新增函數**:
1. `syncChatMessageToFirestore()`: 同步聊天訊息到 Firestore
2. `upsertChatMessageToFirestore()`: 創建或更新訊息文檔
3. `updateChatRoomInfo()`: 更新聊天室資訊
4. `deleteChatMessageFromFirestore()`: 刪除訊息文檔

**Firestore 集合結構**:
```
chat_rooms/{bookingId}/
  - bookingId: string
  - customerId: string
  - driverId: string
  - customerName: string
  - driverName: string
  - pickupAddress: string
  - bookingTime: timestamp
  - lastMessage: string
  - lastMessageTime: timestamp
  - updatedAt: timestamp

chat_rooms/{bookingId}/messages/{messageId}
  - id: string
  - senderId: string
  - receiverId: string
  - senderName: string
  - receiverName: string
  - messageText: string
  - translatedText: string (預留)
  - createdAt: timestamp
  - readAt: timestamp (nullable)
```

## 🔧 修復歷程

### 修復 1: Firestore 權限錯誤（2025-10-12 08:06）

**問題**: 客戶端和司機端聊天室列表載入失敗，出現 `[cloud_firestore/permission-denied]` 錯誤

**原因**: Firestore Security Rules 缺少 `chat_rooms` collection 的規則

**修復**:
1. 添加 `chat_rooms` collection 的讀取規則
2. 添加 `chat_rooms/{roomId}/messages` 子集合的讀取規則
3. 修改 ChatService 查詢邏輯（使用兩個獨立查詢）
4. 添加 rxdart 依賴
5. 部署 Firestore Security Rules

**文檔**: `docs/20251012_0806_05_Firestore聊天權限錯誤修復.md`

### 修復 2: Firestore 索引錯誤（2025-10-12 09:58）

**問題**: 客戶端和司機端聊天室列表載入失敗，出現 `[cloud_firestore/failed-precondition]` 錯誤
