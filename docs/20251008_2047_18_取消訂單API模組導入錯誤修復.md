# 取消訂單 API 模組導入錯誤修復

**日期**：2025-10-08 20:47  
**問題**：取消訂單 API 導入不存在的模組導致編譯失敗  
**狀態**：✅ 已修復

---

## 📋 問題描述

### 錯誤訊息
```
Server Error
Error: Module not found: Can't resolve '@/lib/database'

./src/app/api/bookings/[id]/cancel/route.ts:2:0
Module not found: Can't resolve '@/lib/database'
  1 | import { NextRequest, NextResponse } from 'next/server';
> 2 | import { DatabaseService } from '@/lib/database';
  3 |
```

### 發生時機
1. 用戶點擊「取消訂單」按鈕
2. 客戶端調用取消訂單 API
3. Next.js 嘗試編譯 API 路由
4. ❌ 模組導入失敗
5. ❌ 返回 HTML 錯誤頁面而不是 JSON

### 連鎖影響
- 客戶端收到 HTML 響應而不是 JSON
- 觸發之前修復的錯誤處理邏輯
- 顯示錯誤訊息：「API 返回非 JSON 響應 (500)，請檢查管理後台是否正常運行」
- 用戶無法取消訂單

---

## 🔍 問題診斷過程

### 步驟 1：檢查客戶端日誌

**根據之前的 API 錯誤處理修復**，客戶端現在會記錄詳細的響應資訊：

```
[BookingService] 開始取消訂單: xxx
[BookingService] 請求 URL: http://10.0.2.2:3001/api/bookings/xxx/cancel
[BookingService] 響應狀態碼: 500
[BookingService] 響應 Content-Type: text/html
[BookingService] 響應內容: <!DOCTYPE html><html>...
[BookingService] 取消訂單失敗: Exception: API 返回非 JSON 響應 (500)，請檢查管理後台是否正常運行
```

**關鍵發現**：
- 狀態碼 500 → API 內部錯誤
- Content-Type: text/html → 返回 HTML 而不是 JSON
- 這說明 API 路由有問題

---

### 步驟 2：檢查管理後台錯誤頁面

**用戶提供的錯誤訊息**：
```
Server Error
Error: Module not found: Can't resolve '@/lib/database'
```

**分析**：
- Next.js 無法找到 `@/lib/database` 模組
- 這導致 API 路由無法編譯
- Next.js 返回錯誤頁面（HTML）而不是 JSON

---

### 步驟 3：檢查取消訂單 API 文件

<augment_code_snippet path="web-admin/src/app/api/bookings/[id]/cancel/route.ts" mode="EXCERPT">
````typescript
import { NextRequest, NextResponse } from 'next/server';
import { DatabaseService } from '@/lib/database';  // ❌ 錯誤的導入
````
</augment_code_snippet>

**問題**：
- 第 2 行導入 `@/lib/database`
- 但這個模組不存在

---

### 步驟 4：檢查其他 API 路由文件

**查看創建訂單 API**：

<augment_code_snippet path="web-admin/src/app/api/bookings/route.ts" mode="EXCERPT">
````typescript
import { NextRequest, NextResponse } from 'next/server';
import { DatabaseService } from '@/lib/supabase';  // ✅ 正確的導入
````
</augment_code_snippet>

**查看支付訂金 API**：

<augment_code_snippet path="web-admin/src/app/api/bookings/[id]/pay-deposit/route.ts" mode="EXCERPT">
````typescript
import { NextRequest, NextResponse } from 'next/server';
import { DatabaseService } from '@/lib/supabase';  // ✅ 正確的導入
````
</augment_code_snippet>

**關鍵發現**：
- 其他 API 路由都使用 `@/lib/supabase`
- 只有取消訂單 API 使用了錯誤的 `@/lib/database`

---

### 步驟 5：確認根本原因

**根本原因**：
- 取消訂單 API 在創建時使用了錯誤的模組路徑
- 應該是 `@/lib/supabase` 而不是 `@/lib/database`
- 這是一個簡單的拼寫錯誤或複製貼上錯誤

**為什麼之前沒有發現**：
- 可能在開發時沒有測試取消訂單功能
- 或者測試時管理後台沒有重新編譯
- 直到用戶實際使用時才觸發編譯錯誤

---

## 🔧 修復方案

### 修復代碼

**修復前**（錯誤）：

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { DatabaseService } from '@/lib/database';  // ❌ 錯誤
```

**修復後**（正確）：

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { DatabaseService } from '@/lib/supabase';  // ✅ 正確
```

---

### 修復步驟

1. **修改導入語句**
   - 將 `@/lib/database` 改為 `@/lib/supabase`
   - 保持其他代碼不變

2. **重啟管理後台**
   ```bash
   # 停止管理後台（Ctrl+C）
   # 重新啟動
   cd web-admin
   npm run dev
   ```

3. **確認沒有編譯錯誤**
   - 查看管理後台終端
   - 確認沒有錯誤訊息
   - 確認顯示「Ready in ...ms」

4. **測試 API 端點**
   ```bash
   curl -X POST http://localhost:3001/api/bookings/test-id/cancel \
     -H "Content-Type: application/json" \
     -d '{
       "customerUid": "test-uid",
       "reason": "測試取消功能"
     }'
   ```

## 💡 開發心得

### 1. 模組導入的重要性

**教訓**：
- 模組路徑錯誤會導致編譯失敗
- Next.js 會返回 HTML 錯誤頁面而不是 JSON
- 這會導致客戶端無法正確處理響應

**最佳實踐**：
- 使用 IDE 的自動導入功能
- 檢查其他類似文件的導入方式
- 在開發時測試所有 API 端點

---

### 2. 錯誤處理的價值

**學習**：
- 之前的 API 錯誤處理修復非常有用
- 詳細的日誌幫助快速定位問題
- 清晰的錯誤訊息提示檢查管理後台

**如果沒有錯誤處理修復**：
- 會拋出 `FormatException`
- 錯誤訊息不清楚
- 難以診斷問題

**有了錯誤處理修復**：
- 顯示清晰的錯誤訊息
- 日誌記錄完整資訊
- 快速定位到管理後台問題

---

### 3. 測試的重要性

**經驗**：
- 應該在開發時測試所有功能
- 不要等到用戶使用時才發現問題
- 使用自動化測試可以避免這類錯誤
