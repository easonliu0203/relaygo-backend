## 📋 功能概述

在公司端（web-admin）訂單管理系統中，增強訂單管理功能，允許管理員對已配對司機的訂單進行調整和取消操作。

### 核心功能

1. **更改已配對的司機** - 允許管理員重新分配司機
2. **取消訂單功能** - 實作完整的取消訂單流程

---

## 🏗️ 系統架構

### 後端 API 端點

#### 1. PUT /api/admin/bookings/[id]/change-driver
**功能**: 更改訂單的司機

**請求體**:
```typescript
{
  newDriverId: string,
  reason?: string  // 更改原因（可選）
}
```

**返回資料**:
```typescript
{
  success: true,
  data: {
    id: string,
    driver_id: string,
    previous_driver_id: string,
    updated_at: string
  },
  message: '成功更改司機'
}
```

**處理邏輯**:
1. 獲取訂單資訊
2. 檢查訂單狀態是否允許更改司機
3. 檢查是否已有司機
4. 檢查新司機是否與舊司機相同
5. 獲取新司機資訊並驗證
6. 檢查新司機是否可用
7. 檢查車型是否匹配（允許不匹配，但記錄警告）
8. 檢查新司機的時間衝突
9. 更新訂單的 driver_id

**允許更改的訂單狀態**:
- `assigned` - 已派單
- `driver_confirmed` - 司機已確認
- `driver_departed` - 司機已出發
- `driver_arrived` - 司機已到達

#### 2. POST /api/admin/bookings/[id]/cancel
**功能**: 取消訂單

**請求體**:
```typescript
{
  reason: string,        // 取消原因（必填）
  cancelledBy?: string   // 取消者 ID（管理員 ID）
}
```

**返回資料**:
```typescript
{
  success: true,
  data: {
    id: string,
    status: 'cancelled',
    cancelled_at: string,
    cancellation_reason: string
  },
  message: '訂單已取消',
  hadDriver: boolean  // 是否已分配司機
}
```

**處理邏輯**:
1. 獲取訂單資訊
2. 檢查訂單是否已取消
3. 檢查訂單是否已完成
4. 檢查訂單是否正在進行中（trip_started 狀態不允許取消）
5. 更新訂單狀態為 'cancelled'
6. 記錄取消時間、取消原因、取消者

**更新的欄位**:
- `status` → 'cancelled'
- `cancelled_at` → 當前時間
- `cancellation_reason` → 取消原因
- `cancelled_by` → 管理員 ID
- `updated_at` → 當前時間

**不允許取消的訂單狀態**:
- `cancelled` - 已取消
- `completed` - 已完成
- `trip_started` - 行程已開始

#### 更改司機 API

**文件**: `web-admin/src/app/api/admin/bookings/[id]/change-driver/route.ts`

**關鍵代碼**:
```typescript
export async function PUT(request: NextRequest, { params }: { params: { id: string } }) {
  const bookingId = params.id;
  const { newDriverId, reason } = await request.json();

  // 1. 獲取訂單資訊
  const { data: booking } = await db.supabase
    .from('bookings')
    .select('*')
    .eq('id', bookingId)
    .single();

  // 2. 檢查訂單狀態
  const allowedStatuses = ['assigned', 'driver_confirmed', 'driver_departed', 'driver_arrived'];
  if (!allowedStatuses.includes(booking.status)) {
    return NextResponse.json({ 
      success: false,
      error: `訂單狀態 ${booking.status} 不允許更改司機` 
    }, { status: 400 });
  }

  // 3. 檢查新司機
  const { data: newDriver } = await db.supabase
    .from('users')
    .select(`
      id,
      role,
      drivers!user_id (id, vehicle_type, is_available)
    `)
    .eq('id', newDriverId)
    .eq('role', 'driver')
    .single();

  const newDriverInfo = newDriver.drivers?.[0] || newDriver.drivers;

  // 4. 檢查可用性
  if (!newDriverInfo.is_available) {
    return NextResponse.json({ 
      success: false,
      error: '新司機目前不可用' 
    }, { status: 400 });
  }

  // 5. 檢查時間衝突
  const [hours, minutes] = booking.start_time.split(':').map(Number);
  const startMinutes = hours * 60 + minutes;
  const endMinutes = startMinutes + (booking.duration_hours * 60);

  const { data: existingBookings } = await db.supabase
    .from('bookings')
    .select('id, start_time, duration_hours')
    .eq('driver_id', newDriverId)
    .eq('start_date', booking.start_date)
    .in('status', ['assigned', 'driver_confirmed', 'driver_departed', 'driver_arrived', 'trip_started'])
    .neq('id', bookingId);

  const hasConflict = existingBookings.some(existingBooking => {
    const [existingHours, existingMinutes] = existingBooking.start_time.split(':').map(Number);
    const existingStartMinutes = existingHours * 60 + existingMinutes;
    const existingEndMinutes = existingStartMinutes + (existingBooking.duration_hours * 60);
    return (startMinutes < existingEndMinutes) && (endMinutes > existingStartMinutes);
  });

  if (hasConflict) {
    return NextResponse.json({ 
      success: false,
      error: '新司機在該時間段已有其他訂單' 
    }, { status: 400 });
  }

  // 6. 更新訂單
  const previousDriverId = booking.driver_id;
  const { data: updatedBooking } = await db.supabase
    .from('bookings')
    .update({
      driver_id: newDriverId,
      updated_at: new Date().toISOString(),
    })
    .eq('id', bookingId)
    .select()
    .single();

  return NextResponse.json({
    success: true,
    data: { ...updatedBooking, previous_driver_id: previousDriverId },
    message: '成功更改司機',
  });
}
```

**文件**: `web-admin/src/app/api/admin/bookings/[id]/cancel/route.ts`

**關鍵代碼**:
```typescript
export async function POST(request: NextRequest, { params }: { params: { id: string } }) {
  const bookingId = params.id;
  const { reason, cancelledBy } = await request.json();

  // 1. 驗證取消原因
  if (!reason || reason.trim() === '') {
    return NextResponse.json({ 
      success: false,
      error: '請提供取消原因' 
    }, { status: 400 });
  }

  // 2. 獲取訂單資訊
  const { data: booking } = await db.supabase
    .from('bookings')
    .select('*')
    .eq('id', bookingId)
    .single();

  // 3. 檢查訂單狀態
  if (booking.status === 'cancelled') {
    return NextResponse.json({ 
      success: false,
      error: '訂單已經被取消' 
    }, { status: 400 });
  }

  if (booking.status === 'completed') {
    return NextResponse.json({ 
      success: false,
      error: '已完成的訂單無法取消' 
    }, { status: 400 });
  }

  if (booking.status === 'trip_started') {
    return NextResponse.json({ 
      success: false,
      error: '行程已開始，無法取消訂單' 
    }, { status: 400 });
  }

  // 4. 更新訂單狀態
  const updateData = {
    status: 'cancelled',
    cancelled_at: new Date().toISOString(),
    cancellation_reason: reason.trim(),
    updated_at: new Date().toISOString(),
  };

  if (cancelledBy) {
    updateData.cancelled_by = cancelledBy;
  }

  const { data: updatedBooking } = await db.supabase
    .from('bookings')
    .update(updateData)
    .eq('id', bookingId)
    .select()
    .single();

  return NextResponse.json({
    success: true,
    data: updatedBooking,
    message: '訂單已取消',
    hadDriver: !!booking.driver_id
  });
}
```

```typescript
// 更改司機 - 打開司機選擇對話框
const handleChangeDriver = async (booking: any) => {
  setSelectedBooking(booking);
  setIsChangingDriver(true);
  setDriverModalVisible(true);
  setLoadingDrivers(true);

  try {
    const response = await fetch(
      `/api/admin/drivers/available?vehicleType=${booking.vehicleType}&date=${booking.scheduledDate}&time=${booking.scheduledTime}&duration=${booking.durationHours}`
    );
    const data = await response.json();

    if (data.success) {
      setAvailableDrivers(data.data || []);
    } else {
      message.error(data.error || '獲取司機列表失敗');
    }
  } catch (error) {
    message.error('獲取司機列表失敗');
  } finally {
    setLoadingDrivers(false);
  }
};

// 確認更改司機
const handleConfirmChangeDriver = async (newDriverId: string) => {
  if (!selectedBooking) return;

  // 檢查訂單狀態，如果司機已出發或已到達，顯示確認對話框
  const warningStatuses = ['driver_departed', 'driver_arrived'];
  if (warningStatuses.includes(selectedBooking.status)) {
    Modal.confirm({
      title: '確認更改司機',
      content: `此訂單司機已${selectedBooking.status === 'driver_departed' ? '出發' : '到達'}，確定要更改司機嗎？`,
      okText: '確認更改',
      cancelText: '取消',
      onOk: async () => {
        await executeChangeDriver(newDriverId);
      },
    });
  } else {
    await executeChangeDriver(newDriverId);
  }
};

// 執行更改司機
const executeChangeDriver = async (newDriverId: string) => {
  setAssigningDriver(true);
  try {
    const response = await fetch(`/api/admin/bookings/${selectedBooking.id}/change-driver`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ newDriverId }),
    });

    const data = await response.json();

    if (data.success) {
      message.success('成功更改司機');
      setDriverModalVisible(false);
      setSelectedBooking(null);
      setIsChangingDriver(false);
      loadOrders();
    } else {
      message.error(data.error || '更改司機失敗');
    }
  } catch (error) {
    message.error('更改司機失敗');
  } finally {
    setAssigningDriver(false);
  }
};
```

### 1. 狀態管理的重要性

**學習要點**:
- 使用 `isChangingDriver` 狀態區分手動派單和更改司機模式
- 重用司機選擇對話框，根據狀態調用不同的處理函數

**最佳實踐**:
```typescript
// 根據模式調用不同的處理函數
onClick={() => {
```

### 2. 確認對話框的使用

**學習要點**:
- 對於危險操作（如更改已出發的司機），使用確認對話框
- 使用 Modal.confirm 提供二次確認

**實作**:

```typescript
const warningStatuses = ['driver_departed', 'driver_arrived'];

  Modal.confirm({
    title: '確認更改司機',

    okText: '確認更改',
```

## 🐛 遇到的困難和解決方法

### 困難 1: 重用司機選擇對話框

**問題**: 手動派單和更改司機都需要司機選擇對話框

**解決**:
1. 使用 `isChangingDriver` 狀態區分模式
2. 根據狀態調整對話框標題和按鈕行為
3. 創建不同的處理函數

### 困難 2: 確認對話框的時機

**問題**: 何時顯示確認對話框？

**解決**:
1. 定義需要確認的訂單狀態列表
2. 在執行操作前檢查狀態
3. 使用 Modal.confirm 提供二次確認

### 困難 3: 取消原因的驗證

**問題**: 如何確保用戶輸入取消原因？

**解決**:
1. 前端驗證：檢查輸入是否為空
2. 後端驗證：再次檢查並返回錯誤
3. 使用必填標記（紅色星號）提示用戶
