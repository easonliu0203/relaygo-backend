## 🔍 問題診斷過程

### 1. 分析錯誤訊息

**錯誤特徵**:
- "Network Error" - 表示網路請求失敗
- 不是 HTTP 錯誤碼（如 404, 500）
- 是連接層面的錯誤

**可能原因**:
1. 服務未運行
2. 請求 URL 錯誤
3. CORS 問題
4. 端口錯誤

### 2. 檢查 API 服務層代碼

**文件**: `web-admin/src/services/api.ts`

**發現問題 1** (第 7 行):
```typescript
const API_BASE_URL = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3000';
```

**發現問題 2** (第 10-16 行):
```typescript
const apiClient: AxiosInstance = axios.create({
  baseURL: API_BASE_URL,  // ❌ 設置為 http://localhost:3000
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
});
```

**發現問題 3** (第 240 行):
```typescript
static async getBookings(params?: any) {
  return this.get('/api/admin/bookings', { params });
}
```

### 3. 分析請求流程

**實際請求流程**:
1. 前端調用 `ApiService.getBookings()`
2. 使用 `this.get('/api/admin/bookings', { params })`
3. axios 使用 `baseURL = 'http://localhost:3000'`
4. 最終請求 URL: `http://localhost:3000/api/admin/bookings`

**問題**:
- `/api/admin/bookings` 是 Next.js API 路由，在 `web-admin` 內部
- `web-admin` 運行在 `http://localhost:3001`
- 但請求發送到了 `http://localhost:3000`（後端服務器）
- `http://localhost:3000/api/admin/bookings` 不存在
- 導致 Network Error

### 4. 根本原因總結

**架構混淆**:
- **Next.js API 路由**: 在 web-admin 內部（port 3001）
- **後端 API**: 在 backend 服務器（port 3000）

**錯誤配置**:
- 所有 API 請求都使用同一個 `apiClient`
- `apiClient` 的 `baseURL` 設置為後端服務器
- 導致 Next.js API 路由請求發送到錯誤的服務器

## 🔧 修復方案

### 方案選擇

**方案 A**: 修改 baseURL 為空字符串
- 缺點: 所有 API 都會使用相對路徑，包括後端 API

**方案 B**: 創建兩個 axios 實例（✅ 採用）
- `apiClient`: 用於外部後端 API（baseURL = 'http://localhost:3000'）
- `internalApiClient`: 用於 Next.js API 路由（baseURL = ''）
- 優點: 清晰區分內部和外部 API
- 優點: 靈活性高，易於維護

**決定**: 採用方案 B

### 修復步驟

#### 步驟 1: 創建內部 API 客戶端

**文件**: `web-admin/src/services/api.ts`

**修改內容**:
```typescript
// 創建外部 API axios 實例（用於後端服務器）
const apiClient: AxiosInstance = axios.create({
  baseURL: API_BASE_URL,  // 'http://localhost:3000'
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// 創建內部 API axios 實例（用於 Next.js API 路由）
const internalApiClient: AxiosInstance = axios.create({
  baseURL: '',  // ✅ 使用相對路徑
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
});
```

#### 步驟 2: 添加請求和回應攔截器

**為兩個客戶端分別添加攔截器**:

```typescript
// 請求攔截器（外部 API）
apiClient.interceptors.request.use(/* ... */);

// 請求攔截器（內部 API）
internalApiClient.interceptors.request.use(/* ... */);

// 回應攔截器（外部 API）
apiClient.interceptors.response.use(/* ... */);

// 回應攔截器（內部 API）
internalApiClient.interceptors.response.use(/* ... */);
```

**關鍵差異**:
- 日誌訊息區分 "External API" 和 "Internal API"
- 便於調試和追蹤問題

#### 步驟 3: 添加內部 API 請求方法

**在 ApiService 類中添加**:

```typescript
// 內部 API 請求方法（用於 Next.js API 路由）
static async internalGet<T = any>(url: string, config?: AxiosRequestConfig): Promise<T> {
  const response = await internalApiClient.get(url, config);
  return response.data;
}

static async internalPost<T = any>(url: string, data?: any, config?: AxiosRequestConfig): Promise<T> {
  const response = await internalApiClient.post(url, data, config);
  return response.data;
}

static async internalPut<T = any>(url: string, data?: any, config?: AxiosRequestConfig): Promise<T> {
  const response = await internalApiClient.put(url, data, config);
  return response.data;
}

static async internalDelete<T = any>(url: string, config?: AxiosRequestConfig): Promise<T> {
  const response = await internalApiClient.delete(url, config);
  return response.data;
}
```

#### 步驟 4: 修改訂單管理方法

**修改前**:
```typescript
// 訂單管理
static async getBookings(params?: any) {
  return this.get('/api/admin/bookings', { params });  // ❌ 使用外部 API
}

static async getBookingById(id: string) {
  return this.get(`/api/admin/bookings/${id}`);  // ❌ 使用外部 API
}

static async updateBooking(id: string, data: any) {
  return this.put(`/api/admin/bookings/${id}`, data);  // ❌ 使用外部 API
}
```

**修改後**:
```typescript
// 訂單管理（使用內部 API）
static async getBookings(params?: any) {
  return this.internalGet('/api/admin/bookings', { params });  // ✅ 使用內部 API
}

static async getBookingById(id: string) {
  return this.internalGet(`/api/admin/bookings/${id}`);  // ✅ 使用內部 API
}

static async updateBooking(id: string, data: any) {
  return this.internalPut(`/api/admin/bookings/${id}`, data);  // ✅ 使用內部 API
}
```

## 💡 開發心得和經驗總結

### 1. 架構理解的重要性

**問題根源**:
- 沒有清楚區分 Next.js API 路由和外部後端 API
- 所有 API 請求使用同一個配置

**教訓**:
- 需要清楚了解應用的架構
- Next.js 的 API 路由是內部的，不需要跨域請求
- 外部後端 API 需要完整的 URL

### 2. 錯誤訊息的價值

**"Network Error" 的含義**:
- 不是 HTTP 錯誤（404, 500 等）
- 是連接層面的錯誤
- 通常表示無法連接到服務器

**診斷方法**:
- 檢查請求的完整 URL
- 檢查服務器是否運行
- 檢查端口是否正確

### 3. 代碼組織

**良好實踐**:
- ✅ 分離內部和外部 API 客戶端
- ✅ 使用不同的方法名（`get` vs `internalGet`）
- ✅ 清晰的日誌訊息（"External API" vs "Internal API"）

**改進空間**:
- 可以使用 TypeScript 類型區分
- 可以創建專門的 API 類別
