## 📝 犯過的錯誤和教訓

### 錯誤 1: 假設測試帳號已存在於 Supabase

**錯誤**: 以為測試帳號在 Firebase Auth 中存在，就一定在 Supabase 中存在

**教訓**: 
- Firebase Auth 和 Supabase 是兩個獨立的系統
- 需要確保資料同步
- 應該有自動同步機制（例如 Firebase Auth Trigger）

### 錯誤 2: 直接使用 drivers 表的欄位

**錯誤**: 假設 `drivers` 表有 `first_name`, `last_name`, `phone` 欄位

**教訓**:
- 在使用表欄位前，必須先查看表結構定義
- 不要假設表結構，要確認
- 使用 JOIN 查詢時，要清楚每個表的欄位

### 錯誤 3: 沒有考慮唯一性約束

**錯誤**: 使用重複的車牌號碼創建 drivers 記錄

**教訓**:
- 注意表的唯一性約束（UNIQUE constraint）
- 測試數據也要使用唯一的值
- 創建記錄前先檢查是否已存在
