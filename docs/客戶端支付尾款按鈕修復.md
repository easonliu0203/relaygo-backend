# 客戶端支付尾款按鈕修復

## 🐛 問題描述

### 嚴重的用戶體驗問題：客戶無法完成尾款支付

**問題場景**：
1. 客戶點擊「結束行程」
2. 系統自動導航到「支付尾款」頁面
3. **問題發生**：客戶在支付尾款頁面時，不小心：
   - 滑掉畫面（關閉頁面）
   - 點擊其他分頁（例如：「我的訂單」、「首頁」等）
   - 按下返回鍵
4. 客戶想要回來支付尾款時：
   - 訂單狀態顯示「待付尾款」（`awaitingBalance`）✅ 正確
   - 訂單在「我的訂單 > 進行中」頁面顯示 ✅ 正確
   - **但是**：點擊訂單進入詳情頁面後，**沒有「支付尾款」按鈕** ❌ 問題
5. **結果**：客戶無法完成支付，訂單卡在「待付尾款」狀態

### 影響範圍
- **客戶端**：`mobile/lib/apps/customer/presentation/pages/order_detail_page.dart`
- **影響用戶**：所有需要支付尾款的客戶
- **嚴重程度**：🔴 高（阻止客戶完成支付流程）

---

## ✅ 修復方案

### 在客戶端訂單詳情頁面添加「支付尾款」按鈕

**顯示條件**：
- 訂單狀態為 `BookingStatus.awaitingBalance`（待付尾款）

**按鈕設計**：
- **文字**：「立即支付尾款 (NT$ {balanceAmount})」
- **顏色**：金色 `0xFFFFB74D`（與 `awaitingBalance` 狀態顏色一致）
- **圖示**：`Icons.payment`
- **位置**：在「結束行程」按鈕之後，「聯絡司機」按鈕之前

**點擊行為**：
- 導航到支付尾款頁面：`/payment-balance/${order.id}`
- 與「結束行程」後的自動導航邏輯相同

## 💡 技術細節

### 按鈕顯示邏輯

```dart
if (order.status == BookingStatus.awaitingBalance) ...[
  // 顯示「支付尾款」按鈕
]
```

**說明**：
- 只有當訂單狀態為 `awaitingBalance` 時才顯示
- 使用 Dart 的 spread operator (`...`) 來展開 widget 列表
- 與其他按鈕的顯示邏輯保持一致

### 導航邏輯

```dart
onPressed: () {
  // 導航到支付尾款頁面
  context.push('/payment-balance/${order.id}');
}
```

**說明**：
- 使用 `go_router` 的 `context.push()` 方法
- 路由路徑：`/payment-balance/${order.id}`
- 與「結束行程」後的自動導航邏輯完全相同

### 金額顯示

```dart
Text(
  '立即支付尾款 (NT\$ ${order.balanceAmount.toStringAsFixed(0)})',
  style: const TextStyle(fontSize: 16, fontWeight: FontWeight.bold),
)
```

**說明**：
- 使用 `order.balanceAmount` 計算屬性（在 `BookingOrder` 模型中定義）
- 使用 `toStringAsFixed(0)` 格式化為整數（不顯示小數點）
- 顯示格式：「立即支付尾款 (NT$ 1500)」
