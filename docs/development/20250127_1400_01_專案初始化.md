## 概述

建立包車/接送叫車 APP 專案的完整架構，包含 Flutter 移動應用、Next.js 後台管理系統、Node.js 後端 API 服務，以及 PostgreSQL 資料庫設計。

## 專案需求分析

### 核心功能
1. **三角色系統**: 乘客、司機、公司後台
2. **預約流程**: 下單 → 派單 → 支付訂金 → 行程執行 → 每日結帳 → 評價
3. **即時功能**: 定位追蹤、聊天翻譯
4. **計費系統**: 車型價格、超時計費、推薦獎金
5. **多語言支援**: 國際化與即時翻譯

### 技術架構選擇
- **前端**: Flutter (跨平台移動應用)
- **後台**: Next.js + React + TypeScript
- **後端**: Node.js + Express + TypeScript
- **資料庫**: Supabase (PostgreSQL)
- **即時服務**: Firebase (認證、推播、聊天)
- **翻譯服務**: GPT-4o mini

### 5. 資料庫設計
建立完整的 PostgreSQL 資料庫結構，包含：
- 用戶管理 (users, user_profiles, drivers)
- 預約系統 (bookings, multi_day_trips)
- 支付系統 (payments, vehicle_pricing)
- 即時功能 (driver_locations, chat_rooms, chat_messages)
- 推薦系統 (referral_codes, referrals)
- 評價系統 (reviews)
- 系統設定 (system_settings)

### 6. 共用類型定義
建立 TypeScript 類型定義，確保前後端資料結構一致：
- 基礎實體介面
- 業務邏輯類型
- API 回應格式
- 常數定義
