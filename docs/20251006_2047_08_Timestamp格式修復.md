# Timestamp 格式修復

**日期**：2025-10-06 20:47  
**編號**：08  
**主旨**：修復 Firestore Timestamp 格式錯誤導致客戶端解析失敗

---

## 📋 問題描述

### 症狀
1. ✅ Firestore `orders_rt` 集合有新訂單
2. ✅ GeoPoint 格式正確（顯示為經緯度座標）
3. ❌ 客戶端從「訂單不存在」變成「載入訂單失敗」
4. ❌ 表示找到訂單但解析失敗

### 錯誤分析

**客戶端行為變化**：
- **修復 GeoPoint 前**：「訂單不存在」（找不到訂單或 GeoPoint 解析失敗）
- **修復 GeoPoint 後**：「載入訂單失敗」（找到訂單但其他欄位解析失敗）

**推論**：GeoPoint 已修復，但還有其他欄位格式問題

---

## 🔍 問題診斷過程

### 1. 檢查 Firestore 資料

**Edge Function 日誌**：
```json
{
  "bookingTime": "2025-10-06T05:12:00",           // ❌ 字串格式
  "createdAt": "2025-10-06T12:12:59.257565+00:00", // ❌ 字串格式
  "matchedAt": null,
  "completedAt": null
}
```

**Firestore 存儲**：
```
bookingTime: "2025-10-06T05:12:00"           // ❌ string 類型
createdAt: "2025-10-06T12:12:59.257565+00:00" // ❌ string 類型
```

---

### 2. 檢查客戶端期望

**檔案**：`mobile/lib/core/models/booking_order.dart`

**客戶端解析代碼**：
```dart
factory BookingOrder.fromFirestore(DocumentSnapshot doc) {
  final data = doc.data() as Map<String, dynamic>;
  
  return BookingOrder(
    ...
    bookingTime: (data['bookingTime'] as Timestamp).toDate(),  // ✅ 期望 Timestamp
    ...
    createdAt: (data['createdAt'] as Timestamp).toDate(),      // ✅ 期望 Timestamp
    matchedAt: data['matchedAt'] != null 
        ? (data['matchedAt'] as Timestamp).toDate()            // ✅ 期望 Timestamp
        : null,
    completedAt: data['completedAt'] != null 
        ? (data['completedAt'] as Timestamp).toDate()          // ✅ 期望 Timestamp
        : null,
  );
}
```

**發現**：
- ✅ 客戶端期望 `bookingTime`、`createdAt`、`matchedAt`、`completedAt` 都是 **Firestore Timestamp** 類型
- ❌ Edge Function 寫入的是 **string** 類型
- ❌ Dart 無法將 string 轉換為 Timestamp，導致解析失敗

---

### 3. 根本原因

**問題**：
- Edge Function 直接將 ISO 8601 字串寫入 Firestore
- Firestore 將其存儲為 **string** 類型
- 客戶端期望 **timestamp** 類型
- 導致類型轉換失敗：`(data['bookingTime'] as Timestamp)` 拋出異常

**類似於 GeoPoint 問題**：
- GeoPoint：需要使用 `geoPointValue` 格式
- Timestamp：需要使用 `timestampValue` 格式

---

## 🔧 修復方案

### 修改 1：更新 `syncBookingToFirestore` 函數

**目的**：標記時間欄位，讓 `convertToFirestoreFields` 知道需要轉換為 Timestamp

#### 修改前（錯誤）

```typescript
const firestoreData = {
  ...
  // 時間資訊
  bookingTime: bookingTime,  // ❌ 字串格式

  ...
  // 時間戳記
  createdAt: bookingData.createdAt,  // ❌ 字串格式
  matchedAt: bookingData.actualStartTime || null,
  completedAt: bookingData.actualEndTime || null,
}
```

**問題**：
- 直接使用字串
- Firestore 存儲為 **string** 類型

---

#### 修改後（正確）✅

```typescript
const firestoreData = {
  ...
  // 時間資訊（使用 _timestamp 標記，convertToFirestoreFields 會轉換為 Firestore Timestamp）
  bookingTime: {
    _timestamp: bookingTimeStr,  // ✅ 使用 _timestamp 標記
  },

  ...
  // 時間戳記（使用 _timestamp 標記）
  createdAt: {
    _timestamp: bookingData.createdAt,  // ✅ 使用 _timestamp 標記
  },
  matchedAt: bookingData.actualStartTime ? {
    _timestamp: bookingData.actualStartTime,  // ✅ 使用 _timestamp 標記
  } : null,
  completedAt: bookingData.actualEndTime ? {
    _timestamp: bookingData.actualEndTime,  // ✅ 使用 _timestamp 標記
  } : null,
}
```

**關鍵變更**：
- ✅ 使用 `{ _timestamp: "..." }` 格式包裝時間字串
- ✅ `convertToFirestoreFields` 會檢測 `_timestamp` 並轉換為 Firestore Timestamp

---

### 修改 2：更新 `convertToFirestoreFields` 函數

**目的**：檢測 `_timestamp` 標記並轉換為 Firestore REST API 的 `timestampValue` 格式

#### 修改前（錯誤）

```typescript
function convertToFirestoreFields(data: any): any {
  const fields: any = {}

  for (const [key, value] of Object.entries(data)) {
    ...
    } else if (typeof value === 'object') {
      // 檢查是否是 GeoPoint 格式
      if ('_latitude' in value && '_longitude' in value) {
        fields[key] = {
          geoPointValue: { ... }
        }
      } else {
        // ❌ 所有其他對象都當作 Map 處理
        fields[key] = { mapValue: { fields: convertToFirestoreFields(value) } }
      }
    }
  }

  return fields
}
```

**問題**：
- 沒有檢測 Timestamp 格式
- 包含 `_timestamp` 的對象會被當作普通 Map 處理

---

#### 修改後（正確）✅

```typescript
function convertToFirestoreFields(data: any): any {
  const fields: any = {}

  for (const [key, value] of Object.entries(data)) {
    ...
    } else if (typeof value === 'object') {
      // 檢查是否是 GeoPoint 格式（包含 _latitude 和 _longitude）
      if ('_latitude' in value && '_longitude' in value) {
        fields[key] = {
          geoPointValue: {
            latitude: value._latitude,
            longitude: value._longitude,
          }
        }
      }
      // ✅ 新增：檢查是否是 Timestamp 格式（包含 _timestamp）
      else if ('_timestamp' in value) {
        const timestampStr = value._timestamp as string
        // 將 ISO 8601 字串轉換為 Firestore Timestamp
        const date = new Date(timestampStr)
        fields[key] = {
          timestampValue: date.toISOString()  // ✅ 使用 timestampValue
        }
      }
      // 處理其他嵌套對象
      else {
        fields[key] = { mapValue: { fields: convertToFirestoreFields(value) } }
      }
    }
  }

  return fields
}
```

**關鍵變更**：
- ✅ 檢測包含 `_timestamp` 的對象
- ✅ 轉換為 Firestore REST API 的 `timestampValue` 格式
- ✅ Firestore 存儲為 **timestamp** 類型

## 💡 開發心得和經驗總結

### 1. Firestore 資料類型的重要性（再次強調）

**教訓**：
- Firestore 有特定的資料類型：string, number, boolean, **geopoint**, **timestamp**, map, array
- 不能隨意使用普通的 JavaScript 值
- 必須使用 Firestore REST API 的正確格式

**Timestamp 的正確格式**：
```json
{
  "timestampValue": "2025-10-06T05:12:00.000Z"
}
```

---

### 2. 客戶端錯誤訊息的價值

**有效的診斷**：
- 「訂單不存在」→ 「載入訂單失敗」的變化非常有價值
- 表示 GeoPoint 修復成功，但還有其他問題
- 逐步排除問題，更容易診斷

**改進**：
- 在客戶端添加更詳細的錯誤訊息
- 記錄具體是哪個欄位解析失敗
- 方便診斷和修復

---

### 3. 遇到的困難

**困難 1：不熟悉 Firestore REST API 的 Timestamp 格式**
- 最初以為字串就可以
- 實際上需要使用 `timestampValue` 格式
- 解決：查閱 Firestore REST API 文檔

**困難 2：如何在 Edge Function 中表示 Timestamp**
- 不能直接使用 Firestore SDK（Deno 環境）
- 需要使用 REST API 的格式
- 解決：使用 `_timestamp` 作為標記，在 `convertToFirestoreFields` 中轉換

---

### 4. 犯過的錯誤

**錯誤 1：假設字串就是 Timestamp**
- 最初直接寫入 ISO 8601 字串
- 導致 Firestore 存儲為 string 類型
- 教訓：必須使用正確的 Firestore 資料類型

**錯誤 2：只修復了 GeoPoint，沒有檢查其他欄位**
- 修復 GeoPoint 後以為問題解決了
- 實際上還有 Timestamp 格式問題
- 教訓：全面檢查所有欄位的資料類型
