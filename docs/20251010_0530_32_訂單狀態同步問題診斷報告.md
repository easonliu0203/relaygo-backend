## 📋 問題描述

### 問題 1: 客戶端訂單狀態不更新

**當前狀況**:
- 在客戶端（customer app）的「我的訂單」頁面中

- 即使公司端（web-admin）已經配對司機或取消訂單，客戶端的訂單狀態仍然不會改變

**當前狀況**:

## 🔍 診斷過程

### 1. 系統架構分析

**CQRS 架構**:
```
寫入模型（Write Model）: Supabase PostgreSQL
讀取模型（Read Model）: Firestore

資料流：
1. 公司端 API → Supabase (寫入)
2. Supabase Trigger → outbox 表 (事件佇列)
3. Cron Job (每 30 秒) → sync-to-firestore Edge Function
4. Edge Function → Firestore (雙寫: orders_rt 和 bookings)
5. 客戶端/司機端 → Firestore (讀取)
```

## 🎯 根本原因總結

### 原因 1: Trigger 缺少司機詳細資訊

**問題**:
- Trigger 只包含 `driverId`（Firebase UID）
- 沒有包含司機的姓名、電話、車型、車牌、評分等資訊

**影響**:
- 客戶端無法顯示司機的詳細資訊
- 只能看到 `driverId`，但無法顯示給用戶

### 原因 2: Edge Function 沒有查詢司機資訊

**問題**:
- Edge Function 直接使用 outbox 的 payload
- 沒有查詢 Supabase 獲取司機的詳細資訊

**影響**:
- Firestore 中只有 `driverId`，沒有司機詳細資訊

### 原因 3: 同步延遲

**問題**:
- Cron Job 每 30 秒執行一次
- 配對司機後，最多需要等待 30 秒才能同步到 Firestore

**影響**:
- 客戶端和司機端看到的資料有延遲
- 用戶體驗不佳

### 原因 4: Cron Job 可能未正確設置

**問題**:
- Cron Job 需要手動在 Supabase Dashboard 中設置
- 可能沒有正確設置或啟用

**影響**:
- 如果 Cron Job 沒有運行，outbox 事件永遠不會被處理
- Firestore 永遠不會更新

## 💡 修復方案

### 方案 1: 修改 Trigger 包含司機詳細資訊（推薦）

**優點**:
- 一次性解決問題
- 所有訂單變更都會自動包含司機資訊
- 不需要修改 Edge Function

**缺點**:
- Trigger 會變得更複雜
- 需要多次查詢資料庫（users、user_profiles、drivers 表）

**實作**:
```sql
CREATE OR REPLACE FUNCTION bookings_to_outbox()
RETURNS TRIGGER AS $$
DECLARE
  driver_info JSONB;
BEGIN
  -- 查詢司機詳細資訊
  IF NEW.driver_id IS NOT NULL THEN
    SELECT jsonb_build_object(
      'id', u.firebase_uid,
      'name', CONCAT(up.first_name, ' ', up.last_name),
      'phone', up.phone,
      'vehicleType', d.vehicle_type,
      'vehiclePlate', d.vehicle_plate,
      'rating', d.rating
    ) INTO driver_info
    FROM users u
    LEFT JOIN user_profiles up ON u.id = up.user_id
    LEFT JOIN drivers d ON u.id = d.user_id
    WHERE u.id = NEW.driver_id;
  ELSE
    driver_info := NULL;
  END IF;

  INSERT INTO outbox (
    aggregate_type,
    aggregate_id,
    event_type,
    payload
  ) VALUES (
    'booking',
    NEW.id::TEXT,
    CASE
      WHEN TG_OP = 'INSERT' THEN 'created'
      WHEN TG_OP = 'UPDATE' THEN 'updated'
      WHEN TG_OP = 'DELETE' THEN 'deleted'
    END,
    jsonb_build_object(
      'id', NEW.id,
      'bookingNumber', NEW.booking_number,
      'customerId', (SELECT firebase_uid FROM users WHERE id = NEW.customer_id),
      'status', NEW.status,
      -- ... 其他欄位 ...
      'driver', driver_info,  -- 完整的司機資訊
      -- ... 其他欄位 ...
    )
  );

  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### 方案 2: 修改 Edge Function 查詢司機資訊

**優點**:
- Trigger 保持簡單
- Edge Function 可以靈活處理資料

**缺點**:
- Edge Function 需要查詢 Supabase
- 增加 Edge Function 的複雜度和執行時間

**實作**:
```typescript
async function syncBookingToFirestore(event: OutboxEvent): Promise<void> {
  const bookingId = event.aggregate_id
  const bookingData = event.payload

  // 如果有 driverId，查詢司機詳細資訊
  let driverInfo = null
  if (bookingData.driverId) {
    const { data: driver } = await supabase
      .from('users')
      .select(`
        firebase_uid,
        user_profiles!user_id (
          first_name,
          last_name,
          phone
        ),
        drivers!user_id (
          vehicle_type,
          vehicle_plate,
          rating
        )
      `)
      .eq('firebase_uid', bookingData.driverId)
      .single()

    if (driver) {
      const profile = driver.user_profiles?.[0] || driver.user_profiles
      const driverData = driver.drivers?.[0] || driver.drivers
      
      driverInfo = {
        id: driver.firebase_uid,
        name: `${profile?.first_name || ''} ${profile?.last_name || ''}`.trim(),
        phone: profile?.phone || '',
        vehicleType: driverData?.vehicle_type || '',
        vehiclePlate: driverData?.vehicle_plate || '',
        rating: driverData?.rating || 0,
      }
    }
  }

  // 組合 Firestore 資料
  const firestoreData = {
    ...bookingData,
    driver: driverInfo,  // 添加完整的司機資訊
  }

  // 雙寫到 Firestore
  await upsertFirestoreDocument(bookingId, firestoreData)
}
```

### 方案 3: 配對司機 API 直接調用 Edge Function（即時同步）

**優點**:
- 即時同步，無延遲
- 確保配對司機後立即更新 Firestore

**缺點**:
- 增加 API 的執行時間
- 如果 Edge Function 失敗，API 也會失敗

**實作**:

    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.SUPABASE_SERVICE_ROLE_KEY}`,
        'Content-Type': 'application/json',
      },
    }
  );

    console.log('✅ Firestore 同步成功');
  }

  console.warn('⚠️ Firestore 同步異常:', error);
  // 不拋出錯誤，因為訂單已成功更新
}
