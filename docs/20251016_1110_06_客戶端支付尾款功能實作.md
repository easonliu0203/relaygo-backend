## 📋 功能需求

### 業務流程

**完整流程**:
1. 客戶點擊「結束行程」按鈕
2. 系統更新訂單狀態為 `trip_ended`
3. **自動跳轉到支付尾款頁面** ⭐
4. 顯示尾款金額（總金額 - 已付訂金）
5. 客戶選擇支付方式
6. 客戶點擊「支付」按鈕
7. 系統處理模擬支付
8. 更新訂單狀態為 `completed`
9. 跳轉到訂單完成頁面

### 技術要求

1. **參考現有實現**

   - 模擬支付的處理邏輯

2. **需要創建的組件**
   - Backend API: `POST /api/booking-flow/bookings/:id/pay-balance`
   - Flutter Service: `payBalance()` 方法
   - Flutter UI: 支付尾款頁面
   - Flutter UI: 訂單完成頁面

3. **業務邏輯**

   - 觸發 Firestore 同步

### 步驟 2: Flutter Service - payBalance() 方法

**文件**: `mobile/lib/core/services/booking_service.dart`

**新增方法**:
```dart
/// 支付尾款（使用 Supabase API）
Future<Map<String, dynamic>> payBalance(String bookingId, String paymentMethod) async {
  final user = _auth.currentUser;
  if (user == null) throw Exception('用戶未登入');

  final url = '$_baseUrl/booking-flow/bookings/$bookingId/pay-balance';
  final response = await http.post(
    Uri.parse(url),
    headers: {'Content-Type': 'application/json'},
    body: json.encode({
      'paymentMethod': paymentMethod,
      'customerUid': user.uid,
    }),
  );

  if (response.statusCode == 200) {
    final data = json.decode(response.body);
    return data['data'];
  } else {
    throw Exception('支付尾款失敗');
  }
}
```

**參數說明**:
- `bookingId`: 訂單 ID
- `paymentMethod`: 支付方式（credit_card, cash, etc.）

**返回值**:
```dart
{
  'paymentId': '...',
  'transactionId': '...',
  'amount': 45.0,
  'status': 'processing',
  'isAutoPayment': true,
  'estimatedProcessingTime': 2,
  'message': '尾款支付成功'
}
```
