# 司機端訂單配對問題修復

**日期**: 2025-10-10 08:00  
**問題**: 公司端配對訂單後，司機端應用中看不到訂單  
**狀態**: ✅ 已修復

---

## 📋 問題描述

### 症狀

1. 在公司端（web-admin）的待處理訂單頁面手動配對訂單給測試司機
2. 配對成功後，司機端應用的訂單列表中沒有顯示該訂單
3. 終端機日誌顯示權限錯誤（已修復）

### 測試帳號

- **司機端**: `driver.test@relaygo.com` / `RelayGO2024!Driver`
- **Firebase UID**: `CMfTxhJFlUVDkosJPyUoJvKjCQk1`

---

## 🔍 問題診斷

### 診斷步驟

1. **檢查 Firestore 權限規則** ✅
   - 規則正確，已包含 `driverId` 權限檢查
   - 已成功部署

2. **檢查 Firestore 索引** ✅
   - 索引已創建並部署
   - 包含 `driverId` + `createdAt` 和 `driverId` + `status` + `createdAt`

3. **檢查 Firestore 資料** ⚠️
   - 發現所有訂單的狀態都是 `pending`
   - 沒有訂單的 `driverId` 欄位被設置

4. **檢查公司端配對 API** ❌
   - **根本原因**：公司端配對時將狀態設置為 `'assigned'`
   - 司機端查詢時只查詢 `'matched'` 和 `'inProgress'` 狀態
   - 狀態不匹配導致司機端查詢不到訂單

### 根本原因

**狀態定義不一致**：

| 位置 | 狀態值 | 說明 |
|------|--------|------|
| 公司端 API | `'assigned'` | 配對司機時設置的狀態 |
| Flutter 應用 | `'matched'`, `'inProgress'`, `'completed'`, `'cancelled'`, `'pending'` | 定義的 5 個狀態 |
| 司機端查詢 | `'matched'`, `'inProgress'` | 查詢進行中訂單的狀態 |

**問題**：`'assigned'` 不在 Flutter 應用定義的狀態中，導致司機端查詢不到訂單。

---

## 🔧 修復方案

### 選擇的方案

**修改公司端 API，將狀態設置為 `'matched'` 而不是 `'assigned'`**

**理由**：
1. 更簡單，只需修改後端 API
2. 保持 Flutter 應用的狀態定義一致
3. `'matched'` 語義上更符合「已配對」的含義
4. 避免修改多個前端組件

### 修改的文件

#### 1. 手動配對 API

**文件**: `web-admin/src/app/api/admin/bookings/[id]/assign-driver/route.ts`

**修改內容**:

// 第 166 行：將狀態從 'assigned' 改為 'matched'
const { data: updatedBooking, error: updateError } = await db.supabase
  .from('bookings')
  .update({
    driver_id: driverId,
    status: 'matched',  // 使用 'matched' 而不是 'assigned'，與 Flutter 應用的狀態定義一致
    updated_at: new Date().toISOString(),
  })
  .eq('id', bookingId)
  .select()
  .single();
