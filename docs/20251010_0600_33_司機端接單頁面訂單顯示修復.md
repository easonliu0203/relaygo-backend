## 📋 問題描述

### 問題現象

**當前狀況**:
1. 在司機端（driver app）的接單頁面中
2. 當公司端（web-admin）配對司機後
3. 司機端的接單頁面沒有顯示已配對的訂單
4. 司機無法看到被分配給自己的訂單

**預期行為**:
- 當公司端配對司機後，司機端的接單頁面應該立即顯示該訂單
- 司機應該能看到訂單的詳細資訊（客戶姓名、上車地點、目的地、預約時間等）
- 司機應該能對訂單進行操作（確認接單、出發、到達等）

**客戶端查詢邏輯**:
```dart
/// 獲取用戶的訂單列表（從 Firestore 鏡像讀取）
Stream<List<BookingOrder>> getUserBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  // 從 orders_rt 鏡像集合讀取（Read-Only）
  return _firestore
      .collection('orders_rt')
      .where('customerId', isEqualTo: currentUserId)
      .orderBy('createdAt', descending: true)
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());

```

**分析**:
- ✅ 客戶端使用 `customerId` 查詢訂單

- ✅ 從 `orders_rt` 集合讀取

### 4. 根本原因總結

**問題**:
- ❌ 司機端訂單頁面沒有實作查詢邏輯
- ❌ 沒有創建司機端的訂單查詢方法
- ❌ 沒有創建司機端的 Riverpod Provider

**影響**:
- 司機端無法查詢 Firestore 中 `driverId` 等於當前司機的訂單
- 即使 Firestore 資料正確，司機端也看不到訂單

## 🔧 修復方案

### 1. 在 BookingService 中添加司機端查詢方法

**文件**: `mobile/lib/core/services/booking_service.dart`

**新增方法**:
```dart
/// 獲取司機的訂單列表（從 Firestore 鏡像讀取）
/// 用於司機端即時畫面展示，資料來自 Supabase 的單向鏡像
Stream<List<BookingOrder>> getDriverBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  // 從 orders_rt 鏡像集合讀取（Read-Only）
  // 查詢 driverId 等於當前司機的訂單
  return _firestore
      .collection('orders_rt')
      .where('driverId', isEqualTo: currentUserId)
      .orderBy('createdAt', descending: true)
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());
}

/// 獲取司機的進行中訂單（從 Firestore 鏡像讀取）
/// 用於司機端即時畫面展示，資料來自 Supabase 的單向鏡像
Stream<List<BookingOrder>> getDriverActiveBookings() {
  if (currentUserId == null) {
    return Stream.value([]);
  }

  // 從 orders_rt 鏡像集合讀取（Read-Only）
  // 查詢 driverId 等於當前司機且狀態為進行中的訂單
  return _firestore
      .collection('orders_rt')
      .where('driverId', isEqualTo: currentUserId)
      .where('status', whereIn: [
        BookingStatus.matched.name,
        BookingStatus.inProgress.name,
      ])
      .orderBy('createdAt', descending: true)
      .snapshots()
      .map((snapshot) => snapshot.docs
          .map((doc) => BookingOrder.fromFirestore(doc))
          .toList());
}
```

**關鍵點**:
- ✅ 使用 `driverId` 查詢（而不是 `customerId`）
- ✅ 使用 Firestore 的即時監聽（`snapshots()`）
- ✅ 從 `orders_rt` 集合讀取
- ✅ 支持查詢所有訂單和進行中訂單

### 2. 創建司機端 Riverpod Provider

**文件**: `mobile/lib/apps/driver/providers/driver_booking_provider.dart`（新建）

**代碼**:
```dart
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:mobile/core/models/booking_order.dart';
import 'package:mobile/core/services/booking_service.dart';

/// 司機的所有訂單列表 Provider
final driverBookingsProvider = StreamProvider<List<BookingOrder>>((ref) {
  final bookingService = ref.watch(bookingServiceProvider);
  return bookingService.getDriverBookings();
});

/// 司機的進行中訂單列表 Provider
final driverActiveBookingsProvider = StreamProvider<List<BookingOrder>>((ref) {
  final bookingService = ref.watch(bookingServiceProvider);
  return bookingService.getDriverActiveBookings();
});
```

**關鍵點**:
- ✅ 使用 `StreamProvider` 提供即時資料流
- ✅ 依賴 `bookingServiceProvider`
- ✅ 提供兩個 Provider：所有訂單和進行中訂單

#### 3.2 進行中訂單列表
```dart
Widget _buildActiveOrdersList() {
  final activeOrdersAsync = ref.watch(driverActiveBookingsProvider);

  return activeOrdersAsync.when(
    data: (orders) {
      if (orders.isEmpty) {
        return _buildEmptyState(
          icon: Icons.assignment_outlined,
          title: '沒有進行中的訂單',
          subtitle: '您目前沒有進行中的訂單',
        );
      }

      return RefreshIndicator(
        onRefresh: () async {
          ref.invalidate(driverActiveBookingsProvider);
        },
        child: ListView.builder(
          padding: const EdgeInsets.all(16.0),
          itemCount: orders.length,
          itemBuilder: (context, index) {
            return _buildOrderCard(orders[index]);
          },
        ),
      );
    },
    loading: () => const Center(child: CircularProgressIndicator()),
    error: (error, stack) => _buildErrorState(error.toString()),
  );
}
```

## 📝 開發心得

### 1. CQRS 架構的查詢模式

**學習**:
- 客戶端使用 `customerId` 查詢訂單
- 司機端使用 `driverId` 查詢訂單
- 兩者都從同一個 Firestore 集合（`orders_rt`）讀取
- 這是 CQRS 架構的典型查詢模式

**最佳實踐**:
- 為不同角色創建專門的查詢方法
- 使用 Firestore 的即時監聽（`snapshots()`）
- 使用 Riverpod Provider 管理資料流

### 2. Firestore 查詢優化

**查詢條件**:
```dart
.where('driverId', isEqualTo: currentUserId)
.where('status', whereIn: [
  BookingStatus.matched.name,
  BookingStatus.inProgress.name,
])
.orderBy('createdAt', descending: true)
```

**注意事項**:
- Firestore 複合查詢需要創建索引
- `whereIn` 最多支持 10 個值
- `orderBy` 必須在 `where` 之後

### 3. Flutter UI 設計

**組件選擇**:
- `TabController` - 實作分頁
- `RefreshIndicator` - 下拉刷新
- `ListView.builder` - 高效列表渲染
- `Card` - 訂單卡片容器

**狀態管理**:
- `ConsumerStatefulWidget` - 支持 Riverpod 和生命週期
- `ref.watch()` - 監聽 Provider
- `ref.invalidate()` - 刷新資料
