# 多應用程式架構建立指南

**日期**: 2025-09-28 06:00  
**編號**: 08  
**主旨**: Flutter 多應用程式架構建立 - 客戶端與司機端應用程式  

## 📋 **專案概述**

成功建立了支援客戶端和司機端的多應用程式架構，使用 Flutter Flavors 技術實現一個程式碼庫支援兩個不同的應用程式。

## 🏗️ **架構設計**

### **技術選型**
- **架構模式**: Flutter Flavors (產品變體)
- **狀態管理**: Riverpod
- **路由管理**: GoRouter
- **UI 框架**: Material Design 3
- **國際化**: Flutter Localizations

### **優勢分析**
✅ **程式碼共用**: 核心業務邏輯和 UI 組件可重複使用  
✅ **獨立部署**: 兩個應用程式可獨立建置和發布  
✅ **配置隔離**: 不同的 Bundle ID、Firebase 配置、主題色彩  
✅ **維護效率**: 單一程式碼庫，降低維護成本

## ⚙️ **Flavors 配置**

### **Android Flavors**
```kotlin
// android/app/build.gradle.kts
flavorDimensions += "app"
productFlavors {
    create("customer") {
        dimension = "app"
        applicationId = "com.ridebooking.customer"
        versionNameSuffix = "-customer"
        resValue("string", "app_name", "包車服務 - 客戶端")
    }
    create("driver") {
        dimension = "app"
        applicationId = "com.ridebooking.driver"
        versionNameSuffix = "-driver"
        resValue("string", "app_name", "包車服務 - 司機端")
    }
}
```

### **應用程式識別**
- **客戶端**: `com.ridebooking.customer`
- **司機端**: `com.ridebooking.driver`
- **主題色彩**: 客戶端藍色 (#2196F3)，司機端綠色 (#4CAF50)

## 🚀 **建置和執行**

### **開發環境執行**
```bash
# 執行客戶端應用程式
flutter run --flavor customer --target lib/apps/customer/main_customer.dart

# 執行司機端應用程式
flutter run --flavor driver --target lib/apps/driver/main_driver.dart
```

### **建置 APK**
```bash
# 建置客戶端 APK
flutter build apk --flavor customer --target lib/apps/customer/main_customer.dart

# 建置司機端 APK
flutter build apk --flavor driver --target lib/apps/driver/main_driver.dart
```

### **建置 iOS**
```bash
# 建置客戶端 iOS
flutter build ios --flavor customer --target lib/apps/customer/main_customer.dart

# 建置司機端 iOS
flutter build ios --flavor driver --target lib/apps/driver/main_driver.dart
```
