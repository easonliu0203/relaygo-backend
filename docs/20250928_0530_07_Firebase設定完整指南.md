## 🔥 **第一步：Firebase Console 服務啟用**

### 1.1 必須啟用的服務
請在 Firebase Console 中啟用以下服務：

#### **Authentication (認證)**
- 進入 Authentication > Sign-in method
- 啟用以下登入方式：
  - ✅ Email/Password
  - ✅ Phone (電話號碼)
  - ✅ Google
  - ✅ Apple (iOS)
  - ✅ Anonymous (匿名)

#### **Firestore Database (文檔資料庫)**
- 進入 Firestore Database
- 點擊「建立資料庫」
- 選擇「以測試模式啟動」(稍後會套用安全規則)
- 選擇地區：asia-east1 (台灣)

#### **Realtime Database (即時資料庫)**
- 進入 Realtime Database
- 點擊「建立資料庫」
- 選擇「以鎖定模式啟動」
- 選擇地區：asia-southeast1

#### **Cloud Storage (雲端儲存)**
- 進入 Storage
- 點擊「開始使用」
- 選擇「以測試模式啟動」
- 選擇地區：asia-east1

#### **Cloud Messaging (推播通知)**
- 進入 Cloud Messaging
- 系統會自動啟用

#### **Crashlytics (錯誤報告)**
- 進入 Crashlytics
- 點擊「啟用 Crashlytics」

## 📱 **第二步：下載配置檔案**

### 2.1 Android 配置
1. 進入專案設定 > 您的應用程式 > Android 應用程式
2. 下載 `google-services.json`
3. 將檔案放置到：`mobile/android/app/google-services.json`

### 2.2 iOS 配置
1. 進入專案設定 > 您的應用程式 > iOS 應用程式
2. 下載 `GoogleService-Info.plist`
3. 將檔案放置到：`mobile/ios/Runner/GoogleService-Info.plist`

### 2.3 Web 配置
1. 進入專案設定 > 您的應用程式 > Web 應用程式
2. 複製 Firebase 配置物件
3. 更新 `mobile/lib/core/config/firebase_options.dart` 中的配置值

## ⚙️ **第三步：環境變數配置**

### 3.1 更新 mobile/.env
從 Firebase Console 獲取以下值並更新 `mobile/.env`：

```env
# Firebase 配置 (從 Firebase Console 獲取)
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_API_KEY=your-api-key
FIREBASE_APP_ID=your-app-id
FIREBASE_MESSAGING_SENDER_ID=your-sender-id
FIREBASE_AUTH_DOMAIN=your-project-id.firebaseapp.com
FIREBASE_STORAGE_BUCKET=your-project-id.appspot.com
FIREBASE_MEASUREMENT_ID=your-measurement-id

# API 端點
API_BASE_URL=http://localhost:3000/api
SUPABASE_URL=your-supabase-url
SUPABASE_ANON_KEY=your-supabase-anon-key

# Google Maps API
GOOGLE_MAPS_API_KEY=your-google-maps-api-key

# 功能開關
ENABLE_CRASHLYTICS=true
ENABLE_ANALYTICS=true
ENABLE_PERFORMANCE=true

# 開發環境設定
DEBUG_MODE=true
LOG_LEVEL=debug
```

### 3.2 更新 backend/.env
```env
# 服務器配置
PORT=3000
NODE_ENV=development
JWT_SECRET=your-jwt-secret-key

# Firebase Admin SDK (從 Firebase Console > 專案設定 > 服務帳戶)
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project-id.iam.gserviceaccount.com

# FCM 服務器金鑰 (從 Firebase Console > 專案設定 > Cloud Messaging)
FCM_SERVER_KEY=your-fcm-server-key

# Supabase 配置
SUPABASE_URL=your-supabase-url
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key

# Redis 配置
REDIS_URL=redis://localhost:6379

# 支付配置
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
```

## 🔒 **第四步：部署安全規則**

### 4.1 安裝 Firebase CLI
```bash
npm install -g firebase-tools
firebase login
```

### 4.2 初始化 Firebase 專案
```bash
cd firebase
firebase init
# 選擇：Firestore, Realtime Database, Storage, Hosting
# 選擇現有專案
```

### 4.3 部署安全規則
```bash
firebase deploy --only firestore:rules
firebase deploy --only database:rules
firebase deploy --only storage:rules
```

## 🚨 **常見問題和解決方案**

### Q1: "DefaultFirebaseOptions have not been configured"
**解決方案**: 確保 `firebase_options.dart` 中的配置值已從 Firebase Console 正確複製

### Q2: "FirebaseApp with name '[DEFAULT]' doesn't exist"
**解決方案**: 確保在使用 Firebase 服務前已調用 `Firebase.initializeApp()`

### Q3: "Permission denied" 錯誤
**解決方案**: 檢查 Firestore 和 Storage 安全規則是否正確部署

### Q4: FCM Token 為 null
**解決方案**: 確保已正確配置 `google-services.json` 和 `GoogleService-Info.plist`
