## 📋 功能概述

在公司端（web-admin）訂單管理系統中，開發司機配對功能，用於將待處理訂單中未分配司機的訂單分配給可用司機。

### 核心功能

1. **自動配對功能** - 一鍵自動分配所有未分配的訂單
2. **手動配對功能** - 為單個訂單手動選擇司機
3. **衝突檢查** - 自動檢查時間段重疊，避免重複派單
4. **負載平衡** - 優先分配給訂單數量較少的司機

   - 更新訂單

## 🧮 衝突檢查算法

### 時間衝突判斷邏輯

**問題**: 如何判斷兩個訂單的時間段是否重疊？

**解決方案**: 使用區間重疊算法

**算法**:
```
訂單 A: start_time_A, duration_A
訂單 B: start_time_B, duration_B

end_time_A = start_time_A + duration_A
end_time_B = start_time_B + duration_B

重疊條件: (start_A < end_B) AND (end_A > start_B)
```

**示例**:
```
訂單 A: 14:00 - 17:00 (3小時)
訂單 B: 16:00 - 18:00 (2小時)

檢查: (14:00 < 18:00) AND (17:00 > 16:00)
結果: true (有衝突)
```

**實作**:
```typescript
// 將時間轉換為分鐘數
const [hours, minutes] = time.split(':').map(Number);
const startMinutes = hours * 60 + minutes;
const endMinutes = startMinutes + (duration * 60);

// 檢查重疊
const overlap = (startMinutes < existingEndMinutes) && (endMinutes > existingStartMinutes);
```

### 衝突檢查流程

1. **獲取司機在該日期的所有訂單**
   ```sql
   SELECT * FROM bookings
   WHERE driver_id = ?
   AND start_date = ?
   AND status IN ('assigned', 'driver_confirmed', 'driver_departed', 'driver_arrived', 'trip_started')
   ```

2. **對每個現有訂單檢查時間重疊**
   ```typescript
   const hasConflict = existingBookings.some(booking => {
     const existingStartMinutes = ...;
     const existingEndMinutes = ...;
     return (startMinutes < existingEndMinutes) && (endMinutes > existingStartMinutes);
   });
   ```

3. **返回衝突狀態**
   - 有衝突：標記為紅色，禁用選擇按鈕
   - 無衝突：標記為綠色，可以選擇

#### 自動分配司機 API

**文件**: `web-admin/src/app/api/admin/bookings/auto-assign/route.ts`

**關鍵代碼**:
```typescript
// 1. 獲取未分配司機的訂單
const { data: bookings } = await db.supabase
  .from('bookings')
  .select('*')
  .is('driver_id', null)
  .in('status', ['pending_payment', 'paid_deposit']);

// 2. 獲取所有可用司機
const { data: drivers } = await db.supabase
  .from('users')
  .select(`
    id,
    drivers!user_id (id, vehicle_type, is_available)
  `)
  .eq('role', 'driver');

const availableDrivers = (drivers || []).filter(driver => {
  const driverInfo = driver.drivers?.[0] || driver.drivers;
  return driverInfo && driverInfo.is_available;
});

// 3. 統計每個司機的訂單數量
const driverBookingCounts = new Map<string, number>();
const driverBookingsByDate = new Map<string, any[]>();

// 4. 對每個訂單分配司機
for (const booking of bookings) {
  // 找到符合車型的司機
  const matchingDrivers = availableDrivers.filter(driver => {
    const driverInfo = driver.drivers?.[0] || driver.drivers;
    return driverInfo && driverInfo.vehicle_type === booking.vehicle_type;
  });

  // 檢查時間衝突並選擇最佳司機
  let selectedDriver = null;
  let minBookings = Infinity;

  for (const driver of matchingDrivers) {
    const key = `${driver.id}_${booking.start_date}`;
    const driverDayBookings = driverBookingsByDate.get(key) || [];

    const hasConflict = driverDayBookings.some(existingBooking => {
      // 時間重疊檢查
      return (startMinutes < existingEndMinutes) && (endMinutes > existingStartMinutes);
    });

    if (!hasConflict) {
      // 選擇訂單數量最少的司機（負載平衡）
      const bookingCount = driverBookingCounts.get(driver.id) || 0;
      if (bookingCount < minBookings) {
        minBookings = bookingCount;
        selectedDriver = driver;
      }
    }
  }

  // 分配司機
  if (selectedDriver) {
    await db.supabase
      .from('bookings')
      .update({
        driver_id: selectedDriver.id,
        status: 'assigned',
        updated_at: new Date().toISOString(),
      })
      .eq('id', booking.id);

    results.assigned++;
  } else {
    results.failed++;
  }
}
```

### 2. 負載平衡策略

**問題**: 如何公平分配訂單給司機？

**解決方案**:
- 統計每個司機的當前訂單數量
- 選擇訂單數量最少的司機
- 使用 Map 資料結構快速查詢

**實作**:
```typescript
const driverBookingCounts = new Map<string, number>();

// 統計訂單數量
existingBookings.forEach(booking => {
  driverBookingCounts.set(
    booking.driver_id,
    (driverBookingCounts.get(booking.driver_id) || 0) + 1
  );
});

// 選擇訂單最少的司機
let selectedDriver = null;
let minBookings = Infinity;

for (const driver of matchingDrivers) {
  const bookingCount = driverBookingCounts.get(driver.id) || 0;
  if (bookingCount < minBookings) {
    minBookings = bookingCount;
    selectedDriver = driver;
  }
}
```

## 🚧 遇到的困難和解決方法

### 困難 1: 時間衝突檢查的複雜性

**問題**:
- 需要考慮日期和時間
- 需要計算結束時間
- 需要檢查多個訂單

**解決**:
- 將時間轉換為分鐘數統一處理
- 使用區間重疊算法
- 按日期分組減少檢查次數

### 困難 2: 負載平衡的實作

**問題**:
- 需要統計每個司機的訂單數量
- 需要在分配過程中動態更新

**解決**:
- 使用 Map 資料結構
- 在分配後立即更新計數
- 確保下一個訂單使用最新的計數

### 困難 3: 前端狀態管理

**問題**:
- 多個狀態需要管理（loading, modal, drivers, etc.）
- 需要在不同操作間切換狀態

**解決**:
- 使用 useState 管理各個狀態
- 清晰的狀態命名
- 在適當的時機重置狀態
