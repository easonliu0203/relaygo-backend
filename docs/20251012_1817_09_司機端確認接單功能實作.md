## 📋 需求描述

### 業務需求

實現完整的訂單確認流程，當司機確認接單後自動創建聊天室，讓客戶和司機可以開始溝通。

### 功能需求

1. **司機端訂單詳情頁面**:
   - 當訂單狀態為 `matched`（已配對）時，顯示「確認接單」按鈕
   - 點擊按鈕後顯示確認對話框
   - 調用 Backend API 確認接單
   - 顯示載入狀態（防止重複點擊）
   - API 調用成功後更新訂單狀態並顯示成功訊息

2. **Backend API**:
   - 端點: `POST /api/booking-flow/bookings/:bookingId/accept`
   - 驗證司機權限
   - 更新訂單狀態為 `driver_confirmed`
   - 自動創建聊天室

3. **業務流程**:
   ```
   公司端配對訂單 (matched)
     ↓
   司機看到「確認接單」按鈕
     ↓
   司機點擊確認
     ↓
   Backend 更新狀態 (driver_confirmed)
     ↓
   Backend 自動創建聊天室
     ↓
   客戶端和司機端都能看到聊天室
   ```

```dart
/// 司機確認接單（使用 Backend API）
Future<Map<String, dynamic>> driverAcceptBooking(String bookingId) async {
  try {
    final user = _auth.currentUser;
    if (user == null) {
      throw Exception('用戶未登入');
    }

    // 調用 Backend API
    final url = '$_baseUrl/booking-flow/bookings/$bookingId/accept';
    
    final requestBody = {
      'driverUid': user.uid,
    };

    final response = await http.post(
      Uri.parse(url),
      headers: {
        'Content-Type': 'application/json',
      },
      body: json.encode(requestBody),
    );

    if (response.statusCode == 200) {
      final data = json.decode(response.body);
      if (data['success'] == true) {
        debugPrint('[BookingService] 司機確認接單成功');
        debugPrint('[BookingService] 聊天室已創建: ${data['data']['chatRoom']}');
        return data['data'];
      } else {
        throw Exception(data['error'] ?? '確認接單失敗');
      }
    } else {
      final error = json.decode(response.body);
      throw Exception(error['error'] ?? '確認接單失敗');
    }
  } catch (e) {
    debugPrint('[BookingService] 司機確認接單失敗: $e');
    throw Exception('確認接單失敗: $e');
  }
}
```
</augment_code_snippet>

**關鍵點**:
- 使用 `http.post` 調用 Backend API
- 傳遞 `driverUid` 參數（Firebase UID）
- Backend 會自動創建聊天室
- 返回更新後的訂單資料和聊天室資訊

```typescript
async driverAcceptBooking(req: Request, res: Response): Promise<void> {
  try {
    const { bookingId } = req.params;
    const driverId = req.user?.id;

    // 1. 檢查預約狀態
    const booking = await this.getBookingById(bookingId);
    
    // 2. 驗證司機權限
    if (booking.driverId !== driverId) {
      throw new Error('無權限確認此訂單');
    }

    // 3. 更新資料庫
    await this.updateBookingStatus(bookingId, BookingStatus.DRIVER_CONFIRMED);

    // 4. 創建聊天室
    const chatRoom = await chatService.createChatRoom(
      bookingId,
      booking.customerId,
      driverId
    );

    res.json({
      success: true,
      data: {
        booking: updatedBooking,
        chatRoom: chatRoom
      }
    });
  } catch (error) {
    res.status(400).json({
      success: false,
      error: error.message
    });
  }
}
```

## 📊 訂單狀態流程

### 完整流程

```
pending_payment (待付訂金)
  ↓ 客戶支付訂金
paid_deposit (已付訂金)
  ↓ 公司端配對司機
matched (已配對) ← 顯示「確認接單」按鈕
  ↓ 司機確認接單 ← 本次實作
driver_confirmed (司機確認) ← 聊天室在這裡自動創建 ✅
  ↓ 司機出發
driver_departed (司機出發)
  ↓ 司機到達
driver_arrived (司機到達)
  ↓ 客戶開始行程
in_progress (進行中)
  ↓ 客戶結束行程
completed (已完成)
```

### 聊天室創建時機

**觸發條件**: 訂單狀態從 `matched` 變更為 `driver_confirmed`  
**觸發方式**: 司機點擊「確認接單」按鈕  
**處理位置**: `BookingFlowController.driverAcceptBooking()`  
**創建方法**: `chatService.createChatRoom(bookingId, customerId, driverId)`

## 💡 技術要點

### 1. 異步處理

使用 `async/await` 處理 API 調用：
```dart
onPressed: () async {
  // 異步處理邏輯

}
```

### 2. 載入狀態

使用 `showDialog` 顯示載入狀態，防止重複點擊：
```dart
showDialog(
  context: context,
  barrierDismissible: false,  // 不可點擊外部關閉
  builder: (context) => const Center(
    child: CircularProgressIndicator(),
  ),
);
```

### 3. 錯誤處理

使用 `try-catch` 捕獲異常並顯示錯誤訊息：
```dart
try {
  await bookingService.driverAcceptBooking(order.id);
} catch (e) {
  ScaffoldMessenger.of(context).showSnackBar(
    SnackBar(content: Text('❌ 接單失敗: $e')),
  );
}
```

### 4. 狀態刷新

使用 Riverpod 的 `invalidate` 刷新訂單資料：
```dart
ref.invalidate(bookingProvider(order.id));
```
