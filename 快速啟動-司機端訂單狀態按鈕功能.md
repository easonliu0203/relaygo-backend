# 快速啟動 - 司機端訂單狀態按鈕功能

## 🚀 5 分鐘快速啟動指南

### 前置條件檢查

確保以下服務正在運行：
- ✅ Backend API (http://localhost:3001)
- ✅ Supabase (已連接)
- ✅ Firebase (已配置)
- ✅ Supabase Edge Function (sync-to-firestore)

### 步驟 1: 更新數據庫約束 (1 分鐘)

在 Supabase Dashboard 的 SQL Editor 中執行：

```sql
-- 刪除舊約束
ALTER TABLE bookings DROP CONSTRAINT IF EXISTS bookings_status_check;

-- 添加新約束（包含 matched 狀態）
ALTER TABLE bookings ADD CONSTRAINT bookings_status_check 
CHECK (status IN (
    'pending_payment', 'paid_deposit', 'matched', 'assigned', 
    'driver_confirmed', 'driver_departed', 'driver_arrived', 
    'in_progress', 'trip_started', 'trip_ended', 
    'pending_balance', 'completed', 'cancelled', 'refunded'
));
```

或者執行完整腳本：
```bash
# 在 Supabase SQL Editor 中執行
supabase/update-booking-status-constraint.sql
```

### 步驟 2: 重啟 Backend (1 分鐘)

```bash
cd backend
npm install  # 如果是第一次運行
npm run dev
```

確認 Backend 正在運行：
```bash
curl http://localhost:3001/api/booking-flow/test
```

應該返回：
```json
{
  "success": true,
  "message": "Booking Flow API is working",
  "timestamp": "2025-10-15T..."
}
```

### 步驟 3: 編譯 Flutter 應用 (2 分鐘)

```bash
cd mobile
flutter pub get
flutter run  # 或者 flutter build apk
```

### 步驟 4: 測試功能 (1 分鐘)

#### 方法 A: 使用 Flutter 應用測試

1. 使用司機帳號登入
2. 找到一個狀態為「已配對」的訂單
3. 進入訂單詳情頁面
4. 應該看到「出發前往載客 🚗」按鈕（藍色）
5. 點擊按鈕 → 確認 → 等待成功提示
6. 應該看到「抵達上車地點 📍」按鈕（綠色）
7. 點擊按鈕 → 確認 → 等待成功提示
8. 打開聊天室，確認收到兩條系統訊息

#### 方法 B: 使用 API 測試

```bash
# 測試司機出發
curl -X POST http://localhost:3001/api/booking-flow/bookings/{BOOKING_ID}/depart \
  -H "Content-Type: application/json" \
  -d '{"driverUid": "{DRIVER_FIREBASE_UID}"}'

# 測試司機到達
curl -X POST http://localhost:3001/api/booking-flow/bookings/{BOOKING_ID}/arrive \
  -H "Content-Type: application/json" \
  -d '{"driverUid": "{DRIVER_FIREBASE_UID}"}'
```

### 步驟 5: 驗證結果

#### 檢查 Supabase
```sql
-- 查看訂單狀態
SELECT id, booking_number, status, updated_at 
FROM bookings 
WHERE id = '{BOOKING_ID}';

-- 查看 outbox 事件
SELECT * FROM outbox 
WHERE aggregate_id = '{BOOKING_ID}' 
ORDER BY created_at DESC 
LIMIT 5;
```

#### 檢查 Firestore

在 Firebase Console 中：
1. 打開 `chat_rooms/{BOOKING_ID}/messages` 集合
2. 應該看到兩條系統訊息：
   - 「司機已出發，正在前往上車地點 🚗」
   - 「司機已到達上車地點，請準備上車 📍」

3. 打開 `orders_rt/{BOOKING_ID}` 文檔
4. 確認 `status` 欄位為 `inProgress`

## 🎯 完整測試流程

### 測試場景 1: 正常流程

1. **準備訂單**
   - 公司端派單給司機
   - 訂單狀態：`matched` (Supabase)
   - Firestore 狀態：`pending`

2. **司機確認接單**
   - 司機點擊「確認接單」
   - 訂單狀態：`driver_confirmed` (Supabase)
   - Firestore 狀態：`matched`
   - 聊天室創建成功

3. **司機出發**
   - 司機點擊「出發前往載客 🚗」
   - 訂單狀態：`driver_departed` (Supabase)
   - Firestore 狀態：`inProgress`
   - 聊天室收到訊息：「司機已出發，正在前往上車地點 🚗」

4. **司機到達**
   - 司機點擊「抵達上車地點 📍」
   - 訂單狀態：`driver_arrived` (Supabase)
   - Firestore 狀態：`inProgress`
   - 聊天室收到訊息：「司機已到達上車地點，請準備上車 📍」

### 測試場景 2: 錯誤處理

1. **權限錯誤**
   - 使用其他司機帳號
   - 點擊按鈕
   - 應該顯示：「無權限操作此訂單」

2. **狀態錯誤**
   - 在錯誤的訂單狀態下點擊
   - 應該顯示：「訂單狀態不正確」

3. **網路錯誤**
   - 關閉網路
   - 點擊按鈕
   - 應該顯示網路錯誤訊息

## 🔧 常見問題快速解決

### 問題 1: 按鈕沒有顯示

**檢查**:
```dart
// 在 Flutter 中打印訂單狀態
debugPrint('Order status: ${order.status}');
```

**解決**:
- 確保訂單狀態為 `matched` (顯示出發按鈕) 或 `inProgress` (顯示到達按鈕)
- 檢查 Firestore 同步是否正常

### 問題 2: API 返回 404

**檢查**:
```bash
# 確認 Backend 正在運行
curl http://localhost:3001/api/booking-flow/test

# 確認路由正確
curl http://localhost:3001/api/booking-flow/bookings/test-id/depart
```

**解決**:
- 重啟 Backend 服務
- 檢查 `bookingFlow-minimal.ts` 是否正確加載

### 問題 3: 系統訊息沒有發送

**檢查**:
```bash
# 查看 Backend 日誌
# 應該看到：
# [API] ✅ 系統訊息已發送
```

**解決**:
- 檢查 Firebase Admin SDK 配置
- 檢查 `.env` 文件中的 Firebase 憑證
- 檢查 Firestore 安全規則

### 問題 4: Firestore 沒有同步

**檢查**:
```sql
-- 查看 outbox 表
SELECT * FROM outbox 
WHERE processed_at IS NULL 
ORDER BY created_at DESC;
```

**解決**:
- 確保 Edge Function 正在運行
- 檢查 Edge Function 日誌
- 等待 5-30 秒（同步有延遲）
