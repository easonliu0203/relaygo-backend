## 📚 技術要點

### 為什麼使用 StatefulWidget?

**問題**: 在函數中創建 TextEditingController 並在 .then() 中 dispose

```dart
void _showDialog() {
  final controller = TextEditingController();  // ❌ 問題
  showDialog(...).then((_) {
    controller.dispose();  // ❌ 時機錯誤
  });
}
```

**原因**:
1. `.then()` 回調在對話框開始關閉時就執行
2. 但 TextField 可能還在使用 controller
3. 導致 `_dependents.isEmpty` 錯誤

**解決**: 使用 StatefulWidget

```dart
class _MyDialog extends StatefulWidget {
  @override
  State<_MyDialog> createState() => _MyDialogState();
}

class _MyDialogState extends State<_MyDialog> {
  late final TextEditingController _controller;
  
  @override
  void initState() {
    super.initState();
    _controller = TextEditingController();  // ✅ 在 initState 創建
  }
  
  @override
  void dispose() {
    _controller.dispose();  // ✅ 在 dispose 釋放
    super.dispose();
  }
  
  @override
  Widget build(BuildContext context) {
    return AlertDialog(...);  // ✅ 使用 _controller
  }
}
```

這樣 Flutter 會在正確的時機自動調用 dispose。
