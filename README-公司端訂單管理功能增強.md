## 📋 功能概述

在公司端（web-admin）訂單管理系統中，增強訂單管理功能，允許管理員對已配對司機的訂單進行調整和取消操作。

### 核心功能

1. **更改已配對的司機** - 允許管理員重新分配司機
2. **取消訂單功能** - 實作完整的取消訂單流程

---

## 🚀 快速開始

### 1. 更改司機

**步驟**:
1. 訪問訂單頁面：
   - 待處理訂單頁面：http://localhost:3001/orders/pending
   - 主訂單頁面（所有訂單）：http://localhost:3001/orders
2. 找到已分配司機的訂單（有司機資訊的訂單）
3. 點擊訂單操作欄的「🔄 更改司機」按鈕
4. 在彈出的對話框中查看可用司機列表
5. 選擇一位無衝突的司機（綠色標記）
6. 點擊「選擇」按鈕確認更改

**注意事項**:
- ⚠️ 如果司機已出發或已到達，會顯示確認對話框
- ❌ 有時間衝突的司機會被標記為紅色，無法選擇
- ✅ 成功更改後，訂單列表會自動更新

### 2. 取消訂單

**步驟**:
1. 訪問訂單頁面：
   - 待處理訂單頁面：http://localhost:3001/orders/pending
   - 主訂單頁面（所有訂單）：http://localhost:3001/orders
2. 找到要取消的訂單
3. 點擊訂單操作欄的「❌ 取消訂單」按鈕
4. 在彈出的對話框中輸入取消原因（必填）
5. 點擊「確認取消」按鈕

**注意事項**:
- ⚠️ 如果訂單已分配司機，會顯示警告訊息
- ❌ 已完成或行程已開始的訂單無法取消
- ✅ 取消原因最多 500 字

## 🔧 API 端點

### 1. 更改司機

**端點**: `PUT /api/admin/bookings/[id]/change-driver`

**請求體**:
```json
{
  "newDriverId": "new-driver-uuid",
  "reason": "更改原因（可選）"
}
```

**返回**:
```json
{
  "success": true,
  "data": {
    "id": "booking-uuid",
    "driver_id": "new-driver-uuid",
    "previous_driver_id": "old-driver-uuid",
    "updated_at": "2025-10-10T04:00:00Z"
  },
  "message": "成功更改司機"
}
```

**允許更改的訂單狀態**:
- `assigned` - 已派單
- `driver_confirmed` - 司機已確認
- `driver_departed` - 司機已出發
- `driver_arrived` - 司機已到達

### 2. 取消訂單

**端點**: `POST /api/admin/bookings/[id]/cancel`

**請求體**:
```json
{
  "reason": "取消原因（必填）",
  "cancelledBy": "admin-uuid（可選）"
}
```

**返回**:
```json
{
  "success": true,
  "data": {
    "id": "booking-uuid",
    "status": "cancelled",
    "cancelled_at": "2025-10-10T04:00:00Z",
    "cancellation_reason": "客戶取消"
  },
  "message": "訂單已取消",
  "hadDriver": true
}
```

**不允許取消的訂單狀態**:
- `cancelled` - 已取消
- `completed` - 已完成
- `trip_started` - 行程已開始

## 🐛 常見問題

### Q1: 為什麼無法更改司機？

**可能原因**:
1. 訂單狀態不允許更改（已完成、已取消、行程已開始）
2. 新司機不可用
3. 新司機在該時間段有其他訂單（時間衝突）

**解決方法**:
- 檢查訂單狀態
- 選擇標記為綠色「可用」的司機
- 避免選擇有時間衝突的司機

### Q2: 為什麼無法取消訂單？

**可能原因**:
1. 訂單已經被取消
2. 訂單已完成
3. 行程已開始

**解決方法**:
- 檢查訂單狀態
- 只能取消未完成且未開始的訂單

### Q3: 取消訂單後司機會怎樣？

**答案**:
- 訂單狀態變為 `cancelled`
- 司機會被自動釋放（由司機端應用管理）
- 司機可以接受其他訂單

### Q4: 更改司機後原司機會怎樣？

**答案**:
- 訂單的 `driver_id` 更新為新司機
- 原司機會被釋放
- 系統記錄 `previous_driver_id`（在 API 返回中）
