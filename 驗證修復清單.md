## 🔍 常見問題排查

### 問題 1: 仍然出現 Network Error

**可能原因**:
- 瀏覽器快取未清除
- 服務未重新啟動

**解決方法**:
1. 清除瀏覽器快取
2. 硬性重新整理（Ctrl + Shift + R）
3. 重新啟動 web-admin 服務
4. 使用無痕模式測試

### 問題 2: 顯示「暫無訂單資料」

**可能原因**:
- Supabase 中沒有訂單資料
- 篩選條件過於嚴格

**解決方法**:
1. 檢查 Supabase 資料庫:
   ```sql
   SELECT COUNT(*) FROM bookings;
   ```
2. 清空所有篩選條件
3. 使用客戶端應用創建測試訂單

### 問題 3: 統計數據為 0

**可能原因**:
- API 返回的資料格式不正確
- 前端計算邏輯有問題

**解決方法**:
1. 檢查 Network 標籤的 API 回應
2. 確認回應包含 `total` 欄位
3. 檢查 Console 是否有錯誤

### 問題 4: 請求仍然發送到 port 3000

**可能原因**:
- 代碼未正確修改
- 服務未重新啟動

**解決方法**:
1. 確認 `web-admin/src/services/api.ts` 已修改
2. 確認使用 `internalGet` 而不是 `get`
3. 重新啟動服務
4. 清除瀏覽器快取
