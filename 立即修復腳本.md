#### 2. 添加環境變數

**點擊「Add variable」或「New secret」，依次添加以下 4 個變數**：

##### 變數 1：FIREBASE_PROJECT_ID

**Name**: `FIREBASE_PROJECT_ID`

**Value**: 您的 Firebase Project ID

**如何獲取**：
1. 打開：https://console.firebase.google.com
2. 選擇您的專案
3. 點擊齒輪圖標 → Project Settings
4. 在「General」標籤中找到「Project ID」
5. 複製並貼上

**範例值**：`my-charter-app-12345`

---

##### 變數 2：FIREBASE_API_KEY

**Name**: `FIREBASE_API_KEY`

**Value**: 您的 Firebase Web API Key

**如何獲取**：
1. 在 Firebase Console 的 Project Settings 中
2. 滾動到「Your apps」section
3. 找到您的 Web App（或點擊「Add app」→「Web」創建一個）
4. 複製「Web API Key」

**範例值**：`AIzaSyDXXXXXXXXXXXXXXXXXXXXXXXXXXXX`（以 AIza 開頭）

---

##### 變數 3：SUPABASE_URL

**Name**: `SUPABASE_URL`

**Value**: `https://vlyhwegpvpnjyocqmfqc.supabase.co`

**說明**：這是固定值，直接複製貼上即可。

---

##### 變數 4：SUPABASE_SERVICE_ROLE_KEY

**Name**: `SUPABASE_SERVICE_ROLE_KEY`

**Value**: 您的 Supabase Service Role Key

**如何獲取**：
1. 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/api
2. 找到「service_role」section（在 Project API keys 下方）
3. 點擊「Reveal」按鈕
4. 複製整個 key

**範例值**：`eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6...`（很長的 JWT token）

## 🆘 常見問題

### Q1: 找不到環境變數設置頁面

**A**: 
1. 確認您在正確的專案中
2. 路徑：Project Settings → Edge Functions
3. 或直接使用連結：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/functions

### Q2: 找不到 Firebase Web API Key

**A**:
1. Firebase Console → Project Settings
2. 滾動到「Your apps」
3. 如果沒有 Web App，點擊「Add app」→ 選擇「Web」（</>圖標）
4. 註冊 App 後會顯示 Web API Key

### Q3: 設置環境變數後仍然失敗

**A**:
1. 檢查環境變數的值是否正確（沒有多餘空格）
2. 查看 Edge Function 日誌獲取具體錯誤
3. 執行 `supabase/check-error-messages.sql` 查看錯誤分析
4. 告訴我具體的錯誤訊息

### Q4: 需要重新部署 Edge Function 嗎？

**A**: 不需要！環境變數會自動應用到所有 Edge Functions。
