## 📋 問題診斷結果

### 根本原因

**Flutter 應用在模擬器/真機上運行時，`localhost` 指向的是設備本身，而不是開發機器！**

**錯誤流程**:
```
Flutter 應用（Android 模擬器）
    ↓ 嘗試連接 http://localhost:3001
    ❌ localhost 指向模擬器本身
    ❌ 模擬器上沒有運行 API 服務器
    ❌ 連接失敗
```

**正確流程**:
```
Flutter 應用（Android 模擬器）
    ↓ 連接 http://10.0.2.2:3001
    ✅ 10.0.2.2 是 Android 模擬器訪問主機的特殊 IP
    ✅ 連接到開發機器上的 API 服務器
    ✅ 成功
```

## 🔧 修復方案

### 修復內容

**文件**: `mobile/lib/core/services/supabase_service.dart`

**修改**: 根據平台自動選擇正確的 API URL

**修改前**:
```dart
static const String _apiBaseUrl = 'http://localhost:3001/api';
```

**修改後**:
```dart
static String get _apiBaseUrl {
  try {
    if (Platform.isAndroid) {
      // Android 模擬器使用 10.0.2.2 訪問主機的 localhost
      return 'http://10.0.2.2:3001/api';
    } else if (Platform.isIOS) {
      // iOS 模擬器可以直接使用 localhost
      return 'http://localhost:3001/api';
    } else {
      // 其他平台（Web、Desktop）使用 localhost
      return 'http://localhost:3001/api';
    }
  } catch (e) {
    // 如果無法檢測平台，默認使用 Android 模擬器的地址
    return 'http://10.0.2.2:3001/api';
  }
}
```

## 🔍 不同平台的 API URL

### Android 模擬器

**URL**: `http://10.0.2.2:3001/api`

**說明**: 
- `10.0.2.2` 是 Android 模擬器訪問主機 localhost 的特殊 IP
- 這是 Android 模擬器的標準配置

### iOS 模擬器

**URL**: `http://localhost:3001/api`

**說明**:
- iOS 模擬器可以直接使用 localhost
- 也可以使用 `127.0.0.1`

### 真機測試

**URL**: `http://192.168.1.XXX:3001/api`（替換為開發機器的實際 IP）

**獲取開發機器 IP**:
```bash
# Windows
ipconfig

# 查找 IPv4 地址，例如：192.168.1.100
```

**修改代碼**（如果在真機上測試）:
```dart
static String get _apiBaseUrl {
  // 真機測試時使用開發機器的 IP
  return 'http://192.168.1.100:3001/api';
}
```

### Web 平台

**URL**: `http://localhost:3001/api`

**說明**:
- Web 應用在瀏覽器中運行
- 可以直接使用 localhost

```bash
node test-profile-api.js

# 3. 檢查防火牆

```

### 問題 2: 連接超時

**症狀**:
```
❌ [SupabaseService] 異常: TimeoutException
```

**原因**: 網路連接慢或 API 響應慢

**解決方案**:
```dart
// 增加超時時間
final response = await http.post(
  url,
  headers: {'Content-Type': 'application/json'},
  body: json.encode(requestData),
).timeout(Duration(seconds: 30)); // 增加到 30 秒
```

### 問題 3: CORS 錯誤（Web 平台）

**症狀**:
```
Access to XMLHttpRequest has been blocked by CORS policy
```

**解決方案**: 在 Next.js API 中添加 CORS 頭

```typescript
// web-admin/src/app/api/profile/upsert/route.ts
export async function POST(request: NextRequest) {
  const response = NextResponse.json({...});
  
  // 添加 CORS 頭
  response.headers.set('Access-Control-Allow-Origin', '*');
  response.headers.set('Access-Control-Allow-Methods', 'GET, POST, OPTIONS');
  response.headers.set('Access-Control-Allow-Headers', 'Content-Type');
  
  return response;
}
```

### 問題 4: API 返回 404

**症狀**:
```
📤 [SupabaseService] 響應狀態: 404
```

**原因**: API 路由不存在或路徑錯誤

**檢查**:
1. 確認文件存在: `web-admin/src/app/api/profile/upsert/route.ts`
2. 確認 URL 正確: `http://10.0.2.2:3001/api/profile/upsert`
3. 重新啟動 web-admin

## 💡 開發心得

### 1. 模擬器網路配置

**教訓**: 模擬器的網路配置與真機不同。

**Android 模擬器**:
- `localhost` → 模擬器本身
- `10.0.2.2` → 主機的 localhost
- `10.0.2.3` → 主機的路由器

**iOS 模擬器**:
- `localhost` → 主機的 localhost（直接可用）

### 2. 調試日誌的重要性

**教訓**: 詳細的日誌可以快速定位問題。

**最佳實踐**:
- 記錄請求 URL
- 記錄請求數據
- 記錄響應狀態
- 記錄響應內容
- 記錄異常信息

### 3. 平台檢測

**教訓**: 不同平台需要不同的配置。

**最佳實踐**:
```dart
import 'dart:io' show Platform;

if (Platform.isAndroid) {
  // Android 特定配置
} else if (Platform.isIOS) {
  // iOS 特定配置
}
```
