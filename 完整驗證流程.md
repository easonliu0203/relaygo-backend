### 階段 2：驗證 Firestore 資料（3 分鐘）

#### 步驟 2.1：檢查資料存在

1. **打開 Firebase Console**
   ```
   https://console.firebase.google.com
   ```

2. **選擇您的專案 → Firestore Database**

3. **查看 `orders_rt` 集合**

4. **確認有訂單文檔**
   - 應該看到多個訂單
   - 文檔 ID 應該是 UUID 格式

#### 步驟 2.2：檢查資料格式（⭐ 最重要）

1. **點擊任一訂單文檔**

2. **確認欄位和類型**：

   ✅ **基本資訊**：
   ```
   customerId: string "..."
   driverId: null
   status: string "pending"
   ```

   ✅ **地點資訊**：
   ```
   pickupAddress: string "..."
   pickupLocation: geopoint (25.033, 121.5654)    // ⭐ 必須是 geopoint
   dropoffAddress: string "..."                    // ⭐ 不是 destination
   dropoffLocation: geopoint (25.033, 121.5654)   // ⭐ 必須是 geopoint
   ```

   ✅ **時間資訊**：
   ```
   bookingTime: string "2025-10-05T09:00:00"      // ⭐ 組合的時間
   createdAt: string "..."
   ```

   ✅ **乘客和費用資訊**：
   ```
   passengerCount: number 1                        // ⭐ 必須存在
   luggageCount: null
   notes: string "..." 或 null
   estimatedFare: number 1000                      // ⭐ 必須存在
   depositAmount: number 300
   depositPaid: boolean false                      // ⭐ 必須存在
   ```

3. **檢查清單**：
   - [ ] `pickupLocation` 類型是 **geopoint**（不是 map）
   - [ ] `dropoffLocation` 類型是 **geopoint**（不是 map）
   - [ ] 有 `dropoffAddress` 欄位（不是 `destination`）
   - [ ] 有 `bookingTime` 欄位（不是分開的 `startDate` 和 `startTime`）
   - [ ] 有 `passengerCount` 欄位
   - [ ] 有 `estimatedFare` 欄位
   - [ ] 有 `depositPaid` 欄位

4. **如果格式不正確**：
   - 可能是舊資料
   - 創建新訂單測試
   - 或手動觸發 Edge Function 重新同步
