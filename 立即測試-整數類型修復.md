# 立即測試：整數類型修復

**問題**：客戶端 App 顯示 `type 'double' is not a subtype of type 'int'`  
**根本原因**：`passengerCount` 是 double 類型，客戶端期望 int 類型  
**修復**：已更新為正確的整數類型  
**狀態**：✅ 已部署，待測試

---

## 🎯 問題根源

### 錯誤訊息

**客戶端顯示**：
```
載入訂單失敗
type 'double' is not a subtype of type 'int'
```

**含義**：
- Dart 嘗試將 double 類型的值賦值給 int 類型的變數
- Dart 是強類型語言，不會自動轉換
- 必須使用正確的資料類型

---

### 資料類型不匹配

**Edge Function 寫入**（修復前）：
```
passengerCount: 1  ← 轉換為 doubleValue ❌
```

**Firestore 存儲**（修復前）：
```
passengerCount: 1  ← number (double) 類型 ❌
```

**客戶端期望**：
```dart
required int passengerCount  ← 期望 int 類型 ✅
```

**結果**：類型不匹配，解析失敗

---

## 🔧 修復內容

### 修復前（錯誤）

**Edge Function 寫入**：
```typescript
passengerCount: 1,  // ❌ 普通數值
```

**轉換為 Firestore REST API 格式**：
```json
{ "doubleValue": 1 }  // ❌ 轉換為 double
```

**Firestore 存儲**：
```
passengerCount: 1  ← number (double) 類型 ❌
```

**客戶端解析**：
```dart
❌ 錯誤：type 'double' is not a subtype of type 'int'
```

---

### 修復後（正確）✅

**Edge Function 寫入**：
```typescript
passengerCount: {
  _integer: 1  // ✅ 使用 _integer 標記
},
```

**轉換為 Firestore REST API 格式**：
```json
{ "integerValue": "1" }  // ✅ 轉換為 integer
```

**Firestore 存儲**：
```
passengerCount: 1  ← number (integer) 類型 ✅
```

**客戶端解析**：
```dart
✅ 成功：passengerCount 是 int 類型
```

## 💡 關鍵要點

### 1. Firestore 區分整數和浮點數

**必須使用正確的類型**：
- 整數：`integerValue`（例如：1, 2, 3）
- 浮點數：`doubleValue`（例如：1.5, 2.0, 3.14）
- 客戶端才能正確解析

### 2. Dart 是強類型語言

**不會自動轉換類型**：
- double 不能自動轉換為 int
- 必須明確轉換：`value.toInt()`
- 或確保資料類型正確

### 3. 如何檢查資料類型

**在 Firebase Console 中**：
- 打開 Firestore → 點擊文檔
- 查看欄位的顯示格式
- **integer**：顯示為 `1`（沒有小數點）
- **double**：顯示為 `1.0`（有小數點）
- **timestamp**：顯示為日期時間
- **geopoint**：顯示為座標
