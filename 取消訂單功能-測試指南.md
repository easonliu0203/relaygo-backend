## 📋 修復摘要

### 問題
```
'package:flutter/src/widgets/framework.dart': 
Failed assertion: line 6161 pos 14: '_dependents.isEmpty': is not true.
```

### 根本原因
在取消訂單對話框中，`TextEditingController` 在對話框關閉前被 dispose，導致 `TextField` 仍然持有對已銷毀 controller 的引用。

### 修復方案
使用 `showDialog().then()` 回調，在對話框完全關閉後才 dispose controller。

## 🔍 故障排除

### 問題 1：對話框無法打開

**症狀**：
- 點擊「取消訂單」按鈕沒有反應

**可能原因**：
- 訂單狀態不允許取消
- 按鈕被禁用

**解決方案**：
1. 檢查訂單狀態（只有 `pending` 和 `matched` 可以取消）
2. 檢查按鈕的 `onPressed` 是否為 null
3. 查看控制台日誌

---

### 問題 2：API 調用失敗

**症狀**：
- 顯示「取消失敗：...」訊息

**可能原因**：
- 管理後台未啟動
- API 路由錯誤
- 資料庫欄位缺失

**解決方案**：

**檢查管理後台**：
```bash
# 確認管理後台正在運行
curl http://localhost:3001/api/health
```

**檢查 API 路由**：
```bash
# 測試取消訂單 API
curl -X POST http://localhost:3001/api/bookings/{訂單ID}/cancel \
  -H "Content-Type: application/json" \
  -d '{
    "customerUid": "測試用戶UID",
    "reason": "測試取消功能"
  }'
```

**檢查資料庫欄位**：
```sql
-- 確認欄位存在
SELECT column_name 
FROM information_schema.columns 
WHERE table_name = 'bookings' 
  AND column_name IN ('cancellation_reason', 'cancelled_at');
```
