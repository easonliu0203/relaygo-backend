**症狀**：
- Firestore 有資料
- 客戶端顯示「訂單不存在」

**根本原因**：
- 欄位名稱不匹配

**修復方案**：
- 更新欄位映射

- 組合 `bookingTime`（從 `startDate` 和 `startTime`）

**修復時間**：2025-10-06 00:23  
**文檔**：`docs/20251006_0023_06_Firestore欄位映射修復.md`

### 問題 3：載入訂單失敗（GeoPoint 格式錯誤）✅ 已修復

**症狀**：
- 客戶端從「訂單不存在」變成「載入訂單失敗」
- 表示找到訂單但解析失敗

**根本原因**：
- Edge Function 寫入 `{ latitude: 0, longitude: 0 }`
- Firestore 存儲為 **map** 類型
- 客戶端期望 **geopoint** 類型
- Dart 無法將 Map 轉換為 GeoPoint

**修復方案**：
- 使用 `{ _latitude: ..., _longitude: ... }` 格式
- 在 `convertToFirestoreFields` 中檢測並轉換為 `geoPointValue`
- Firestore 存儲為 **geopoint** 類型

**修復時間**：2025-10-06 08:40  
**文檔**：`docs/20251006_0840_07_GeoPoint格式修復.md`

---

### 問題 4：載入訂單失敗（Timestamp 格式錯誤）✅ 已修復

**症狀**：
- GeoPoint 修復後仍然顯示「載入訂單失敗」
- 表示還有其他欄位格式問題

**根本原因**：
- Edge Function 寫入 ISO 8601 字串
- Firestore 存儲為 **string** 類型
- 客戶端期望 **timestamp** 類型
- Dart 無法將 string 轉換為 Timestamp

**修復方案**：
- 使用 `{ _timestamp: "..." }` 格式
- 在 `convertToFirestoreFields` 中檢測並轉換為 `timestampValue`
- Firestore 存儲為 **timestamp** 類型

**影響的欄位**：
- `bookingTime`
- `createdAt`
- `matchedAt`
- `completedAt`

**修復時間**：2025-10-06 20:47  
**文檔**：`docs/20251006_2047_08_Timestamp格式修復.md`

---

### 問題 5：載入訂單失敗（整數類型錯誤）✅ 已修復

**症狀**：
- GeoPoint 和 Timestamp 修復後仍然顯示「載入訂單失敗」
- 新錯誤：`type 'double' is not a subtype of type 'int'`

**根本原因**：
- Edge Function 將所有數值都轉換為 `doubleValue`
- Firestore 存儲為 **double** 類型
- 客戶端期望某些欄位是 **int** 類型
- Dart 無法自動將 double 轉換為 int

**修復方案**：
- 使用 `{ _integer: 1 }` 格式標記整數欄位
- 在 `convertToFirestoreFields` 中檢測並轉換為 `integerValue`
- 自動檢測 JavaScript 整數並使用 `integerValue`
- Firestore 存儲為 **integer** 類型

**影響的欄位**：
- `passengerCount`（必須是 int）
- `luggageCount`（必須是 int）

**修復時間**：2025-10-06 21:58  
**文檔**：`docs/20251006_2158_09_整數類型修復.md`

## 💡 關鍵經驗總結

### 1. Firestore 資料類型的重要性

**教訓**：
- Firestore 有特定的資料類型，不能隨意使用
- 必須使用正確的 REST API 格式
- 客戶端（特別是強類型語言）無法自動轉換

**正確的格式**：
- GeoPoint：`{ geoPointValue: { latitude: ..., longitude: ... } }`
- Timestamp：`{ timestampValue: "2025-10-06T06:09:00.000Z" }`
- Integer：`{ integerValue: "1" }`
- Double：`{ doubleValue: 0 }`

---

### 2. 錯誤訊息的價值

**診斷過程**：
1. 「訂單不存在」→ 欄位映射問題
2. 「載入訂單失敗」→ GeoPoint 格式問題
3. 「載入訂單失敗」→ Timestamp 格式問題
4. `type 'double' is not a subtype of type 'int'` → 整數類型問題

**每次錯誤訊息都更具體，幫助逐步縮小問題範圍**

---

### 3. 強類型語言的特性

**Dart 的類型系統**：
- 不會自動轉換類型
- Map 不能轉換為 GeoPoint
- String 不能轉換為 Timestamp
- Double 不能轉換為 int

**必須使用正確的資料類型**
