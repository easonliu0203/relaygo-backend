## 功能概述

在司機確認接單後，司機端訂單詳情頁面會顯示兩個狀態更新按鈕：

1. **出發前往載客 🚗** (藍色按鈕)
   - 顯示條件：訂單狀態為「已配對」
   - 點擊後：訂單狀態變為「進行中」
   - 自動通知：在聊天室發送「司機已出發，正在前往上車地點 🚗」

2. **抵達上車地點 📍** (綠色按鈕)
   - 顯示條件：訂單狀態為「進行中」
   - 點擊後：訂單狀態保持「進行中」（內部狀態更新）
   - 自動通知：在聊天室發送「司機已到達上車地點，請準備上車 📍」

## 技術架構

### 資料流程

```
司機點擊按鈕
    ↓
Flutter UI 顯示確認對話框
    ↓
用戶確認
    ↓
Flutter Service 調用 Backend API
    ↓
Backend API 驗證權限和狀態
    ↓
Backend API 更新 Supabase 訂單狀態
    ↓
Supabase Trigger 寫入 outbox 表
    ↓
Backend API 發送系統訊息到 Firestore
    ↓
Backend API 返回成功響應
    ↓
Flutter UI 顯示成功提示並刷新訂單
    ↓
Supabase Edge Function 消費 outbox 事件
    ↓
Edge Function 同步訂單狀態到 Firestore
    ↓
客戶端 Firestore 監聽器接收更新
    ↓
客戶端 UI 自動更新
```

### 狀態映射

| Supabase 狀態 | Firestore 狀態 | 顯示按鈕 |
|--------------|---------------|---------|
| `matched` | `pending` | 確認接單 |
| `driver_confirmed` | `matched` | 出發前往載客 |
| `driver_departed` | `inProgress` | 抵達上車地點 |
| `driver_arrived` | `inProgress` | (下一階段功能) |
