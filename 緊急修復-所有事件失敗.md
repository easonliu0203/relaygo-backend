**最可能的錯誤和修復方案**：

---

## 🚨 錯誤 A：環境變數未設置（最可能 90%）

### 錯誤訊息特徵
```
Cannot read property 'FIREBASE_PROJECT_ID' of undefined
或
FIREBASE_API_KEY is not defined
```

### 修復步驟（3 分鐘）

1. **打開環境變數設置**
   ```
   https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/functions
   ```

2. **設置以下環境變數**：

   **FIREBASE_PROJECT_ID**
   - 值：您的 Firebase Project ID
   - 獲取：Firebase Console → Project Settings → General → Project ID

   **FIREBASE_API_KEY**
   - 值：您的 Firebase Web API Key
   - 獲取：Firebase Console → Project Settings → General → Web API Key

   **SUPABASE_URL**
   - 值：`https://vlyhwegpvpnjyocqmfqc.supabase.co`

   **SUPABASE_SERVICE_ROLE_KEY**
   - 值：您的 Service Role Key
   - 獲取：Supabase Dashboard → Settings → API → service_role (點擊 Reveal)

3. **保存**

4. **重新測試**
   - 返回 Functions 頁面
   - 點擊 sync-to-firestore → Invoke
   - 應該看到 `success: 7`

---

## 🚨 錯誤 B：Firestore 權限問題（可能性 5%）

### 錯誤訊息特徵
```
Firestore 更新失敗 (403): Forbidden
或
Permission denied
```

### 修復步驟

1. **檢查 Firebase API Key**
   - 確認使用的是 Web API Key（不是 Server Key）
   - 重新獲取並更新環境變數

2. **檢查 Firestore 安全規則**
   - 打開 Firebase Console → Firestore → Rules
   - 確認允許寫入 `orders_rt` 集合

---

## 🚨 錯誤 C：Firebase Project ID 錯誤（可能性 3%）

### 錯誤訊息特徵
```
Firestore 更新失敗 (404): Not Found
或
Project not found
```

### 修復步驟

1. **重新獲取 Firebase Project ID**
   - Firebase Console → Project Settings → General
   - 複製正確的 Project ID

2. **更新環境變數**

---

## 🚨 錯誤 D：網路或其他問題（可能性 2%）

```
timeout
或
ETIMEDOUT
或
其他未知錯誤
```

### 修復步驟

1. **查看完整的 Edge Function 日誌**

3. **我會提供針對性修復**
