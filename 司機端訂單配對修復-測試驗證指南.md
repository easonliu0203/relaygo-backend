### 步驟 4: 檢查 Outbox 事件

```sql
-- 檢查 outbox 表中的同步事件
SELECT 
  id,
  aggregate_type,
  aggregate_id,
  event_type,
  payload->>'driverId' AS driver_id_in_payload,
  payload->>'status' AS status_in_payload,
  processed_at,
  created_at
FROM outbox
WHERE aggregate_type = 'booking'
  AND (payload->>'driverId' = 'CMfTxhJFlUVDkosJPyUoJvKjCQk1'
       OR aggregate_id IN (
         SELECT id::text FROM bookings WHERE driver_id = 'CMfTxhJFlUVDkosJPyUoJvKjCQk1'
       ))
ORDER BY created_at DESC
LIMIT 10;
```

**預期結果**:
- ✅ 找到對應的 outbox 事件
- ✅ `status_in_payload` 為 `'matched'`
- ✅ `driver_id_in_payload` 為 `'CMfTxhJFlUVDkosJPyUoJvKjCQk1'`

## 🔍 故障排除

### 問題 1: 公司端配對失敗

**症狀**: 點擊「確認配對」後顯示錯誤訊息

**檢查**:
1. 檢查瀏覽器控制台的錯誤訊息
2. 檢查 Supabase 連接是否正常
3. 檢查司機是否存在

**解決方案**:
```bash
# 檢查 Supabase 連接
cd d:\repo\web-admin
npm run dev
```

---

### 問題 2: Supabase 中沒有訂單

**症狀**: 查詢 Supabase 時找不到訂單

**檢查**:
1. 確認配對操作是否成功
2. 檢查 API 日誌
3. 檢查資料庫連接

**解決方案**:
```sql
-- 檢查所有訂單
SELECT id, status, driver_id, created_at
FROM bookings
ORDER BY created_at DESC
LIMIT 10;
```

---

### 問題 3: Firestore 同步失敗

**症狀**: Supabase 有訂單，但 Firestore 沒有

**檢查**:
1. 檢查 outbox 事件是否已處理
2. 檢查 Edge Function 日誌
3. 檢查 Firebase 服務帳號金鑰

**解決方案**:
```sql
-- 檢查未處理的 outbox 事件
SELECT *
FROM outbox
WHERE processed_at IS NULL
ORDER BY created_at DESC
LIMIT 10;
```

手動觸發同步：
```bash
curl -X POST https://YOUR_PROJECT_ID.supabase.co/functions/v1/sync-to-firestore \
  -H "Authorization: Bearer YOUR_SERVICE_ROLE_KEY"
```

---

### 問題 4: 司機端應用看不到訂單

**症狀**: Firestore 有訂單，但司機端應用看不到

3. 檢查應用日誌
