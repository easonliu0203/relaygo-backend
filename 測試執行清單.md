## 🐛 如果測試失敗

### 問題 1：管理後台編譯錯誤

**症狀**：
```
Error: Module not found: Can't resolve '@/lib/database'
```

**解決方案**：
1. 檢查文件：
   ```bash
   cat web-admin/src/app/api/bookings/[id]/cancel/route.ts | head -5
   ```
2. 確認第 2 行是：
   ```typescript
   import { DatabaseService } from '@/lib/supabase';
   ```
3. 如果不是，手動修改
4. 重啟管理後台

---

### 問題 2：API 返回 HTML

**症狀**：
```
[BookingService] 響應 Content-Type: text/html
```

**解決方案**：
1. 檢查管理後台終端
2. 確認沒有編譯錯誤
3. 重啟管理後台
4. 重新測試

---

### 問題 3：Flutter 框架錯誤

**症狀**：
```
'_dependents.isEmpty': is not true
```

**解決方案**：
1. 檢查文件：
   ```bash
   cat mobile/lib/apps/customer/presentation/pages/order_detail_page.dart | grep -A 5 ".then((reason)"
   ```
2. 確認有 `.then()` 回調
3. 執行 `flutter clean`
4. 重新運行 App
5. 重新測試

---

### 問題 4：取消失敗

**症狀**：
```
取消失敗：Exception: ...
```

**解決方案**：
1. 查看 Flutter 日誌中的「響應狀態碼」
2. 根據狀態碼診斷：
   - `400` → 驗證錯誤（檢查取消原因長度）
   - `403` → 權限錯誤（檢查用戶 UID）
   - `404` → 訂單不存在（檢查訂單 ID）
   - `500` → API 內部錯誤（檢查管理後台日誌）
