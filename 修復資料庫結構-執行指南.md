## 📋 問題描述

### 錯誤訊息

從管理後台日誌可以看到：

```
❌ 更新訂單失敗: {
  code: 'PGRST204',
  message: "Could not find the 'cancellation_reason' column of 'bookings' in the schema cache"
}
```

### 根本原因

`bookings` 表缺少以下欄位：
- `cancellation_reason` (TEXT) - 取消訂單的原因
- `cancelled_at` (TIMESTAMP WITH TIME ZONE) - 訂單取消的時間

## 🚀 修復步驟

### 方法 1：使用 Supabase Dashboard（推薦）⭐

#### 步驟 1：打開 Supabase Dashboard

1. 前往：https://supabase.com/dashboard
2. 登入您的帳號
3. 選擇專案：`vlyhwegpvpnjyocqmfqc`

#### 步驟 2：打開 SQL Editor

1. 在左側選單點擊「SQL Editor」
2. 點擊「New query」

#### 步驟 3：執行 SQL

複製以下 SQL 並執行：

```sql
-- 修復缺少的資料庫欄位
-- 日期：2025-10-08

-- 1. 添加 cancellation_reason 欄位
ALTER TABLE bookings
ADD COLUMN IF NOT EXISTS cancellation_reason TEXT;

-- 2. 添加 cancelled_at 欄位
ALTER TABLE bookings
ADD COLUMN IF NOT EXISTS cancelled_at TIMESTAMP WITH TIME ZONE;

-- 3. 添加註釋
COMMENT ON COLUMN bookings.cancellation_reason IS '取消訂單的原因（由客戶提供）';
COMMENT ON COLUMN bookings.cancelled_at IS '訂單取消的時間';

-- 4. 創建索引以提高查詢效能
CREATE INDEX IF NOT EXISTS idx_bookings_cancelled_at 
ON bookings(cancelled_at) 
WHERE cancelled_at IS NOT NULL;

-- 5. 驗證欄位已添加
SELECT 
    column_name, 
    data_type, 
    is_nullable
FROM information_schema.columns
WHERE table_schema = 'public' 
    AND table_name = 'bookings'
    AND column_name IN ('cancellation_reason', 'cancelled_at')
ORDER BY column_name;
```

#### 步驟 4：驗證結果

執行後應該看到：

```
column_name          | data_type                   | is_nullable
---------------------|-----------------------------|-----------
cancellation_reason  | text                        | YES
cancelled_at         | timestamp with time zone    | YES
```

如果看到這兩行，表示欄位已成功添加！
