
- [ ] pg_cron 擴展已啟用（應該已有）
  ```
  前往: Database → Extensions → 搜尋 "pg_cron" → Enable
  ```

- [ ] Edge Function `sync-to-firestore` 已部署並正常運行
  ```
  前往: Edge Functions → sync-to-firestore → 檢查狀態
  ```

### 3. 現有 Cron Job 檢查

- [ ] Cron Job `sync-orders-to-firestore` 存在且啟用
  ```sql
  SELECT * FROM cron.job WHERE jobname = 'sync-orders-to-firestore';
  ```

- [ ] Cron Job 最近有執行記錄
  ```sql
  SELECT * FROM cron.job_run_details 
  WHERE jobid = (SELECT jobid FROM cron.job WHERE jobname = 'sync-orders-to-firestore')
  ORDER BY start_time DESC LIMIT 5;
  ```

## 🚀 部署步驟

### 步驟 1: 執行 Migration

- [ ] 在 Supabase Dashboard SQL Editor 中打開
  ```
  URL: https://app.supabase.com/project/vlyhwegpvpnjyocqmfqc/sql
  ```

- [ ] 複製並執行 `supabase/migrations/20251016_create_realtime_sync_trigger.sql`

- [ ] 驗證執行結果
  ```sql
  SELECT * FROM get_realtime_sync_status();
  ```
  
  預期結果：
  - `trigger_exists`: false（尚未啟用）
  - `config_enabled`: false（默認停用）

### 步驟 2: 配置 Service Role Key（可選）

- [ ] 前往 Supabase Dashboard
  ```
  Settings → Database → Custom Postgres Configuration
  ```

- [ ] 添加配置（如果需要）
  ```
  app.settings.service_role_key = YOUR_SERVICE_ROLE_KEY
  ```

  **注意**: 如果不配置，Trigger 會嘗試從環境變數讀取，失敗時會記錄警告但不影響功能

### 步驟 3: 部署前端和 API

- [ ] 確認代碼已提交到版本控制

- [ ] 部署 web-admin
  ```bash
  cd web-admin
  npm run build
  npm run start
  ```

- [ ] 驗證 API 端點可訪問
  ```bash
  # 測試狀態 API
  curl http://localhost:3001/api/admin/realtime-sync/status \
    -H "Authorization: Bearer YOUR_TOKEN"
  ```

### 步驟 4: 測試環境驗證

- [ ] 訪問派單設定頁面
  ```
  http://localhost:3001/settings/dispatch
  ```

- [ ] 確認頁面正常顯示

- [ ] 確認可以看到當前狀態（應該是停用狀態）

## 📊 監控檢查

### 性能指標

- [ ] 檢查平均延遲時間
  ```sql
  SELECT AVG(EXTRACT(EPOCH FROM (processed_at - created_at))) as avg_delay
  FROM outbox
  WHERE created_at >= NOW() - INTERVAL '1 hour'
    AND processed_at IS NOT NULL;
  ```
  
  預期: < 3 秒

- [ ] 檢查 HTTP 請求成功率
  ```sql
  SELECT 
    COUNT(*) FILTER (WHERE status_code = 200) * 100.0 / COUNT(*) as success_rate
  FROM net._http_response
  WHERE created >= NOW() - INTERVAL '1 hour';
  ```
  
  預期: > 95%

- [ ] 檢查待處理事件數
  ```sql
  SELECT COUNT(*) FROM outbox WHERE processed_at IS NULL;
  ```
  
  預期: < 10

### 錯誤檢查

- [ ] 檢查錯誤事件
  ```sql
  SELECT * FROM outbox 
  WHERE error_message IS NOT NULL 
  ORDER BY created_at DESC LIMIT 10;
  ```

- [ ] 檢查失敗的 HTTP 請求
  ```sql
  SELECT * FROM net._http_response 
  WHERE status_code != 200 OR status_code IS NULL
  ORDER BY created DESC LIMIT 10;
  ```
