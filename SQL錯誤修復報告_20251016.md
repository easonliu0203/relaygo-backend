### 錯誤 1: `tablename` 欄位不存在

### 錯誤 2: booking_number 超過長度限制

**文件**: `test_realtime_trigger_only.sql`  
**錯誤訊息**: `ERROR: 22001: value too long for type character varying(20)`  
**位置**: Line 39  
**原因**: 
- 生成的訂單編號：`'TEST_TRIGGER_' || timestamp`
- 長度計算：13 + 10 = 23 字符
- 欄位限制：VARCHAR(20)
- 超出：23 > 20 ❌

---

## ✅ 修復方案

### 修復 1: 更正欄位名稱

**文件**: `check_trigger_config.sql`

**修改前**:
```sql
SELECT 
  schemaname AS "Schema",
  tablename AS "表名",  -- ❌ 錯誤
  n_tup_ins AS "INSERT 次數",
  ...
FROM pg_stat_user_tables
WHERE tablename = 'bookings';  -- ❌ 錯誤
```

**修改後**:
```sql
SELECT 
  schemaname AS "Schema",
  relname AS "表名",  -- ✅ 正確
  n_tup_ins AS "INSERT 次數",
  ...
FROM pg_stat_user_tables
WHERE relname = 'bookings';  -- ✅ 正確
```

**說明**: 
- PostgreSQL 系統視圖 `pg_stat_user_tables` 使用 `relname` 欄位表示表名
- `tablename` 不是有效的欄位名稱

### 修復 2: 縮短訂單編號前綴

#### 文件 1: `test_realtime_trigger_only.sql`

**修改前**:
```sql
test_booking_number := 'TEST_TRIGGER_' || EXTRACT(EPOCH FROM NOW())::BIGINT;
-- 長度：13 + 10 = 23 字符 ❌
```

**修改後**:
```sql
-- 'TRG_' (4 字符) + timestamp (10 字符) = 14 字符 < 20 ✓
test_booking_number := 'TRG_' || EXTRACT(EPOCH FROM NOW())::BIGINT;
-- 長度：4 + 10 = 14 字符 ✅
```

---

**修改後**:

## 📝 技術細節

### PostgreSQL 系統視圖欄位

**pg_stat_user_tables** 常用欄位：
- `schemaname` - Schema 名稱
- `relname` - 表名稱（relation name）✅
- `n_tup_ins` - INSERT 次數
- `n_tup_upd` - UPDATE 次數
- `n_tup_del` - DELETE 次數
- `last_vacuum` - 最後 VACUUM 時間
- `last_autovacuum` - 最後 Auto VACUUM 時間

**注意**: 
- ❌ `tablename` 不是有效欄位
- ✅ `relname` 才是正確欄位

---

### VARCHAR 長度計算

**bookings.booking_number** 欄位定義：
```sql
booking_number VARCHAR(20) NOT NULL UNIQUE
```

**長度限制**: 最多 20 個字符

**Unix timestamp 長度**:
- 當前（2025）：10 個字符
- 2286 年後：11 個字符（會超限）

**安全的前綴長度**:
- 保守：≤ 8 個字符（留 2 個字符餘裕）
- 一般：≤ 10 個字符（剛好）

**我們的選擇**:
- `TRG_` = 4 字符（非常安全）
- `FLOW_` = 5 字符（非常安全）
- `RT_` = 3 字符（非常安全）
