### 步驟 2：執行簡化查詢

**複製以下 SQL 並執行**：

```sql
-- 查看最近的錯誤事件
SELECT 
  '最近的錯誤事件' as 檢查項目,
  id as 事件ID,
  payload->>'bookingNumber' as 訂單編號,
  error_message as 錯誤訊息,
  retry_count as 重試次數,
  created_at as 創建時間
FROM outbox
WHERE error_message IS NOT NULL
  AND created_at >= NOW() - INTERVAL '1 hour'
ORDER BY created_at DESC
LIMIT 5;
```

### 步驟 3：查看結果

**重點查看「錯誤訊息」欄位**，告訴我顯示什麼：

#### 可能的錯誤訊息 A：環境變數問題（最常見）
```
Cannot read property 'FIREBASE_PROJECT_ID' of undefined
或
FIREBASE_API_KEY is not defined
或
Deno.env.get('FIREBASE_PROJECT_ID') returned undefined
```

**如果看到這個** → 跳到「修復方案 A」

---

#### 可能的錯誤訊息 B：Firestore 權限問題
```
Firestore 更新失敗 (403): Forbidden
或
Permission denied
```

**如果看到這個** → 跳到「修復方案 B」

---

#### 可能的錯誤訊息 C：Project ID 錯誤
```
Firestore 更新失敗 (404): Not Found
或
Project not found
```

**如果看到這個** → 跳到「修復方案 C」

## 🔧 修復方案

### 修復方案 A：設置環境變數（最常見）

**如果錯誤訊息提到 FIREBASE_PROJECT_ID 或 FIREBASE_API_KEY**：

1. **打開環境變數設置頁面**
   ```
   https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/functions
   ```

2. **添加 4 個環境變數**：

   | 變數名稱 | 值 | 如何獲取 |
   |----------|-----|----------|
   | `FIREBASE_PROJECT_ID` | 您的 Project ID | Firebase Console → Settings → Project ID |
   | `FIREBASE_API_KEY` | 以 AIza 開頭的 key | Firebase Console → Settings → Web API Key |
   | `SUPABASE_URL` | `https://vlyhwegpvpnjyocqmfqc.supabase.co` | 固定值 |
   | `SUPABASE_SERVICE_ROLE_KEY` | JWT token | Supabase → Settings → API → service_role (Reveal) |

3. **保存**

4. **重新測試**
   - 返回 Functions 頁面
   - 點擊 sync-to-firestore → Invoke
   - 應該看到 `success: 7, failure: 0`

**詳細步驟**：請參考 `立即修復腳本.md`

---

### 修復方案 B：檢查 Firestore 權限

**如果錯誤訊息提到 403 或 Forbidden**：

1. **檢查 Firebase API Key**
   - 確認使用的是 Web API Key（不是 Server Key）
   - 重新獲取並更新環境變數

2. **檢查 Firestore 安全規則**
   - 打開 Firebase Console → Firestore → Rules
   - 確認允許寫入 `orders_rt` 集合

---

### 修復方案 C：檢查 Project ID

**如果錯誤訊息提到 404 或 Not Found**：

1. **重新獲取 Firebase Project ID**
   - Firebase Console → Project Settings → General
   - 複製正確的 Project ID

2. **更新環境變數**
   - 在 Supabase Dashboard 更新 `FIREBASE_PROJECT_ID`
