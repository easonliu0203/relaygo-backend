### 問題
- ❌ 客戶端和司機端聊天室列表載入失敗

### 修復
- ✅ 添加 `chat_rooms` collection 的兩個複合索引
- ✅ 索引 1: `customerId` + `lastMessageTime`
- ✅ 索引 2: `driverId` + `lastMessageTime`
- ✅ 部署索引到 Firebase

## ⏰ 重要提醒

### 索引創建需要時間

**Firestore 複合索引創建通常需要幾分鐘時間**

**在開始測試之前，請先確認索引已經創建完成**:

1. 前往 Firebase Console: https://console.firebase.google.com/project/ride-platform-f1676/firestore/indexes

2. 查看索引狀態：
   - 🟡 **Building**: 索引正在創建中 → **請等待**
   - 🟢 **Enabled**: 索引已啟用 → **可以開始測試**

3. 確認以下兩個索引都是 **Enabled** 狀態：
   - `chat_rooms`: `customerId` (ASC) + `lastMessageTime` (DESC)
   - `chat_rooms`: `driverId` (ASC) + `lastMessageTime` (DESC)

**預計等待時間**:
- 小型資料庫（< 1000 筆資料）: 1-5 分鐘
- 中型資料庫（1000-10000 筆資料）: 5-15 分鐘
- 大型資料庫（> 10000 筆資料）: 15 分鐘以上

## 🔍 常見問題排查

### 問題 1: 仍然出現索引錯誤

**可能原因**:
1. 索引還在創建中（Building 狀態）
2. 索引創建失敗
3. 索引配置錯誤

**解決方案**:
```bash
# 1. 檢查 Firebase Console 確認索引狀態
https://console.firebase.google.com/project/ride-platform-f1676/firestore/indexes

# 2. 如果索引仍在 Building，請等待

# 3. 如果索引創建失敗，重新部署
cd firebase
firebase deploy --only firestore:indexes

# 4. 檢查索引配置
cat firestore.indexes.json
```

### 問題 2: 聊天室列表為空

**可能原因**:
1. 沒有創建訂單
2. 訂單還沒有分配司機
3. Firestore 同步延遲

**解決方案**:
1. 創建一個新訂單並分配司機
2. 等待幾秒鐘讓 Edge Function 同步資料
3. 下拉刷新聊天室列表

### 問題 3: 聊天室排序不正確

**可能原因**:
1. 索引排序方向錯誤
2. `lastMessageTime` 欄位為 null
3. 客戶端排序邏輯錯誤

**解決方案**:
1. 檢查索引配置（應該是 DESCENDING）
2. 發送訊息更新 `lastMessageTime`
3. 檢查 ChatService 的排序邏輯

### 問題 4: 查詢速度慢

**可能原因**:
1. 索引還沒有完全生效
2. 資料量太大
3. 網路連接問題

**解決方案**:
1. 等待索引完全生效（可能需要幾分鐘）
2. 檢查 Firebase Console 的索引狀態
3. 檢查模擬器網路連接
