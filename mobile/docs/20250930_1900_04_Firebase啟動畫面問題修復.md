# Firebase 啟動畫面問題修復

**日期**: 2025-09-30 19:00  
**編號**: 04  
**主旨**: Firebase 啟動畫面問題修復

## 📋 **問題描述**

Relay GO 客戶端和司機端應用程式在 Android 14 模擬器上成功部署後，遇到以下問題：

1. **應用程式卡在啟動畫面**: 顯示 Flutter Logo，無法進入登入頁面
2. **Firebase 初始化失敗**: `enablePersistence()` 方法在 Android 平台不支援
3. **認證測試失敗**: 測試帳號認證失敗，提示憑證錯誤

## 🔍 **問題診斷過程**

### **1. 初始症狀**
- 應用程式啟動後停留在 Flutter 預設啟動畫面
- 日誌顯示 Firebase 初始化錯誤：
  ```
  UnimplementedError: enablePersistence() is only available for Web. 
  Use [Settings.persistenceEnabled] for other platforms.
  ```
- 應用程式在 Firebase 初始化失敗後無法繼續執行

### **2. 根本原因分析**
通過分析日誌和代碼，發現問題的根本原因：

1. **Firebase Firestore 配置錯誤**: 
   - 使用了 Web 專用的 `enablePersistence()` 方法
   - 在 Android 平台上應該使用 `Settings.persistenceEnabled`

2. **錯誤處理不當**:
   - Firebase 初始化失敗時拋出異常，阻止應用程式繼續執行
   - `main()` 函數沒有適當的錯誤處理機制

3. **測試帳號不存在**:
   - Firebase Authentication 中沒有創建測試帳號
   - 應用程式嘗試登入不存在的測試帳號

## 🔧 **修復方案**

### **方案一：修復 Firebase Firestore 配置**

**修改前**:
```dart
// 設定 Firestore 離線持久化
if (!kIsWeb) {
  await _firestore.enablePersistence(); // ❌ 錯誤：Android 不支援
}
```

**修改後**:
```dart
// 設定 Firestore 離線持久化
try {
  _firestore.settings = const Settings(
    persistenceEnabled: true,
    cacheSizeBytes: Settings.CACHE_SIZE_UNLIMITED,
  );
  _logger.i('Firestore 本地快取已啟用');
} catch (e) {
  _logger.w('Firestore 本地快取設定失敗', error: e);
  // 即使快取設定失敗，也不應該阻止應用程式啟動
}
```

### **方案二：改善錯誤處理**

**修改前**:
```dart
void main() async {
  // 初始化 Firebase 服務
  await FirebaseService().initialize(); // ❌ 異常會阻止應用程式啟動
  
  runApp(const ProviderScope(child: CustomerApp()));
}
```

**修改後**:
```dart
void main() async {
  try {
    // 初始化 Firebase 服務
    await FirebaseService().initialize();
  } catch (e) {
    // 記錄初始化錯誤，但不阻止應用程式啟動
    print('初始化過程中發生錯誤: $e');
  }
  
  runApp(const ProviderScope(child: CustomerApp()));
}
```

### **方案三：自動創建測試帳號**

```dart
/// 使用測試帳號登入（如果不存在則自動創建）
Future<UserCredential?> signInWithTestAccount(String accountType) async {
  try {
    // 嘗試登入
    return await signInWithEmailAndPassword(
      email: testAccount.email,
      password: testAccount.password,
    );
  } on AuthException catch (e) {
    if (e.message.contains('用戶不存在') || e.message.contains('user-not-found')) {
      // 測試帳號不存在，自動創建
      final userCredential = await _auth.createUserWithEmailAndPassword(
        email: testAccount.email,
        password: testAccount.password,
      );
      
      await userCredential.user?.updateDisplayName(testAccount.displayName);
      return userCredential;
    } else {
      rethrow;
    }
  }
}
```

## 💡 **開發心得**

### **技術教訓**
1. **平台差異性**: Firebase 的某些 API 在不同平台上有不同的實作方式
2. **錯誤處理**: 關鍵服務的初始化失敗不應該阻止整個應用程式啟動
3. **測試環境**: 開發環境需要完整的測試數據支援

### **最佳實踐**
1. **漸進式初始化**: 將非關鍵服務的初始化設為非阻塞
2. **平台適配**: 根據平台特性選擇合適的 API
3. **自動化測試**: 實作自動創建測試數據的機制

### **遇到的困難**
1. **文檔不完整**: Firebase 文檔沒有清楚說明平台差異
2. **錯誤訊息模糊**: 初始錯誤訊息沒有指向具體的解決方案
3. **多平台複雜性**: Flutter 多平台開發需要考慮各平台的特殊性
