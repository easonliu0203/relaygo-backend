# 訂單顯示測試指南

## 📋 測試目標

驗證支付成功後訂單能夠正確顯示在「預約成功」頁面和「我的訂單」頁面。

## 🔧 測試前準備

### 1. 確認後端服務運行

```bash
# 進入 web-admin 目錄
cd web-admin

# 安裝依賴（首次運行）
npm install

# 啟動開發服務器
npm run dev

# 確認看到以下訊息
# ✓ Ready on http://localhost:3001
```

### 2. 確認代碼已更新

```bash
# 檢查同步方法是否存在
grep -n "_syncBookingToFirestore" mobile/lib/core/services/booking_service.dart
grep -n "_syncPaymentStatusToFirestore" mobile/lib/core/services/booking_service.dart

# 應該看到類似輸出
# 120:  Future<void> _syncBookingToFirestore(
# 195:  Future<void> _syncPaymentStatusToFirestore(
```

### 3. 重新建置應用

```bash
# 進入 mobile 目錄
cd mobile

# 清理舊的建置
flutter clean

# 獲取依賴
flutter pub get

# 運行應用（Android 模擬器）
flutter run --flavor customer --target lib/apps/customer/main_customer.dart
```

### 4. 準備測試工具

- **Android 模擬器**：已啟動並運行
- **Firebase Console**：打開 https://console.firebase.google.com
- **開發者工具**：Android Studio 或 VS Code 的控制台

## 🧪 測試場景

### 測試場景 1：創建訂單並驗證同步

#### 測試步驟

1. **啟動應用並登入**
   - 打開應用
   - 使用測試帳號登入
   - 進入首頁

2. **選擇車型套餐**
   - 點擊「選擇方案」或「立即預約」
   - 等待套餐列表載入（1-5 秒）
   - 選擇任一套餐（例如：豪華轎車）

3. **填寫預約資訊**
   - 上車地點：台北車站
   - 下車地點：台北101
   - 預約時間：選擇未來的時間
   - 乘客人數：2 人
   - 行李數量：1 件
   - 備註：測試訂單

4. **創建訂單**
   - 點擊「確認預約」
   - 等待訂單創建完成

#### 預期結果

**應用顯示**：
- ✅ 顯示「訂單創建成功」訊息
- ✅ 跳轉到支付頁面
- ✅ 顯示訂單編號（例如：mock_booking_1704117600000）
- ✅ 顯示訂金金額

**控制台日誌**：
```
[BookingService] 開始創建訂單
[BookingService] API 返回訂單 ID: mock_booking_1704117600000
[BookingService] 同步訂單到 Firebase: mock_booking_1704117600000
[BookingService] 訂單已同步到 Firebase
```

**Firebase Console 驗證**：
1. 打開 Firebase Console
2. 進入 Firestore Database
3. 找到 `bookings` 集合
4. ✅ 應該看到新創建的訂單文檔
5. ✅ 文檔 ID 與控制台日誌中的訂單 ID 一致
6. ✅ 訂單資料完整（customerId, pickupAddress, status 等）

#### 故障排除

**問題**：控制台沒有顯示同步日誌

**可能原因**：
- 代碼沒有重新編譯
- 使用了舊版本的應用

**解決方法**：
```bash
flutter clean
flutter pub get
flutter run --flavor customer --target lib/apps/customer/main_customer.dart
```

**問題**：Firebase 中沒有訂單

**可能原因**：
- 同步失敗
- Firebase 權限問題

**解決方法**：
- 查看控制台是否有錯誤日誌
- 檢查 Firebase 規則是否允許寫入
- 確認用戶已登入

### 測試場景 2：支付訂金並驗證狀態更新

#### 測試步驟

1. **在支付頁面**
   - 確認訂單資訊正確
   - 確認訂金金額正確

2. **選擇支付方式**
   - 選擇「信用卡」或其他支付方式
   - 點擊「確認支付」

3. **等待支付完成**
   - 顯示支付處理中（約 5 秒）
   - 等待支付成功訊息

#### 預期結果

**應用顯示**：
- ✅ 顯示「支付成功」訊息
- ✅ 自動跳轉到「預約成功」頁面
- ✅ 不再顯示「訂單不存在」錯誤

**控制台日誌**：
```
[BookingService] 開始支付訂金: mock_booking_1704117600000
[BookingService] 支付成功，同步到 Firebase
[BookingService] 更新 Firebase 訂單狀態: mock_booking_1704117600000
[BookingService] Firebase 訂單狀態已更新
```

**Firebase Console 驗證**：
1. 刷新 Firebase Console
2. 找到剛才的訂單文檔
3. ✅ `depositPaid` 欄位已更新為 `true`
4. ✅ `status` 欄位已更新為 `matched`

#### 故障排除

**問題**：支付後仍然顯示「訂單不存在」

**可能原因**：
- 訂單 ID 傳遞錯誤
- Firebase 中沒有該訂單

**解決方法**：
- 檢查控制台日誌中的訂單 ID
- 在 Firebase Console 中搜尋該 ID
- 確認訂單是否存在

**問題**：支付成功但狀態沒有更新

**可能原因**：
- 狀態同步失敗
- Firebase 更新權限問題

**解決方法**：
- 查看控制台是否有錯誤日誌
- 檢查 Firebase 規則是否允許更新
- 手動在 Firebase Console 中更新狀態測試

### 測試場景 3：查看預約成功頁面

#### 測試步驟

1. **支付成功後**
   - 自動跳轉到「預約成功」頁面
   - 或手動點擊「查看訂單」

2. **檢查頁面內容**
   - 查看訂單編號
   - 查看訂單狀態
   - 查看訂單詳情

#### 預期結果

**頁面顯示**：
- ✅ 顯示「預約成功」標題
- ✅ 顯示訂單編號（例如：mock_booking_1704117600000）
- ✅ 顯示訂單狀態：已配對（藍色標籤）
- ✅ 顯示上車地點：台北車站
- ✅ 顯示下車地點：台北101
- ✅ 顯示預約時間：2025-01-02 10:00
- ✅ 顯示乘客人數：2 人
- ✅ 顯示行李數量：1 件
- ✅ 顯示訂金金額：已支付
- ✅ 顯示總金額

**不應該出現**：
- ❌ 「訂單不存在」錯誤訊息
- ❌ 空白頁面
- ❌ 載入中狀態持續超過 2 秒

#### 故障排除

**問題**：仍然顯示「訂單不存在」

**診斷步驟**：
1. 檢查控制台日誌
2. 確認訂單 ID 是否正確
3. 在 Firebase Console 中搜尋該訂單
4. 確認訂單是否存在

**解決方法**：
- 如果訂單不存在，重新測試場景 1
- 如果訂單存在但查詢不到，檢查查詢邏輯
- 確認用戶 ID 是否匹配

**問題**：頁面空白或載入中

**可能原因**：
- 網路連接問題
- Firebase 查詢失敗

**解決方法**：
- 檢查網路連接
- 查看控制台錯誤日誌
- 重新啟動應用

### 測試場景 4：查看我的訂單頁面

#### 測試步驟

1. **返回首頁**
   - 點擊返回按鈕或導航到首頁

2. **進入我的訂單**
   - 點擊「我的訂單」或個人中心
   - 進入訂單列表頁面

3. **查看進行中訂單**
   - 點擊「進行中」分頁
   - 查看訂單列表

#### 預期結果

**頁面顯示**：
- ✅ 「進行中」分頁顯示 1 個訂單
- ✅ 訂單卡片顯示訂單編號
- ✅ 訂單卡片顯示訂單狀態：已配對（藍色）
- ✅ 訂單卡片顯示上車地點和下車地點
- ✅ 訂單卡片顯示預約時間
- ✅ 可以點擊訂單查看詳情

**不應該出現**：
- ❌ 空白列表
- ❌ 「暫無訂單」訊息
- ❌ 載入中狀態持續超過 2 秒

#### 故障排除

**問題**：訂單列表為空

**診斷步驟**：
1. 檢查控制台日誌
2. 確認訂單狀態是否為 `matched`
3. 在 Firebase Console 中查看訂單
4. 確認 `customerId` 是否與當前用戶匹配

**解決方法**：
- 如果訂單狀態不對，重新測試場景 2
- 如果 customerId 不匹配，檢查創建訂單時的用戶 ID
- 嘗試登出後重新登入

**問題**：訂單顯示但狀態不正確

**可能原因**：
- 狀態沒有正確更新
- 狀態映射錯誤

**解決方法**：
- 在 Firebase Console 中檢查訂單的 `status` 欄位
- 確認是否為 `matched`
- 如果不是，手動更新測試

## 🐛 常見問題

### Q1：為什麼控制台沒有顯示同步日誌？

**A**：可能是以下原因：
1. 代碼沒有重新編譯 → 執行 `flutter clean && flutter pub get && flutter run`
2. 使用了舊版本的應用 → 完全關閉應用後重新運行
3. 日誌級別設置問題 → 確認 `debugPrint` 可以正常輸出

### Q2：為什麼 Firebase 中沒有訂單？

**A**：可能是以下原因：
1. 同步失敗 → 查看控制台是否有錯誤日誌
2. Firebase 權限問題 → 檢查 Firestore 規則
3. 網路連接問題 → 確認模擬器可以訪問網路
4. 用戶未登入 → 確認用戶已成功登入

### Q3：為什麼仍然顯示「訂單不存在」？

**A**：可能是以下原因：
1. 訂單 ID 不匹配 → 檢查日誌中的訂單 ID
2. Firebase 中沒有該訂單 → 參考 Q2
3. 查詢邏輯錯誤 → 確認代碼已更新
4. 用戶 ID 不匹配 → 檢查 customerId 欄位

### Q4：為什麼訂單狀態沒有更新？

**A**：可能是以下原因：
1. 支付狀態同步失敗 → 查看控制台錯誤日誌
2. Firebase 更新權限問題 → 檢查 Firestore 規則
3. 訂單 ID 錯誤 → 確認支付時使用的訂單 ID

### Q5：如何清空測試資料？

**A**：
1. 在 Firebase Console 中手動刪除測試訂單
2. 或者使用不同的測試帳號
3. 或者在代碼中添加刪除功能（開發環境）
