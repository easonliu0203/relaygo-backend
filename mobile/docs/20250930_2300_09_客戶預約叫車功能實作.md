## 🏗️ **技術架構設計**

### **資料模型架構**
```dart
// 訂單狀態枚舉
enum BookingStatus {
  pending,    // 待配對
  matched,    // 已配對
  inProgress, // 進行中
  completed,  // 已完成
  cancelled,  // 已取消
}

// 地理位置模型
@freezed
class LocationPoint with _$LocationPoint {
  const factory LocationPoint({
    required double latitude,
    required double longitude,
  }) = _LocationPoint;
}

// 完整訂單模型
@freezed
class BookingOrder with _$BookingOrder {
  const factory BookingOrder({
    required String id,
    required String customerId,
    String? driverId,
    required String pickupAddress,
    required LocationPoint pickupLocation,
    required String dropoffAddress,
    required LocationPoint dropoffLocation,
    required DateTime bookingTime,
    required int passengerCount,
    int? luggageCount,
    String? notes,
    required double estimatedFare,
    required double depositAmount,
    @Default(false) bool depositPaid,
    @Default(BookingStatus.pending) BookingStatus status,
    required DateTime createdAt,
    DateTime? matchedAt,
    DateTime? completedAt,
  }) = _BookingOrder;
}
```

## 💾 **Firestore 資料結構**

### **bookings 集合結構**
```javascript
bookings/{bookingId} {
  id: string,                    // 訂單編號
  customerId: string,            // 客戶 ID
  driverId: string?,             // 司機 ID（配對後）
  pickupAddress: string,         // 上車地點
  pickupLocation: GeoPoint,      // 上車座標
  dropoffAddress: string,        // 下車地點
  dropoffLocation: GeoPoint,     // 下車座標
  bookingTime: Timestamp,        // 預約時間
  passengerCount: number,        // 乘客人數
  luggageCount: number?,         // 行李數量
  notes: string?,                // 備註
  estimatedFare: number,         // 預估費用
  depositAmount: number,         // 訂金金額
  depositPaid: boolean,          // 訂金是否已支付
  status: string,                // 訂單狀態
  createdAt: Timestamp,          // 建立時間
  matchedAt: Timestamp?,         // 配對時間
  completedAt: Timestamp?,       // 完成時間
}
```
