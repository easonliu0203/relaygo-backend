## 📋 **問題概述**

在完成 Relay GO 客戶端和司機端應用程式的基礎架構後，遇到了多個 Android 建置問題。本文檔記錄了完整的問題診斷過程、根本原因分析和修復方案。

## 🔍 **問題診斷過程**

### **階段一：初始建置失敗**

#### **問題現象**
```bash
flutter build apk --flavor customer --target lib/apps/customer/main_customer.dart --debug
```
執行後出現多個編譯錯誤：

1. **Gradle Kotlin DSL 語法錯誤**
2. **AndroidManifest.xml 屬性衝突**
3. **插件兼容性問題**
4. **Kotlin 增量編譯快取錯誤**

### **階段二：逐步問題分析**

#### **1. Gradle Build 配置問題**

**錯誤訊息**:
```
Val cannot be reassigned
Unresolved reference: Properties
Unresolved reference: File
```

**根本原因**:
- `build.gradle.kts` 使用了 Groovy 風格的函數語法
- 缺少必要的 import 語句
- 簽名配置位置不正確

**診斷過程**:
1. 檢查 `build.gradle.kts` 語法
2. 對比 Kotlin DSL 官方文檔
3. 分析錯誤訊息中的行號和上下文

#### **2. AndroidManifest.xml 衝突**

**錯誤訊息**:
```
Attribute android:label value=(Relay GO) from AndroidManifest.xml is also present at AndroidManifest.xml
Attribute android:orientation value=(portrait) from AndroidManifest.xml is also present at AndroidManifest.xml
```

**根本原因**:
- 主 manifest 和 flavor-specific manifest 中重複定義了相同屬性
- Android Manifest 合併規則理解不足

**診斷過程**:
1. 檢查所有 AndroidManifest.xml 檔案
2. 分析 manifest 合併日誌
3. 查閱 Android 官方文檔關於 manifest 合併規則

#### **3. 插件兼容性問題**

**錯誤訊息**:
```
Unresolved reference 'Registrar'
file_picker requires Android NDK 27.0.12077973
pay_android compilation failed
sign_in_with_apple compilation failed
```

**根本原因**:
- 多個插件仍使用已棄用的 Flutter v1 embedding API
- NDK 版本不匹配
- 插件版本與當前 Flutter 版本不兼容

**診斷過程**:
1. 分析每個插件的錯誤訊息
2. 檢查插件的 GitHub issues
3. 查看插件的最新版本和兼容性說明

#### **4. Kotlin 增量編譯快取問題**

**錯誤訊息**:
```
Storage for [...] is already registered
Could not close incremental caches
java.lang.AssertionError: java.lang.Exception
```

**根本原因**:
- 多次失敗建置導致增量編譯快取損壞
- Kotlin 編譯器無法正確管理快取檔案

**診斷過程**:
1. 分析堆疊追蹤訊息
2. 檢查 build 目錄中的快取檔案
3. 嘗試不同的清理策略

## 🔧 **修復方案**

### **1. Gradle Build 配置修復**

#### **修復前**:
```kotlin
fun getKeystoreProperties(flavorName) {
    val keystorePropertiesFile = rootProject.file("app/keystore/${flavorName}/key.properties")
    // ...
}

applicationVariants.all { variant ->
    if (variant.buildType.name == "release") {
        variant.signingConfig = signingConfigs.getByName("${variant.flavorName}Release")
    }
}
```

#### **修復後**:
```kotlin
import java.util.Properties
import java.io.File

fun getKeystoreProperties(flavorName: String): Properties {
    val keystorePropertiesFile = rootProject.file("app/keystore/${flavorName}/key.properties")
    val keystoreProperties = Properties()
    if (keystorePropertiesFile.exists()) {
        keystoreProperties.load(keystorePropertiesFile.inputStream())
    }
    return keystoreProperties
}

productFlavors {
    create("customer") {
        // ...
        signingConfig = signingConfigs.getByName("customerRelease")
    }
    create("driver") {
        // ...
        signingConfig = signingConfigs.getByName("driverRelease")
    }
}
```

**修復要點**:
- 添加正確的 import 語句
- 使用 Kotlin 類型註解
- 將簽名配置移動到 productFlavors 中

### **2. AndroidManifest.xml 衝突修復**

#### **修復前**:
```xml
<!-- android/app/src/main/AndroidManifest.xml -->
<application android:label="Relay GO">

<!-- android/app/src/customer/AndroidManifest.xml -->
<application android:label="Relay GO">
```

#### **修復後**:
```xml
<!-- android/app/src/main/AndroidManifest.xml -->
<application 
    android:label="@string/app_name"
    tools:replace="android:label">

<!-- android/app/src/customer/AndroidManifest.xml -->
<application>
    <!-- 移除 android:label 屬性 -->
```

**修復要點**:
- 使用 `tools:replace` 指令解決衝突
- 統一使用字串資源引用
- 移除重複的屬性定義

### **3. 插件兼容性問題修復**

### **4. Kotlin 快取問題修復**

#### **修復步驟**:
```bash
# 1. 清理 Flutter 建置快取
flutter clean

# 2. 重新獲取依賴
flutter pub get

# 3. 重新建置
flutter build apk --flavor customer --target lib/apps/customer/main_customer.dart --debug
```

**修復要點**:
- 完全清理建置快取
- 重新初始化依賴關係
- 避免增量編譯的快取問題

## 💡 **經驗總結**

### **成功經驗**
1. **系統性診斷**: 逐步分析每個錯誤，避免同時處理多個問題
2. **文檔參考**: 仔細閱讀官方文檔，理解正確的語法和配置
3. **版本管理**: 暫時移除有問題的依賴，確保核心功能先能運作
4. **清理策略**: `flutter clean` 是解決快取問題的有效方法

### **遇到的困難**
1. **錯誤訊息理解**: Kotlin 編譯錯誤訊息有時不夠直觀
2. **插件生態**: Flutter 插件版本兼容性是常見問題
3. **配置複雜度**: 多應用程式配置增加了問題診斷的複雜度

### **解決方法**
1. **分階段修復**: 先修復語法錯誤，再處理兼容性問題
2. **暫時性方案**: 對於非核心功能，可以暫時停用等待修復
3. **測試驅動**: 每次修改後立即測試，及早發現問題

### **犯過的錯誤**
1. **同時修改多個配置**: 導致難以定位問題根源
2. **忽略警告訊息**: 一些警告可能是後續錯誤的前兆
3. **過度依賴增量編譯**: 忽略了清理快取的重要性
