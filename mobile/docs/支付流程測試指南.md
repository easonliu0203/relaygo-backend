# 支付流程測試指南

## 🎯 測試目的

驗證 Android 模擬器上的支付流程是否正常工作，確保網路連接問題已經修復。

## 📋 前置準備

### 1. 確認後端服務運行

```bash
# 進入 web-admin 目錄
cd web-admin

# 啟動開發服務器
npm run dev

# 確認服務啟動成功
# 應該看到：✓ Ready on http://localhost:3001
```

### 2. 測試後端 API

```bash
# 測試健康檢查端點
curl http://localhost:3001/api/health

# 預期回應：
# {"status":"ok","timestamp":"2025-01-01T17:00:00.000Z"}

# 測試價格配置端點
curl http://localhost:3001/api/pricing/packages

# 預期回應：
# {"success":true,"data":[...]}
```

### 3. 確認代碼已更新

檢查以下文件是否已正確配置：

**`mobile/lib/core/services/pricing_service.dart`** (第 55-57 行)：
```dart
// 管理後台 API 基礎 URL
// Android 模擬器使用 10.0.2.2 訪問主機的 localhost
static const String _baseUrl = 'http://10.0.2.2:3001/api';
```

**`mobile/lib/core/services/booking_service.dart`** (第 18-20 行)：
```dart
// 管理後台 API 基礎 URL
// Android 模擬器使用 10.0.2.2 訪問主機的 localhost
static const String _baseUrl = 'http://10.0.2.2:3001/api';
```

## 🚀 測試步驟

### 步驟 1：啟動 Android 模擬器

```bash
# 列出可用的模擬器
flutter emulators

# 啟動模擬器（選擇一個）
flutter emulators --launch <emulator_id>

# 或直接從 Android Studio 啟動
```

### 步驟 2：重新建置並運行應用

```bash
# 進入 mobile 目錄
cd mobile

# 清理舊的建置
flutter clean

# 獲取依賴
flutter pub get

# 運行客戶端應用
flutter run --flavor customer --target lib/apps/customer/main_customer.dart

# 或使用腳本
scripts\run-customer.bat
```

### 步驟 3：登入應用

1. 等待應用啟動完成
2. 使用測試帳號登入：
   - 帳號：`customer@test.com`
   - 密碼：`test123456`

### 步驟 4：測試價格查詢

1. 進入預約頁面
2. 選擇乘客人數
3. 觀察是否正確顯示車型套餐
4. 檢查價格是否正確

**預期結果**：
- ✅ 成功顯示車型套餐列表
- ✅ 價格資訊正確（3-4人座 6小時 $40，8小時 $50）
- ✅ 沒有網路錯誤訊息

### 步驟 5：創建預約訂單

1. 選擇一個車型套餐
2. 填寫預約資訊：
   - 上車地點：台北車站
   - 下車地點：台北101
   - 預約時間：選擇未來的日期和時間
   - 乘客人數：2人
3. 點擊「確認預約」

**預期結果**：
- ✅ 成功創建訂單
- ✅ 顯示訂單詳情
- ✅ 訂單狀態為「待支付」
- ✅ 顯示訂金金額

### 步驟 6：支付訂金

1. 在訂單詳情頁面
2. 點擊「支付訂金」按鈕
3. 選擇支付方式（模擬支付）
4. 確認支付

**預期結果**：
- ✅ 顯示支付處理中
- ✅ 5秒後支付完成
- ✅ 訂單狀態更新為「已支付訂金」
- ✅ 沒有「支付失敗」錯誤

### 步驟 7：查看訂單列表

1. 返回首頁
2. 進入「我的訂單」
3. 查看訂單列表

**預期結果**：
- ✅ 顯示剛才創建的訂單
- ✅ 訂單狀態正確
- ✅ 訂單資訊完整

## ❌ 常見問題排除

### 問題 1：仍然出現 Connection refused

**可能原因**：
- 代碼沒有重新編譯
- 使用了舊的 APK

**解決方法**：
```bash
# 完全清理並重新建置
flutter clean
flutter pub get
flutter run --flavor customer --target lib/apps/customer/main_customer.dart
```

### 問題 2：後端服務無法訪問

**可能原因**：
- 後端服務沒有運行
- 端口被佔用

**解決方法**：
```bash
# 檢查端口是否被佔用
netstat -ano | findstr :3001

# 如果被佔用，結束進程或更換端口
# 重新啟動後端服務
cd web-admin
npm run dev
```

### 問題 3：防火牆阻擋連接

**可能原因**：
- Windows 防火牆阻擋 3001 端口

**解決方法**：
```powershell
# 以管理員身份運行 PowerShell
# 添加防火牆規則
netsh advfirewall firewall add rule name="Web Admin" dir=in action=allow protocol=TCP localport=3001
```

### 問題 4：模擬器網路問題

**可能原因**：
- 模擬器網路設定錯誤
- 模擬器無法訪問外網

**解決方法**：
```bash
# 重啟模擬器
adb reboot

# 測試模擬器網路
adb shell ping 10.0.2.2
adb shell curl http://10.0.2.2:3001/api/health
```
