# 測試帳號認證問題診斷與修復

**日期**: 2025-09-30 20:00  
**編號**: 06  
**主旨**: 測試帳號認證問題診斷與修復

## 📋 **問題描述**

Relay GO 應用程式的測試帳號無法成功登入，系統顯示認證錯誤，導致無法進入主程式進行後續功能測試。

### **測試帳號資訊**
- **客戶端**: `customer.test@relaygo.com` / `RelayGO2024!Customer`
- **司機端**: `driver.test@relaygo.com` / `RelayGO2024!Driver`

## 🔍 **問題診斷過程**

### **階段一：初始錯誤分析**
**症狀**: 測試帳號登入失敗，顯示 `AuthException: 認證憑證無效`

**初步假設**: 
1. 測試帳號不存在於 Firebase Authentication
2. 自動創建測試帳號功能未正常運作
3. 密碼或 Email 格式錯誤

### **階段二：代碼邏輯檢查**
**發現問題**: `signInWithTestAccount` 方法中的錯誤處理邏輯有缺陷

**根本原因**:
```dart
// 問題代碼
} on AuthException catch (e) {
  if (e.message.contains('用戶不存在') || e.message.contains('user-not-found')) {
    // 自動創建邏輯
  }
}
```

**問題分析**:
1. `_handleAuthException` 將 Firebase 錯誤轉換為中文訊息
2. 實際錯誤訊息是 "找不到此電子郵件對應的帳號"
3. 條件檢查不匹配，自動創建邏輯永不觸發

### **階段三：修復實作**
**解決方案**: 直接捕獲 `FirebaseAuthException` 並檢查錯誤代碼

```dart
// 修復後的代碼
} on FirebaseAuthException catch (e) {
  if (e.code == 'user-not-found') {
    // 自動創建測試帳號
    final userCredential = await _auth.createUserWithEmailAndPassword(
      email: testAccount.email,
      password: testAccount.password,
    );
    await userCredential.user?.updateDisplayName(testAccount.displayName);
    return userCredential;
  }
}
```

### **階段四：登入流程修復**
**發現問題**: 登入頁面使用普通登入方法，未調用測試帳號專用方法

**解決方案**: 修改登入邏輯，檢測測試帳號並調用正確方法

```dart
// 檢查是否為測試帳號
final isTestAccount = testAccount != null && 
                     email == testAccount.email && 
                     password == testAccount.password;

if (isTestAccount) {
  await ref.read(authStateProvider.notifier).signInWithTestAccount(widget.appType);
} else {
  await ref.read(authStateProvider.notifier).signInWithEmailAndPassword(email, password);
}
```

### **階段五：Firebase 安全限制發現**
**最終問題**: Firebase 阻止模擬器請求

**錯誤訊息**: 
```
We have blocked all requests from this device due to unusual activity. Try again later.
```

**根本原因**: Firebase 的安全機制檢測到來自模擬器的頻繁測試請求，觸發了自動封鎖

## 🔧 **修復方案**

### **方案一：Firebase Console 手動創建測試帳號**

1. **登入 Firebase Console**
   - 前往 [Firebase Console](https://console.firebase.google.com/)
   - 選擇專案：`ride-platform-f1676`

2. **進入 Authentication 設定**
   - 點擊左側選單 "Authentication"
   - 點擊 "Users" 分頁

3. **創建測試帳號**
   ```
   客戶端測試帳號:
   Email: customer.test@relaygo.com
   Password: RelayGO2024!Customer
   
   司機端測試帳號:
   Email: driver.test@relaygo.com
   Password: RelayGO2024!Driver
   ```

### **方案二：調整 Firebase 安全設定**

1. **進入 Authentication 設定**
   - 點擊 "Settings" 分頁
   - 選擇 "Authorized domains"

2. **添加測試域名**
   - 添加 `localhost` 和模擬器域名
   - 暫時降低安全等級（僅開發環境）

### **方案三：等待自動解除封鎖**
- Firebase 通常會在 1-24 小時內自動解除封鎖
- 避免頻繁重複測試請求

### **方案四：重置測試環境**
```bash
# 清理 Flutter 快取
flutter clean
flutter pub get

# 重新建置應用程式
flutter build apk --flavor customer

# 使用新的模擬器實例
```
