### **2. 登出確認對話框**

```dart
void _showSignOutDialog(BuildContext context, WidgetRef ref) {
  showDialog(
    context: context,
    builder: (BuildContext context) {
      return AlertDialog(
        title: const Text('確認登出'),
        content: const Text('您確定要登出帳號嗎？'),
        actions: [
          TextButton(
            onPressed: () => Navigator.of(context).pop(),
            child: const Text('取消'),
          ),
          ElevatedButton(
            onPressed: () async {
              Navigator.of(context).pop();
              try {
                await ref.read(authStateProvider.notifier).signOut();
                // 顯示成功訊息
              } catch (e) {
                // 顯示錯誤訊息
              }
            },
            style: ElevatedButton.styleFrom(backgroundColor: Colors.red),
            child: const Text('確認登出'),
          ),
        ],
      );
    },
  );
}
```

### **3. 認證狀態管理**

#### **AuthService 登出方法**（已存在）
```dart
Future<void> signOut() async {
  try {
    // Google 登出
    if (await _googleSignIn.isSignedIn()) {
      await _googleSignIn.signOut();
    }
    
    // Firebase 登出
    await _auth.signOut();
  } catch (e) {
    throw AuthException('登出失敗：$e');
  }
}
```

#### **AuthStateNotifier 登出方法**（已存在）
```dart
Future<void> signOut() async {
  state = const AuthState.loading();
  
  try {
    await _authService.signOut();
    // 狀態會通過 authStateChanges 自動更新
  } catch (e) {
    state = AuthState.error(e.toString());
  }
}
```

### **成功經驗**
1. **架構重用**: 現有的 AuthService 和 AuthStateNotifier 完全滿足需求
2. **狀態管理**: Riverpod 的響應式設計讓登出狀態管理非常簡潔
3. **UI 一致性**: 通過主題色彩區分客戶端和司機端，保持整體一致性
4. **用戶體驗**: 確認對話框有效防止誤觸，提升用戶體驗

### **遇到的困難**
1. **Kotlin 編譯快取**: 遇到增量編譯快取錯誤，需要 `flutter clean`
2. **Firebase 狀態同步**: 需要確保 Firebase Auth 狀態正確同步到 UI
3. **UI 響應性**: 需要處理載入狀態和錯誤狀態的 UI 回饋

### **解決方法**
1. **清理快取**: 使用 `flutter clean` 解決 Kotlin 編譯問題
2. **狀態監聽**: 使用 `ref.watch(authStateProvider)` 監聽認證狀態變化
3. **條件渲染**: 根據 `authState is AuthStateLoading` 顯示不同 UI

### **技術教訓**
1. **ConsumerWidget**: 使用 ConsumerWidget 而非 StatelessWidget 以支援 Riverpod
2. **異步處理**: 登出操作需要正確處理異步狀態和錯誤
3. **UI 回饋**: 重要操作需要提供明確的視覺回饋
