## 📋 **問題描述**

Relay GO 應用程式的測試帳號認證失敗，需要在 Firebase Console 中手動創建測試帳號以支援開發和測試。

## 🔧 **解決方案**

### **方法一：Firebase Console 手動創建**

1. **登入 Firebase Console**
   - 前往 [Firebase Console](https://console.firebase.google.com/)
   - 選擇專案：`ride-platform-f1676`

2. **進入 Authentication 設定**
   - 點擊左側選單 "Authentication"
   - 點擊 "Users" 分頁

3. **創建客戶端測試帳號**
   - 點擊 "Add user" 按鈕
   - Email: `customer.test@relaygo.com`
   - Password: `RelayGO2024!Customer`
   - 點擊 "Add user"

4. **創建司機端測試帳號**
   - 點擊 "Add user" 按鈕
   - Email: `driver.test@relaygo.com`
   - Password: `RelayGO2024!Driver`
   - 點擊 "Add user"

### **方法二：應用程式自動註冊**

修改應用程式代碼，在測試帳號不存在時自動創建：

```dart
/// 使用測試帳號登入（如果不存在則自動創建）
Future<UserCredential?> signInWithTestAccount(String accountType) async {
  final testAccount = testAccounts[accountType];
  if (testAccount == null) {
    throw AuthException('無效的測試帳號類型：$accountType');
  }

  // 在開發環境中才允許使用測試帳號
  if (kReleaseMode) {
    throw AuthException('測試帳號僅在開發環境中可用');
  }

  try {
    // 嘗試登入
    return await signInWithEmailAndPassword(
      email: testAccount.email,
      password: testAccount.password,
    );
  } on FirebaseAuthException catch (e) {
    if (e.code == 'user-not-found') {
      // 測試帳號不存在，自動創建
      try {
        final userCredential = await _auth.createUserWithEmailAndPassword(
          email: testAccount.email,
          password: testAccount.password,
        );
        
        // 設定顯示名稱
        await userCredential.user?.updateDisplayName(testAccount.displayName);
        
        return userCredential;
      } catch (createError) {
        throw AuthException('創建測試帳號失敗：$createError');
      }
    } else {
      rethrow;
    }
  }
}
```

## 💡 **開發心得**

Firebase Authentication 的測試帳號管理是開發過程中的重要環節。建議：

1. **統一管理**: 將測試帳號資訊集中管理
2. **自動化**: 考慮實作自動創建測試帳號的功能
3. **安全性**: 確保測試帳號僅在開發環境中可用
4. **文檔化**: 詳細記錄測試帳號的用途和密碼

這個經驗提醒我們在設計認證系統時，要考慮到開發和測試的便利性，同時確保生產環境的安全性。
