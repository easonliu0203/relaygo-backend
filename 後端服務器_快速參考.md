## 🔧 常見問題

### 問題 1: 端口 3000 已被占用

**錯誤訊息**:
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解決方案**:
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# macOS/Linux
lsof -i :3000
kill -9 <PID>
```

---

### 問題 2: 依賴未安裝

**錯誤訊息**:
```
Error: Cannot find module '...'
```

**解決方案**:
```bash
cd backend
npm install
```

---

### 問題 3: 環境變數未配置

**錯誤訊息**:
```
Error: Missing environment variable: ...
```

**解決方案**:
1. 檢查 `backend/.env` 文件是否存在
2. 確認必要的環境變數已配置：
   - `PORT`
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `FIREBASE_PROJECT_ID`
   - `FIREBASE_PRIVATE_KEY`
   - `FIREBASE_CLIENT_EMAIL`

---

### 問題 4: Firebase Admin SDK 初始化失敗

**錯誤訊息**:
```
⚠️  Firebase Admin SDK 初始化失敗
```

**解決方案**:
1. 檢查 Firebase 配置是否正確
2. 確認私鑰格式正確（包含 `\n` 換行符）
3. 確認項目 ID 和客戶端郵箱正確

## 🔄 重啟服務器

### 開發模式（自動重啟）
使用 `npm run dev` 啟動的服務器會在代碼變更時自動重啟。

### 手動重啟
1. 在終端中按 `Ctrl+C` 停止服務器
2. 重新執行 `npm run dev`

### 使用 nodemon 手動重啟
在服務器運行時，在終端中輸入 `rs` 並按 Enter。
