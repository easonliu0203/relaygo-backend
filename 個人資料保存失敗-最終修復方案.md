## 📋 問題總結

### 錯誤訊息
```
更新失敗: SupabaseServiceException: 保存用戶資料失敗
```

### 根本原因

**RLS 政策與 Firebase Auth 不兼容**

1. **原始 RLS 政策使用 Supabase Auth**:
   ```sql
   CREATE POLICY "用戶可以更新自己的資料"
   ON user_profiles
   FOR UPDATE
   USING (
     user_id IN (
       SELECT id FROM users WHERE firebase_uid = auth.jwt() ->> 'sub'
     )
   );
   ```

2. **問題**:
   - `auth.jwt()` 是 Supabase Auth 的 JWT 令牌
   - 應用使用 **Firebase Auth**，不是 Supabase Auth
   - Firebase Auth 不會提供 Supabase JWT
   - `auth.jwt()` 返回 NULL
   - RLS 檢查失敗

3. **錯誤流程**:
   ```
   用戶點擊保存
       ↓
   調用 Supabase API
       ↓
   RLS 檢查: auth.jwt() ->> 'sub'
       ↓
   ❌ auth.jwt() 返回 NULL
       ↓
   ❌ RLS 檢查失敗
       ↓
   ❌ permission denied
   ```

## 🔍 為什麼禁用 RLS 是安全的？

### 當前架構的安全措施

1. **Firebase Auth 認證**:
   - 用戶必須先通過 Firebase Auth 登入
   - Firebase Auth 提供了用戶身份驗證
   - 未登入用戶無法訪問應用

2. **應用層安全**:
   - 應用邏輯確保用戶只能修改自己的資料
   - 使用 Firebase UID 進行用戶識別
   - 代碼中有用戶 ID 映射和驗證

3. **Supabase 安全**:
   - 使用 anon key（有速率限制）
   - 不暴露服務角色密鑰
   - 資料庫層面的外鍵約束

4. **網路安全**:
   - HTTPS 加密傳輸
   - Firebase Auth 令牌驗證
   - Supabase API 密鑰保護

## 🔍 故障排除

### 問題 1: 仍然出現「保存用戶資料失敗」

**檢查 RLS 狀態**:
```sql
SELECT relrowsecurity FROM pg_class WHERE relname = 'user_profiles';
```

**如果返回 `t`**: RLS 仍然啟用，重新執行修復 SQL

**如果返回 `f`**: RLS 已禁用，檢查其他問題：
- 測試帳號是否存在於 `users` 表
- 查看 Flutter 控制台的完整錯誤訊息

### 問題 2: 測試帳號不存在

**檢查**:
```sql
SELECT id, firebase_uid, email FROM users 
WHERE email IN ('customer.test@relaygo.com', 'driver.test@relaygo.com');
```

**如果沒有記錄**: 執行 `fix-test-accounts.js`

### 問題 3: 其他錯誤

**查看完整錯誤**: 在 Flutter 控制台中查看完整的錯誤堆疊追蹤

**常見錯誤**:
- `foreign key violation`: 用戶不存在
- `null value`: 資料驗證問題
- `unique constraint`: 重複記錄
