## 📋 問題描述

在公司端（web-admin）管理頁面中，訪問司機管理和客戶管理頁面時出現網路連接錯誤：
- 司機管理頁面：無法載入司機測試帳號資料
- 客戶管理頁面：無法載入客戶測試帳號資料
- 錯誤訊息：「網路連接失敗，請檢查網路狀態」

---

## 🔍 根本原因

**前端 API 服務使用了錯誤的 HTTP 客戶端**

### 問題代碼

**文件**: `web-admin/src/services/api.ts`

```typescript
// ❌ 錯誤：使用外部 API 客戶端（baseURL: http://localhost:3000）
static async getDrivers(params?: any) {
  return this.get('/api/admin/drivers', { params });
}

static async getCustomers(params?: any) {
  return this.get('/api/admin/customers', { params });
}
```

### 為什麼會失敗？

1. **錯誤的 baseURL**: 
   - `apiClient` 的 `baseURL` 設置為 `http://localhost:3000`
   - 但 web-admin 運行在 `http://localhost:3001`
   - 導致請求發送到錯誤的端口

2. **應該使用內部 API 客戶端**:
   - `internalApiClient` 使用相對路徑（`baseURL: ''`）
   - 會自動調用當前 Next.js 應用的 API 路由
   - 這是正確的做法

3. **網路連接失敗**:
   - 請求發送到 `http://localhost:3000/api/admin/drivers`
   - 但該端口沒有運行服務
   - 導致連接失敗錯誤

## 💡 開發心得

### 1. 外部 API vs 內部 API

**教訓**: Next.js 應用中要區分外部 API 和內部 API

**外部 API** (`apiClient`):
- 用於調用外部後端服務（如獨立的 Node.js 服務器）
- 需要完整的 `baseURL`（如 `http://localhost:3000`）
- 適用於微服務架構

**內部 API** (`internalApiClient`):
- 用於調用 Next.js 自己的 API 路由
- 使用相對路徑（`baseURL: ''`）
- 適用於 Next.js 全棧應用

**最佳實踐**:
```typescript
// 調用 Next.js API 路由
static async getDrivers(params?: any) {
  return this.internalGet('/api/admin/drivers', { params });
}

// 調用外部服務
static async getExternalData() {
  return this.get('/external/api/data');
}
```

### 2. 網路錯誤診斷

**教訓**: 網路連接錯誤可能是 URL 配置問題

**診斷步驟**:
1. 檢查瀏覽器開發者工具的 Network 標籤
2. 查看請求的完整 URL
3. 確認目標端口是否有服務運行
4. 檢查 `baseURL` 配置

### 3. API 客戶端設計

**教訓**: 設計 API 客戶端時要考慮不同的使用場景

**建議**:
- 提供多個客戶端實例（外部、內部）
- 提供對應的方法（`get` vs `internalGet`）
- 在方法名稱中明確區分用途
- 添加詳細的日誌記錄
