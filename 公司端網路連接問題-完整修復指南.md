## 🔍 根本原因

### 原因 1: API 服務配置錯誤

**問題**: 前端 API 服務使用了錯誤的 HTTP 客戶端

**文件**: `web-admin/src/services/api.ts`

**錯誤代碼**:
```typescript
// ❌ 使用外部 API 客戶端（baseURL: http://localhost:3000）
static async getDrivers(params?: any) {
  return this.get('/api/admin/drivers', { params });
}
```

**正確代碼**:
```typescript
// ✅ 使用內部 API 客戶端（baseURL: ''，相對路徑）
static async getDrivers(params?: any) {
  return this.internalGet('/api/admin/drivers', { params });
}
```

### 原因 2: 測試工具 CORS 問題

**問題**: 測試工具從 `file://` 協議打開，導致 CORS 錯誤

**錯誤**:
- 從 `file:///d:/repo/test-web-admin-frontend.html` 打開
- 瀏覽器阻止 `file://` 訪問 `http://` 資源
- 導致 "Failed to fetch" 錯誤

**解決方案**:
- 將測試工具移動到 `web-admin/public/test.html`
- 從 `http://localhost:3001/test.html` 訪問

### 原因 3: 瀏覽器快取

**問題**: 瀏覽器可能快取了舊的 JavaScript 代碼

**解決方案**:
- 重啟 web-admin 服務
- 清除瀏覽器快取
- 使用無痕模式測試

## 💡 開發心得

### 1. CORS 和協議限制

**教訓**: 瀏覽器的 CORS 政策會阻止不同協議之間的請求

**問題**:
- `file://` 協議無法訪問 `http://` 資源
- 這是瀏覽器的安全限制

**解決方案**:
- 將測試工具放在 web 服務器上
- 從 `http://` 協議訪問

### 2. Next.js 熱重載的限制

**教訓**: Next.js 的熱重載可能不總是正確應用更改

**問題**:
- 修改 `api.ts` 後，前端頁面可能仍使用舊代碼
- 瀏覽器可能快取了舊的 JavaScript bundle

**解決方案**:
- 重啟 web-admin 服務
- 清除瀏覽器快取
- 使用無痕模式測試

### 3. 外部 API vs 內部 API

**教訓**: Next.js 應用中要區分外部 API 和內部 API

**外部 API** (`apiClient`):
- 用於調用外部後端服務
- 需要完整的 `baseURL`

**內部 API** (`internalApiClient`):
- 用於調用 Next.js 自己的 API 路由
- 使用相對路徑（`baseURL: ''`）
