# 公司端司機配對機制 - 快速指南

## 📋 功能概述

在公司端（web-admin）訂單管理系統中，開發司機配對功能，用於將待處理訂單中未分配司機的訂單分配給可用司機。

---

## 🚀 快速開始

### 1. 啟動服務

```bash
cd web-admin
npm run dev
```

訪問: http://localhost:3001/orders/pending

### 2. 使用手動派單

**步驟**:
1. 訪問待處理訂單頁面
2. 找到未分配司機的訂單
3. 點擊訂單操作欄的「手動派單」按鈕（👤 圖標）
4. 在彈出的對話框中查看可用司機列表
5. 選擇一位無衝突的司機
6. 點擊「選擇」按鈕確認分配

**注意事項**:
- 有時間衝突的司機會標記為紅色「時間衝突」，無法選擇
- 無衝突的司機標記為綠色「可用」
- 司機列表按訂單數量排序，訂單少的在前

### 3. 使用自動派單

**步驟**:
1. 訪問待處理訂單頁面
2. 點擊頁面頂部的「自動派單」按鈕（⚡ 圖標）
3. 在確認對話框中點擊「確認」
4. 等待分配完成
5. 查看分配結果統計

**注意事項**:
- 自動派單會分配所有未分配司機的訂單
- 系統會自動檢查時間衝突
- 訂單會平均分配給所有可用司機（負載平衡）
- 只分配符合車型的司機

## 🏗️ API 端點

### 1. 獲取可用司機

**端點**: `GET /api/admin/drivers/available`

**查詢參數**:
- `vehicleType`: 車型 (A/B/C/D)
- `date`: 訂單日期 (YYYY-MM-DD)
- `time`: 訂單開始時間 (HH:mm)
- `duration`: 訂單時長（小時）

**示例**:
```bash
curl "http://localhost:3001/api/admin/drivers/available?vehicleType=A&date=2025-10-09&time=14:00&duration=3"
```

**返回**:
```json
{
  "success": true,
  "data": [
    {
      "id": "driver-id",
      "name": "張三",
      "phone": "0912345678",
      "vehicleType": "A",
      "vehiclePlate": "ABC-1234",
      "rating": 4.5,
      "currentBookings": 2,
      "hasConflict": false
    }
  ],
  "total": 5,
  "availableCount": 3
}
```

### 2. 手動分配司機

**端點**: `POST /api/admin/bookings/[id]/assign-driver`

**請求體**:
```json
{
  "driverId": "driver-id"
}
```

**示例**:
```bash
curl -X POST "http://localhost:3001/api/admin/bookings/booking-id/assign-driver" \
  -H "Content-Type: application/json" \
  -d '{"driverId":"driver-id"}'
```

**返回**:
```json
{
  "success": true,
  "data": {
    "id": "booking-id",
    "driver_id": "driver-id",
    "status": "assigned"
  },
  "message": "成功分配司機"
}
```

### 3. 自動分配司機

**端點**: `POST /api/admin/bookings/auto-assign`

**請求體（可選）**:
```json
{
  "bookingIds": ["booking-id-1", "booking-id-2"]
}
```

**示例**:
```bash
curl -X POST "http://localhost:3001/api/admin/bookings/auto-assign" \
  -H "Content-Type: application/json" \
  -d '{}'
```

**返回**:
```json
{
  "success": true,
  "message": "成功分配 5 筆訂單，失敗 2 筆",
  "assigned": 5,
  "failed": 2,
  "total": 7,
  "details": [
    {
      "bookingId": "booking-id-1",
      "driverId": "driver-id-1",
      "success": true
    },
    {
      "bookingId": "booking-id-2",
      "success": false,
      "reason": "所有司機都有時間衝突"
    }
  ]
}
```

## 🧮 時間衝突檢查

### 算法說明

**問題**: 如何判斷兩個訂單的時間段是否重疊？

**解決方案**: 使用區間重疊算法

**公式**:
```
訂單 A: start_time_A, duration_A
訂單 B: start_time_B, duration_B

end_time_A = start_time_A + duration_A
end_time_B = start_time_B + duration_B

重疊條件: (start_A < end_B) AND (end_A > start_B)
```

**示例**:
```
訂單 A: 14:00 - 17:00 (3小時)
訂單 B: 16:00 - 18:00 (2小時)

檢查: (14:00 < 18:00) AND (17:00 > 16:00)
結果: true (有衝突) ❌

訂單 A: 14:00 - 17:00 (3小時)
訂單 C: 18:00 - 20:00 (2小時)

檢查: (14:00 < 20:00) AND (17:00 > 18:00)
結果: false (無衝突) ✅
```

## 📊 負載平衡策略

### 算法說明

**目標**: 公平分配訂單給所有可用司機

**策略**:
1. 統計每個司機的當前訂單數量
2. 選擇訂單數量最少的司機
3. 分配後更新司機的訂單計數

**示例**:
```
司機 A: 2 筆訂單
司機 B: 3 筆訂單
司機 C: 1 筆訂單

新訂單 → 分配給司機 C（訂單最少）

更新後:
司機 A: 2 筆訂單
司機 B: 3 筆訂單
司機 C: 2 筆訂單
```

**位置**: 待處理訂單頁面頂部

   - 電話
   - 車型
   - 車牌號
   - 評分
   - 當前訂單數量
   - 狀態（可用 / 時間衝突）
   - 操作（選擇按鈕）

**視覺反饋**:
- 綠色 Tag: 可用
- 紅色 Tag: 時間衝突
- 禁用按鈕: 有衝突的司機

## 🐛 常見問題

### 問題 1: 沒有可用司機

**現象**: 司機選擇對話框顯示「沒有可用司機」

**原因**:
- 所有司機都不可用（`is_available = false`）
- 沒有符合車型的司機
- 所有司機都有時間衝突

**解決**:
1. 檢查司機的 `is_available` 狀態
2. 確認有符合車型的司機
3. 調整訂單時間避免衝突

### 問題 2: 自動派單失敗

**現象**: 自動派單顯示「失敗 X 筆」

**原因**:
- 找不到符合車型的司機
- 所有司機都有時間衝突
- 司機不可用

**解決**:
1. 查看分配結果的 `details` 欄位
2. 根據失敗原因調整
3. 手動分配失敗的訂單

### 問題 3: 時間衝突檢查不準確

**現象**: 顯示有衝突但實際沒有衝突

**原因**:
- 訂單時間格式錯誤
- 時長計算錯誤
- 時區問題

**解決**:
1. 檢查訂單的 `start_date` 和 `start_time` 格式
2. 確認 `duration_hours` 正確
3. 檢查伺服器時區設置
