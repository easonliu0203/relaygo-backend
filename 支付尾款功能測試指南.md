```sql
SELECT 
  id,

  status,
  customer_id,
  total_amount,
  deposit_amount,
  (total_amount - deposit_amount) as balance_amount

WHERE status = 'trip_ended'
ORDER BY created_at DESC
LIMIT 5;
```

## 🔍 診斷問題

### 問題 1: API 返回 404

**可能原因**:
- Backend 沒有運行
- 路由配置錯誤

**解決方法**:
```bash
# 檢查 Backend 是否運行
curl http://localhost:3000/api/booking-flow/test

# 預期響應
{
  "success": true,
  "message": "Booking Flow API is working"
}
```

---

### 問題 2: API 返回 "訂單不存在"

**可能原因**:
- `bookingId` 錯誤
- 訂單已被刪除

**解決方法**:
```sql
-- 檢查訂單是否存在
SELECT * FROM bookings WHERE id = '<bookingId>';
```

---

### 問題 3: API 返回 "訂單狀態不正確"

**可能原因**:
- 訂單狀態不是 `trip_ended`

**解決方法**:
```sql
-- 檢查訂單狀態
SELECT id, status FROM bookings WHERE id = '<bookingId>';

-- 如果需要，手動更新訂單狀態（僅測試用）
UPDATE bookings 
SET status = 'trip_ended', updated_at = NOW()
WHERE id = '<bookingId>';
```

---

### 問題 4: 沒有返回 paymentUrl

**可能原因**:
- 環境變數 `PAYMENT_PROVIDER` 設置為 `mock`
- PaymentProviderFactory 沒有正確初始化

**解決方法**:
```bash
# 檢查環境變數
echo $PAYMENT_PROVIDER

# 應該是 'gomypay'，如果是 'mock' 則需要修改
export PAYMENT_PROVIDER=gomypay

# 重啟 Backend
npm run dev
```

---

### 問題 5: Flutter App 沒有跳轉到支付頁面

**可能原因**:
- Flutter App 沒有重新編譯
- 路由配置錯誤
- `requiresRedirect` 檢查邏輯錯誤

**解決方法**:
```bash
# 完全重新編譯
cd mobile
flutter clean
flutter pub get
flutter run

# 檢查日誌
# 應該看到：
# [PaymentBalance] 跳轉到支付頁面: https://n.gomypay.asia/...
```

## 📝 注意事項

1. **測試環境 vs 生產環境**
   - 測試環境：`GOMYPAY_TEST_MODE=true`
   - 生產環境：`GOMYPAY_TEST_MODE=false`

2. **支付金額**
   - GOMYPAY 金額以「分」為單位
   - 例如：100 元 = 10000 分

3. **訂單編號格式**
   - 格式：`{bookingId}{D/B}{timestamp}`
   - D = Deposit（訂金）
   - B = Balance（尾款）

4. **回調 URL**
   - 確保 GOMYPAY 後台設置的回調 URL 正確
   - `https://api.relaygo.pro/api/payment/gomypay/callback`
