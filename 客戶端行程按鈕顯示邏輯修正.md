## 📋 問題描述

根據狀態映射表，客戶端訂單詳情頁面的按鈕顯示邏輯需要調整：

### 狀態映射表

```javascript
// === 階段 II: 服務中 ===
'driver_confirmed': 'matched',      // 司機確認後 → 已配對
'driver_departed': 'ON_THE_WAY',    // 司機已出發 → 正在路上
'driver_arrived': 'ON_THE_WAY',     // 司機已到達 → 正在路上
'trip_started': 'inProgress',       // 行程開始 → 進行中
'in_progress': 'inProgress',        // 通用進行中狀態
```

### 需求

1. **開始行程按鈕**：應該在 `ON_THE_WAY` 狀態顯示
   - Supabase 狀態：`driver_departed` 或 `driver_arrived`
   - Firestore 狀態：`ON_THE_WAY`

2. **結束行程按鈕**：應該在 `inProgress` 狀態顯示
   - Supabase 狀態：`trip_started`
   - Firestore 狀態：`inProgress`

## 🔄 完整流程

### 1. 司機確認接單
- **Supabase**: `driver_confirmed`
- **Firestore**: `matched`
- **客戶端顯示**: 司機資訊卡片

### 2. 司機出發
- **Supabase**: `driver_departed`
- **Firestore**: `ON_THE_WAY`
- **客戶端顯示**: 司機資訊卡片 + **「開始行程」按鈕**

### 3. 司機到達
- **Supabase**: `driver_arrived`
- **Firestore**: `ON_THE_WAY`
- **客戶端顯示**: 司機資訊卡片 + **「開始行程」按鈕**

### 4. 客戶開始行程
- **Supabase**: `trip_started`
- **Firestore**: `inProgress`
- **客戶端顯示**: 司機資訊卡片 + **「結束行程」按鈕**

### 5. 客戶結束行程
- **Supabase**: `trip_ended`
- **Firestore**: `awaitingBalance`
- **客戶端顯示**: 司機資訊卡片 + **「支付尾款」按鈕**

## 🧪 測試步驟

### 前置條件
1. 確保 Backend API 正在運行
2. 確保 Realtime Sync 已啟用
3. 準備一個測試訂單

### 測試流程

1. **司機端操作**：
   - 司機確認接單
   - 檢查客戶端是否顯示司機資訊

2. **司機端操作**：
   - 司機點擊「出發前往載客」
   - 檢查客戶端是否顯示「開始行程」按鈕

3. **司機端操作**：
   - 司機點擊「抵達上車地點」
   - 檢查客戶端是否仍然顯示「開始行程」按鈕

4. **客戶端操作**：
   - 客戶點擊「開始行程」
   - 檢查是否成功開始行程
   - 檢查是否顯示「結束行程」按鈕

5. **客戶端操作**：
   - 客戶點擊「結束行程」
   - 檢查是否成功結束行程
   - 檢查是否顯示「支付尾款」按鈕
