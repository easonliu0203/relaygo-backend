## 💡 根本原因分析

### 聊天室創建時機

根據代碼分析（`backend/src/services/events/BookingEventBus.ts`），聊天室是在**訂單狀態變更為 `driver_confirmed`** 時創建的：

```typescript
// 處理司機確認
private async handleDriverConfirmed(eventData: BookingStatusChangedEvent): Promise<void> {
  console.log(`[EventBus] Driver confirmed booking ${eventData.bookingId}`);
  
  // 創建聊天室
  try {
    const booking = await this.getBookingById(eventData.bookingId);
    if (booking && booking.customerId && booking.driverId) {
      await chatService.createChatRoom(
        eventData.bookingId,
        booking.customerId,
        booking.driverId
      );
    }
  } catch (error) {
    console.error('[EventBus] Error creating chat room:', error);
  }
}
```

### 訂單狀態流程

```
pending 
  → paid_deposit 
  → assigned 
  → driver_confirmed  ← 聊天室在這裡創建
  → driver_departed
  → driver_arrived
  → trip_started
  → trip_ended
  → completed
```

### 可能的原因

1. **訂單狀態不是 `driver_confirmed`**:
   - 訂單可能還在 `assigned` 狀態（司機已接單但還沒確認）
   - 需要將訂單狀態更新為 `driver_confirmed`

2. **聊天室創建失敗**:
   - Backend EventBus 沒有正確觸發
   - ChatService 創建聊天室時發生錯誤
   - 需要檢查 Backend 日誌

3. **Firestore 同步失敗**:
   - Edge Function 沒有將聊天室同步到 Firestore
   - 需要檢查 Edge Function 日誌

## 🛠️ 解決方案

### 方案 1: 更新訂單狀態為 `driver_confirmed`

**使用 Supabase SQL Editor**:
```sql
-- 更新訂單狀態為 driver_confirmed
UPDATE bookings
SET 
  status = 'driver_confirmed',
  updated_at = NOW()
WHERE id = 'YOUR_BOOKING_ID';
```

**注意**: 替換 `YOUR_BOOKING_ID` 為實際的訂單 ID

### 方案 2: 手動創建聊天室（臨時方案）

**使用 Firestore Console 手動創建聊天室**:

1. 前往 Firebase Console > Firestore
2. 創建 collection: `chat_rooms`
3. 創建 document，ID 為訂單 ID
4. 添加以下欄位：
   ```
   customerId: "客戶的 Firebase UID"
   driverId: "司機的 Firebase UID"
   customerName: "客戶姓名"
   driverName: "司機姓名"
   pickupAddress: "上車地點"
   bookingTime: Timestamp (預約時間)
   lastMessage: null
   lastMessageTime: null
   customerUnreadCount: 0
   driverUnreadCount: 0
   updatedAt: Timestamp (現在)
   ```

### 方案 3: 發送第一則訊息觸發聊天室創建

**使用聊天 API 發送訊息**:
```bash
curl -X POST http://localhost:3001/api/chat/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "bookingId": "YOUR_BOOKING_ID",
    "receiverId": "RECEIVER_ID",
    "messageText": "你好"
  }'
```

**注意**: 這會觸發聊天室創建和 Firestore 同步

## 📝 常見問題

### Q1: 為什麼訂單狀態不是 `driver_confirmed`？

**A**: 訂單狀態流程需要經過多個步驟：
1. 客戶創建訂單 → `pending`
2. 客戶支付訂金 → `paid_deposit`
3. 系統分配司機 → `assigned`
4. **司機確認訂單 → `driver_confirmed`** ← 聊天室在這裡創建

如果司機只是接單但還沒確認，訂單狀態會停留在 `assigned`。

### Q2: 如何手動將訂單狀態更新為 `driver_confirmed`？

**A**: 使用 Supabase SQL Editor 執行：
```sql
UPDATE bookings
SET status = 'driver_confirmed', updated_at = NOW()
WHERE id = 'YOUR_BOOKING_ID';
```

### Q3: 更新訂單狀態後，聊天室會自動創建嗎？

**A**: 不會。聊天室只在訂單狀態**變更**為 `driver_confirmed` 時創建（通過 EventBus 觸發）。

如果直接更新資料庫，需要：
1. 手動創建聊天室（方案 2）
2. 或發送第一則訊息觸發創建（方案 3）

### Q4: 如何確認聊天室已經創建？

**A**: 
1. 前往 Firebase Console > Firestore > chat_rooms
2. 查看是否有對應訂單 ID 的文檔
3. 或執行診斷腳本：`node diagnose-chat-rooms-issue.js`
