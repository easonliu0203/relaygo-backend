## 🚨 可能的問題和修復方案

### 問題 A：Cron Job 未設置

**症狀**：
- 診斷查詢顯示「Cron Job 未配置」
- `cron.job` 表中沒有 `sync-orders-to-firestore`

**修復方案**：

1. **打開 Supabase SQL Editor**
   ```
   https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/sql
   ```

2. **執行以下 SQL**（我已經為您準備好了）：

```sql
-- 啟用 pg_cron 擴展
CREATE EXTENSION IF NOT EXISTS pg_cron;

-- 刪除舊的 Cron Job（如果存在）
SELECT cron.unschedule('sync-orders-to-firestore') 
WHERE EXISTS (
  SELECT 1 FROM cron.job WHERE jobname = 'sync-orders-to-firestore'
);

-- 創建同步任務（每 30 秒執行一次）
-- 注意：這裡使用匿名代碼塊來獲取 service_role_key
SELECT cron.schedule(
  'sync-orders-to-firestore',
  '*/30 * * * * *',
  $$
  SELECT
    net.http_post(
      url := 'https://vlyhwegpvpnjyocqmfqc.supabase.co/functions/v1/sync-to-firestore',
      headers := jsonb_build_object(
        'Authorization', 'Bearer ' || current_setting('request.jwt.claims', true)::json->>'role'
      )
    ) as request_id;
  $$
);

-- 驗證 Cron Job 已創建
SELECT 
  jobid,
  jobname,
  schedule,
  active
FROM cron.job
WHERE jobname = 'sync-orders-to-firestore';
```

### 問題 B：Cron Job 配置錯誤（service_role_key 問題）

**症狀**：
- Cron Job 存在但從未執行
- 或執行失敗，返回認證錯誤

**根本原因**：
`current_setting('app.settings.service_role_key', true)` 可能不存在

**修復方案**：

我已經創建了一個修復腳本。請執行：

1. **打開 Supabase SQL Editor**

2. **執行修復腳本**（見下方「修復腳本 B」）

---

### 問題 C：Edge Function 環境變數未設置

**症狀**：
- Cron Job 執行成功
- 但 `outbox` 事件有 `error_message`
- 錯誤訊息提到 `FIREBASE_PROJECT_ID` 或 `FIREBASE_API_KEY`

**修復方案**：

1. **打開 Supabase Dashboard**
   ```
   https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/functions
   ```

2. **設置環境變數**：
   - `FIREBASE_PROJECT_ID` = 您的 Firebase Project ID
   - `FIREBASE_API_KEY` = 您的 Firebase Web API Key

3. **重新部署 Edge Function**：
   ```bash
   npx supabase functions deploy sync-to-firestore --project-ref vlyhwegpvpnjyocqmfqc
   ```

---

### 問題 D：Firestore 權限問題

**症狀**：
- Edge Function 執行
- 錯誤訊息提到 `Firestore 更新失敗 (403)`

**修復方案**：

檢查 Firebase API Key 是否有權限寫入 Firestore。

## 📝 修復腳本

### 修復腳本 A：重新創建 Cron Job（簡化版）

```sql
-- 啟用 pg_cron
CREATE EXTENSION IF NOT EXISTS pg_cron;

-- 刪除舊的
SELECT cron.unschedule('sync-orders-to-firestore') 
WHERE EXISTS (SELECT 1 FROM cron.job WHERE jobname = 'sync-orders-to-firestore');

-- 創建新的（使用 anon key 或 service_role key）
-- 方法 1：使用 Supabase 內建的 service_role
SELECT cron.schedule(
  'sync-orders-to-firestore',
  '*/30 * * * * *',
  $$
  SELECT extensions.http_post(
    'https://vlyhwegpvpnjyocqmfqc.supabase.co/functions/v1/sync-to-firestore'::text,
    '{}'::jsonb,
    'application/json'::text,
    jsonb_build_object(
      'Authorization', 'Bearer ' || (SELECT decrypted_secret FROM vault.decrypted_secrets WHERE name = 'service_role_key' LIMIT 1)
    )
  );
  $$
);
```

### 修復腳本 B：手動觸發測試

如果 Cron Job 設置複雜，可以先手動觸發測試：

```sql
-- 手動觸發 Edge Function
SELECT extensions.http_post(
  'https://vlyhwegpvpnjyocqmfqc.supabase.co/functions/v1/sync-to-firestore'::text,
  '{}'::jsonb,
  'application/json'::text,
  jsonb_build_object(
    'Authorization', 'Bearer YOUR_SERVICE_ROLE_KEY_HERE'
  )
);
```

**注意**：將 `YOUR_SERVICE_ROLE_KEY_HERE` 替換為您的 service_role_key。

**如何獲取 service_role_key**：
1. 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/api
2. 複製「service_role」key（secret）
