# 個人資料編輯保存失敗修復 - 測試驗證指南

**日期**: 2025-10-10 11:00  
**問題**: 個人資料編輯保存失敗  
**狀態**: ✅ 已修復

---

## 📋 問題描述

### 錯誤訊息

```
更新失敗: SupabaseServiceException: 保存用戶資料失敗
```

### 根本原因

1. **RLS 政策缺失**: `user_profiles` 表沒有啟用 RLS 政策，導致用戶無法插入/更新自己的資料
2. **用戶 ID 映射問題**: 代碼使用 Firebase UID，但需要先映射到 Supabase 的 `users.id`

---

1. 用戶可以查看自己的資料

**文件**: `mobile/lib/shared/providers/user_profile_provider.dart`

## 🔍 故障排除

### 問題 1: 仍然出現「保存用戶資料失敗」錯誤

**症狀**:
```
SupabaseServiceException: 保存用戶資料失敗
```

**檢查**:
1. 確認 RLS 政策已正確應用
2. 確認用戶存在於 `users` 表中
3. 查看完整的錯誤堆疊追蹤

**診斷步驟**:

```bash
# 在應用控制台中查看完整錯誤
# 應該會顯示更詳細的錯誤訊息
```

**解決方案**:

```sql
-- 1. 檢查 RLS 政策
SELECT * FROM pg_policies WHERE tablename = 'user_profiles';

-- 2. 檢查用戶記錄
SELECT * FROM users WHERE email = 'customer.test@relaygo.com';

-- 3. 如果用戶不存在，創建用戶記錄
-- （參考 fix-test-accounts.js 腳本）
```

### 問題 2: 「用戶不存在」錯誤

**症狀**:
```
用戶不存在。請確保用戶已在 Supabase users 表中創建。Firebase UID: ...
```

**原因**: 測試帳號只存在於 Firebase Auth，不存在於 Supabase `users` 表

**解決方案**:

```bash
# 執行修復腳本
cd d:\repo
node fix-test-accounts.js
```

或手動執行 SQL:

```sql
-- 創建客戶測試帳號
INSERT INTO users (firebase_uid, email, role, status)
VALUES ('YOUR_FIREBASE_UID', 'customer.test@relaygo.com', 'customer', 'active')
ON CONFLICT (firebase_uid) DO NOTHING;

-- 創建司機測試帳號
INSERT INTO users (firebase_uid, email, role, status)
VALUES ('YOUR_FIREBASE_UID', 'driver.test@relaygo.com', 'driver', 'active')
ON CONFLICT (firebase_uid) DO NOTHING;
```

### 問題 3: RLS 政策未生效

**症狀**: 執行了 SQL 腳本但仍然無法保存

**檢查**:

```sql
-- 檢查 RLS 是否真的啟用
SELECT relname, relrowsecurity
FROM pg_class
WHERE relname = 'user_profiles';
```

**解決方案**:

```sql
-- 強制啟用 RLS
ALTER TABLE user_profiles ENABLE ROW LEVEL SECURITY;

-- 重新創建政策
-- （執行 apply-user-profiles-rls.sql）
```

### 問題 4: 權限錯誤

**症狀**:
```
permission denied for table user_profiles
```

**原因**: RLS 政策配置錯誤或 JWT 令牌問題

**檢查**:
1. 確認用戶已登入
2. 確認 Firebase UID 正確
3. 確認 RLS 政策使用正確的 JWT 欄位

**解決方案**:

```sql
-- 檢查 JWT 令牌內容（在應用中）
-- auth.jwt() ->> 'sub' 應該返回 Firebase UID

-- 測試政策
SELECT * FROM user_profiles
WHERE user_id IN (
  SELECT id FROM users WHERE firebase_uid = 'YOUR_FIREBASE_UID'
);
```
