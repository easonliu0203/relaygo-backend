# 🔧 整數類型錯誤修復指南

**日期**: 2025-10-09  
**問題**: `type 'double' is not a subtype of type 'int'`  
**狀態**: ✅ 已修復

---

## 📋 問題描述

### 錯誤訊息
```
載入失敗
type 'double' is not a subtype of type 'int'
```

### 發生位置
- 客戶端「我的訂單 > 進行中」標籤頁
- 客戶端「我的訂單 > 歷史訂單」標籤頁

### 症狀
- 頁面無法正常載入訂單列表
- 出現數值類型轉換錯誤

---

## 🔍 根本原因分析

### 問題根源

1. **Firestore 資料格式不一致**
   - **舊資料**: 整數欄位存儲為 `double` 格式
   - **新資料**: 整數欄位存儲為 `integerValue` 格式 (正確)

2. **Flutter 代碼不兼容**
   - `BookingOrder.fromFirestore()` 方法直接使用值
   - 代碼: `passengerCount: data['passengerCount'] ?? 1`
   - 遇到 double 格式時拋出錯誤

3. **Edge Function 問題**
   - `passengerCount` 硬編碼為 1,沒有使用實際值
   - `luggageCount` 設置為 null,沒有使用實際值

### 影響範圍

**受影響的欄位**:
- `passengerCount` (int) - 乘客數量
- `luggageCount` (int?) - 行李數量

**受影響的集合**:
- `orders_rt` - 客戶端即時訂單
- `bookings` - 完整訂單記錄

---

## ✅ 解決方案

### 方案概述

採用 **混合方案**,確保向後兼容:

1. ✅ **修改 Flutter 代碼** - 支持 int 和 double 兩種格式
2. ✅ **修改 Edge Function** - 使用實際值並確保正確格式
3. ✅ **提供遷移腳本** - 將舊資料轉換為正確格式
4. ✅ **提供驗證腳本** - 檢查資料格式是否正確

### 優點

- ✅ **立即修復**: Flutter 代碼修改後,應用立即可用
- ✅ **向後兼容**: 支持舊資料,不會因為資料格式導致錯誤
- ✅ **修復根源**: Edge Function 使用實際值和正確格式
- ✅ **漸進遷移**: 可以選擇性地遷移舊資料

## 🔧 修復步驟

### 步驟 1: 修改 Flutter 代碼 ✅ 已完成

**文件**: `mobile/lib/core/models/booking_order.dart`

**修改內容**:

1. 添加輔助函數 `_parseInt()` 和 `_parseOptionalInt()`
2. 更新 `fromFirestore()` 方法使用這些函數

**關鍵代碼**:

```dart
/// 解析整數 - 支持 int 和 double 兩種格式
static int _parseInt(dynamic value, {required int defaultValue}) {
  if (value == null) {
    return defaultValue;
  }
  
  // 處理整數格式 (正確格式)
  if (value is int) {
    return value;
  }
  
  // 處理 double 格式 (舊資料兼容)
  if (value is double) {
    return value.toInt();
  }
  
  // 處理 String 格式 (極少數情況)
  if (value is String) {
    try {
      return int.parse(value);
    } catch (e) {
      return defaultValue;
    }
  }
  
  return defaultValue;
}

/// 解析可選的整數 - 支持 null 值
static int? _parseOptionalInt(dynamic value) {
  if (value == null) return null;
  return _parseInt(value, defaultValue: 0);
}
```

**使用方式**:

```dart
factory BookingOrder.fromFirestore(DocumentSnapshot doc) {
  final data = doc.data() as Map<String, dynamic>;
  
  return BookingOrder(
    ...
    passengerCount: _parseInt(data['passengerCount'], defaultValue: 1),
    luggageCount: _parseOptionalInt(data['luggageCount']),
    ...
  );
}
```

### 步驟 2: 修改 Edge Function ✅ 已完成

**文件**: `supabase/functions/sync-to-firestore/index.ts`

**修改內容**:

**修改前** (錯誤):
```typescript
// 乘客資訊（使用 _integer 標記，convertToFirestoreFields 會轉換為整數）
passengerCount: {
  _integer: 1,  // ❌ 硬編碼為 1
},
luggageCount: null,  // ❌ 沒有使用實際值
```

**修改後** (正確):
```typescript
// 乘客資訊（使用 _integer 標記，convertToFirestoreFields 會轉換為整數）
passengerCount: {
  _integer: bookingData.passengerCount || 1,  // ✅ 使用實際值
},
luggageCount: bookingData.luggageCount ? {
  _integer: bookingData.luggageCount,  // ✅ 使用實際值
} : null,
```

**效果**:
- ✅ 使用實際的 passengerCount 值
- ✅ 使用實際的 luggageCount 值
- ✅ 使用 `_integer` 標記確保正確格式

## 📊 技術說明

### 為什麼會有兩種格式?

**歷史演進**:

1. **舊版 Edge Function** (已棄用)
   ```typescript
   const firestoreData = {
     passengerCount: 1,  // ❌ 硬編碼,且可能存儲為 double
     luggageCount: null,  // ❌ 沒有使用實際值
   }
   ```

2. **新版 Edge Function** (當前版本)
   ```typescript
   const firestoreData = {
     passengerCount: {
       _integer: bookingData.passengerCount || 1,  // ✅ 使用實際值
     },
     luggageCount: bookingData.luggageCount ? {
       _integer: bookingData.luggageCount,  // ✅ 使用實際值
     } : null,
   }
   
   // convertToFirestoreFields 會轉換為:
   {
     passengerCount: { integerValue: "1" },  // ✅ Firestore integerValue
     luggageCount: { integerValue: "2" },    // ✅ Firestore integerValue
   }
   ```

### 為什麼 Flutter 代碼需要支持兩種格式?

1. **舊資料未遷移**: Firestore 中仍有 double 格式的整數
2. **向後兼容**: 確保應用在遷移期間仍然可用
3. **漸進遷移**: 可以選擇性地遷移資料,不影響業務
