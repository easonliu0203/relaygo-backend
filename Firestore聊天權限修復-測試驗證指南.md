## 📋 修復內容總結

### 問題
- ❌ 客戶端和司機端聊天室列表載入失敗
- ❌ Firestore 權限錯誤：`[cloud_firestore/permission-denied]`

### 修復
- ✅ 添加 `chat_rooms` collection 的 Firestore Security Rules
- ✅ 修改 ChatService 查詢邏輯（使用兩個獨立查詢）
- ✅ 添加 rxdart 依賴
- ✅ 部署 Firestore Security Rules

---

## 🚀 測試前準備

### 1. 確認修復已部署

**檢查 Firestore Security Rules**:
```bash
cd firebase
firebase deploy --only firestore:rules
```

**預期輸出**:
```
+  cloud.firestore: rules file firestore.rules compiled successfully
+  firestore: released rules firestore.rules to cloud.firestore
+  Deploy complete!
```

### 2. 重新建置 Flutter APP

**清理並重新建置**:
```bash
cd mobile
flutter clean
flutter pub get
flutter pub run build_runner build --delete-conflicting-outputs
```

### 3. 安裝到模擬器

**客戶端 APP**:
```bash
flutter run -t lib/apps/customer/main_customer.dart -d emulator-5554
```

或者使用已生成的 APK：
```bash
flutter install -d emulator-5554 --use-application-binary=build/app/outputs/flutter-apk/app-customer-debug.apk
```

## 🔍 常見問題排查

### 問題 1: 仍然出現權限錯誤

**可能原因**:
1. Firestore Security Rules 沒有正確部署
2. Firebase 緩存問題

**解決方案**:
```bash
# 重新部署 Firestore Security Rules
cd firebase
firebase deploy --only firestore:rules

# 清除 Firebase 緩存
firebase logout
firebase login
```

### 問題 2: 聊天室列表為空

**可能原因**:
1. 沒有創建訂單
2. 訂單還沒有分配司機
3. Firestore 同步延遲

**解決方案**:
1. 創建一個新訂單並分配司機
2. 等待幾秒鐘讓 Edge Function 同步資料
3. 下拉刷新聊天室列表

### 問題 3: 訊息發送失敗

**可能原因**:
1. Supabase API 錯誤
2. 網路連接問題
3. 用戶未登入

**解決方案**:
1. 檢查 Supabase API 是否正常運行
2. 檢查模擬器網路連接
3. 重新登入 APP

### 問題 4: 未讀訊息計數不正確

**可能原因**:
1. 標記已讀 API 失敗
2. Firestore 同步延遲

**解決方案**:
1. 檢查 Supabase API 日誌
2. 等待幾秒鐘讓資料同步
3. 重新進入聊天室
