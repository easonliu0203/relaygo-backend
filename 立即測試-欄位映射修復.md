# 立即測試：欄位映射修復

**問題**：客戶端 App 顯示「訂單不存在」  
**根本原因**：Edge Function 寫入的欄位與客戶端期望的不匹配  
**修復**：已更新 Edge Function 的欄位映射  
**狀態**：✅ 已部署，待測試

---

## 🎯 問題根源

### 修復前（錯誤）

**Edge Function 寫入**：
```json
{
  "destination": "台北101",        // ❌ 錯誤欄位名稱
  "startDate": "2025-10-05",       // ❌ 分開的日期
  "startTime": "09:00:00",         // ❌ 分開的時間
  // ❌ 缺少 dropoffLocation
  // ❌ 缺少 passengerCount
  // ❌ 缺少 estimatedFare
}
```

**客戶端期望**：
```dart
{
  "dropoffAddress": "台北101",     // ✅ 正確欄位名稱
  "bookingTime": "2025-10-05T09:00:00",  // ✅ 組合的時間
  "dropoffLocation": { ... },      // ✅ 必需欄位
  "passengerCount": 1,             // ✅ 必需欄位
  "estimatedFare": 1000,           // ✅ 必需欄位
}
```

**結果**：客戶端無法解析，顯示「訂單不存在」

---

### 修復後（正確）

**Edge Function 寫入**：
```json
{
  "dropoffAddress": "台北101",     // ✅ 正確
  "dropoffLocation": { "latitude": 0, "longitude": 0 },  // ✅ 正確
  "bookingTime": "2025-10-05T09:00:00",  // ✅ 正確
  "passengerCount": 1,             // ✅ 正確
  "estimatedFare": 1000,           // ✅ 正確
  "depositPaid": false,            // ✅ 正確
  ...
}
```

**結果**：客戶端可以正確解析，顯示訂單詳情

## 🆘 如果仍然失敗

### 問題 A：Edge Function 日誌沒有「轉換後的 Firestore 資料」

**可能原因**：
- Edge Function 沒有重新部署
- 或使用的是舊版本

**解決**：
1. 確認部署成功：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/functions
2. 查看 sync-to-firestore 的「Version」
3. 如果不是最新版本，重新部署：
   ```bash
   npx supabase functions deploy sync-to-firestore --project-ref vlyhwegpvpnjyocqmfqc
   ```

---

### 問題 B：Firestore 文檔仍然是舊格式

**可能原因**：
- 查看的是舊訂單
- 新訂單還沒有同步

**解決**：
1. 創建一個新訂單（從客戶端 App）
2. 等待 30 秒（Cron Job 執行）
3. 或手動觸發 Edge Function
4. 檢查新訂單的 Firestore 文檔

---

### 問題 C：客戶端 App 仍然顯示「訂單不存在」

**可能原因**：
- 客戶端緩存
- 或查詢的是舊訂單

**解決**：
1. 重新啟動客戶端 App
2. 創建一個新訂單
3. 查看新訂單的詳情
