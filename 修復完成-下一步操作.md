### 修復前（錯誤）
```typescript
// ❌ 使用 Web API Key

  }
})
```

**錯誤訊息**：
```
"reason": "ACCESS_TOKEN_TYPE_UNSUPPORTED"
"message": "Expected OAuth 2 access token"
```

---

### 修復後（正確）
```typescript
// ✅ 使用 Service Account
const FIREBASE_SERVICE_ACCOUNT = Deno.env.get('FIREBASE_SERVICE_ACCOUNT')!
const serviceAccount = JSON.parse(FIREBASE_SERVICE_ACCOUNT)

// ✅ 生成 JWT 並交換為 OAuth 2.0 Access Token
async function getAccessToken(): Promise<string> {
  // 創建 JWT
  const jwt = createJWT(serviceAccount)
  
  // 交換為 Access Token
  const response = await fetch('https://oauth2.googleapis.com/token', {
    method: 'POST',
    body: new URLSearchParams({
      grant_type: 'urn:ietf:params:oauth:grant-type:jwt-bearer',
      assertion: jwt,
    }),
  })
  
  const data = await response.json()
  return data.access_token  // ✅ 正確的 OAuth 2.0 token
}

// ✅ 使用 OAuth 2.0 Access Token
const token = await getAccessToken()
const response = await fetch(url, {
  headers: {
    'Authorization': `Bearer ${token}`,  // ✅ 成功
  }
})
```

## 🔧 技術細節

### OAuth 2.0 認證流程

1. **創建 JWT**
   - Header: `{ alg: 'RS256', typ: 'JWT' }`
   - Payload: `{ iss, scope, aud, exp, iat }`
   - 使用 Service Account private key 簽名

2. **交換 JWT 為 Access Token**
   - POST `https://oauth2.googleapis.com/token`
   - Body: `grant_type=urn:ietf:params:oauth:grant-type:jwt-bearer&assertion=<JWT>`
   - 返回: `{ access_token, expires_in }`

3. **使用 Access Token**
   - Header: `Authorization: Bearer <access_token>`
   - 有效期: 1 小時
   - 緩存機制: 避免重複請求

## 🆘 故障排除

### 問題 A：JSON 格式錯誤

**錯誤訊息**：
```
JSON parse error
或
Unexpected token
```

**解決**：
- 確認 JSON 格式正確
- 使用 JSON 驗證工具：https://jsonlint.com
- 確保沒有多餘的空格或換行

---

### 問題 B：獲取 Access Token 失敗

**錯誤訊息**：
```
❌ 獲取 Access Token 失敗
```

**解決**：
1. 檢查 Service Account JSON 是否正確
2. 檢查 private_key 是否完整（包含 BEGIN 和 END）
3. 查看 Edge Function 日誌獲取詳細錯誤

---

### 問題 C：仍然 401 錯誤

**可能原因**：
- Service Account 沒有 Firestore 權限
- Project ID 不正確

**解決**：
1. 在 Firebase Console → IAM & Admin 檢查權限
2. 確認 Service Account 有「Firebase Admin SDK Administrator Service Agent」角色
3. 確認 Project ID 正確
