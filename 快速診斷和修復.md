## 🎯 三步驟快速修復

### 步驟 1：診斷問題（2 分鐘）

**執行診斷查詢**：

1. 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/sql
2. 點擊「New query」
3. 複製 `supabase/diagnose-current-issue.sql` 內容並執行
4. 查看「診斷總結」部分

**可能的診斷結果**：

| 診斷結果 | 問題 | 跳到 |
|----------|------|------|
| 「Cron Job 未配置」 | Cron Job 不存在 | 步驟 2A |
| 「Cron Job 從未執行」 | Cron Job 配置錯誤 | 步驟 2B |
| 「有錯誤事件」 | Edge Function 執行失敗 | 步驟 2C |
| 「事件等待處理中」 | 正常，只需等待或手動觸發 | 步驟 2D |

---

### 步驟 2A：修復 Cron Job 未配置（3 分鐘）

**如果診斷顯示「Cron Job 未配置」**：

1. **獲取 service_role_key**：
   - 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/api
   - 找到「service_role」section
   - 點擊「Reveal」
   - 複製 key

2. **執行修復腳本**：
   - 打開 SQL Editor
   - 複製 `supabase/setup_cron_jobs_fixed.sql` 內容
   - **重要**：將所有 `YOUR_SERVICE_ROLE_KEY` 替換為剛才複製的 key
   - 執行

3. **驗證**：
   ```sql
   SELECT * FROM cron.job WHERE jobname = 'sync-orders-to-firestore';
   ```
   - 應該看到一筆記錄，`active = true`

4. **等待 30 秒**，然後跳到步驟 3

---

### 步驟 2B：修復 Cron Job 配置錯誤（3 分鐘）

**如果診斷顯示「Cron Job 從未執行」**：

1. **刪除舊的 Cron Job**：
   ```sql
   SELECT cron.unschedule('sync-orders-to-firestore');
   ```

2. **按照步驟 2A 重新創建**

---

### 步驟 2C：修復 Edge Function 錯誤（5 分鐘）

**如果診斷顯示「有錯誤事件」**：

1. **查看錯誤訊息**：
   ```sql
   SELECT error_message FROM outbox 
   WHERE error_message IS NOT NULL 
   ORDER BY created_at DESC LIMIT 1;
   ```

2. **根據錯誤訊息修復**：

   **錯誤 A：`FIREBASE_PROJECT_ID` 或 `FIREBASE_API_KEY` 未定義**
   
   - 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/functions
   - 設置環境變數：
     - `FIREBASE_PROJECT_ID` = 您的 Firebase Project ID
     - `FIREBASE_API_KEY` = 您的 Firebase Web API Key
   - 重新部署：
     ```bash
     npx supabase functions deploy sync-to-firestore --project-ref vlyhwegpvpnjyocqmfqc
     ```

   **錯誤 B：`Firestore 更新失敗 (403)`**
   
   - 檢查 Firebase API Key 是否正確
   - 檢查 Firestore 安全規則

   **錯誤 C：其他錯誤**
   
   - 查看 Edge Function 日誌：
     - 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/functions
     - 點擊 sync-to-firestore → Logs

---

### 步驟 2D：手動觸發測試（1 分鐘）

**如果診斷顯示「事件等待處理中」**：

**方法 1：在 Dashboard 手動觸發**（推薦）

1. 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/functions
2. 點擊 `sync-to-firestore`
3. 點擊「Invoke」按鈕
4. 查看返回結果

**方法 2：使用 SQL 手動觸發**

1. 獲取 service_role_key（見步驟 2A）
2. 打開 SQL Editor
3. 複製 `supabase/manual-trigger-sync.sql` 內容
4. 替換 `YOUR_SERVICE_ROLE_KEY`
5. 執行

---

### 步驟 3：驗證修復（2 分鐘）

**執行驗證查詢**：

```sql
SELECT 
  b.booking_number,
  o.processed_at,
  CASE 
    WHEN o.processed_at IS NOT NULL THEN '✅ 已同步'
    ELSE '❌ 未同步'
  END as sync_status,
  o.error_message
FROM bookings b
LEFT JOIN outbox o ON o.payload->>'id' = b.id::TEXT
WHERE b.created_at >= NOW() - INTERVAL '30 minutes'
ORDER BY b.created_at DESC
LIMIT 5;
```

**預期結果**：
- `sync_status` = '✅ 已同步'
- `processed_at` 有值
- `error_message` 是 NULL

**如果成功**：
1. 檢查 Firestore（Firebase Console）
2. 測試 App（查看訂單詳情）
3. 確認不再顯示「訂單不存在」

## 🚨 最常見的問題和解決方案

### 問題 1：Cron Job 未設置

**症狀**：
- 診斷顯示「Cron Job 未配置」
- `processed_at` 一直是 NULL

**解決**：
- 執行步驟 2A

**原因**：
- 初次部署時忘記設置 Cron Job
- 或 Cron Job 被意外刪除

---

### 問題 2：service_role_key 認證失敗

**症狀**：
- Cron Job 存在但從未執行
- 或執行失敗，返回 401 錯誤

**解決**：
- 重新獲取 service_role_key
- 執行步驟 2A 重新創建 Cron Job

**原因**：
- `current_setting('app.settings.service_role_key')` 不存在
- 需要直接在 SQL 中硬編碼 key

---

### 問題 3：環境變數未設置

**症狀**：
- `error_message` 提到 `FIREBASE_PROJECT_ID` 或 `FIREBASE_API_KEY`

**解決**：
- 在 Supabase Dashboard 設置環境變數
- 重新部署 Edge Function

**原因**：
- Edge Function 需要這些環境變數才能連接 Firestore
