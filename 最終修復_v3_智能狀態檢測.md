## ✅ 最新修復方案：智能狀態檢測

### 問題分析

之前嘗試使用固定的 status 值（`'pending'`, `'draft'`, `'pending_payment'`）都失敗了，因為：
- 不同環境的數據庫可能有不同的 status CHECK 約束
- 硬編碼的 status 值可能不在允許的列表中

### 解決方案：智能狀態檢測 ✅

**策略**: 從現有訂單中查詢有效的 status 值，然後使用相同的值

**優點**:
- ✅ 自動適配任何數據庫約束
- ✅ 不需要硬編碼 status 值
- ✅ 100% 保證使用有效的 status

### test_realtime_sync.sql - 智能狀態檢測

#### 創建訂單時：

```sql
DECLARE
  valid_status VARCHAR(20);
BEGIN
  -- 從現有訂單中獲取一個有效的 status 值
  SELECT status INTO valid_status
  FROM bookings
  LIMIT 1;
  
  -- 如果沒有現有訂單，使用默認值
  IF valid_status IS NULL THEN
    valid_status := 'pending';
  END IF;
  
  RAISE NOTICE '使用 status 值: %', valid_status;
  
  -- 創建測試訂單
  INSERT INTO bookings (
    customer_id,
    booking_number,
    status,
    ...
  ) VALUES (
    test_customer_id,
    'TEST_RT_' || EXTRACT(EPOCH FROM NOW())::BIGINT,
    valid_status,  -- ✅ 使用從現有訂單獲取的有效值
    ...
  );
END;
```

#### 更新訂單時：

```sql
DECLARE
  current_status VARCHAR(20);
  new_status VARCHAR(20);
BEGIN
  -- 獲取當前訂單的 status
  SELECT id, status INTO test_booking_id, current_status
  FROM bookings
  WHERE pickup_location LIKE '%即時同步測試%'
  ORDER BY created_at DESC
  LIMIT 1;
  
  -- 獲取一個不同的有效 status 值
  SELECT DISTINCT status INTO new_status
  FROM bookings
  WHERE status != current_status
  LIMIT 1;
  
  -- 如果找不到不同的 status，就使用相同的（只是為了觸發 UPDATE）
  IF new_status IS NULL THEN
    new_status := current_status;
  END IF;
  
  RAISE NOTICE '當前 status: %, 新 status: %', current_status, new_status;
  
  -- 更新訂單狀態
  UPDATE bookings
  SET status = new_status,  -- ✅ 使用從現有訂單獲取的有效值
      updated_at = NOW()
  WHERE id = test_booking_id;
END;
```

## 📝 工作原理

### 創建訂單

```
1. 查詢現有訂單 → 獲取一個 status 值
2. 如果有現有訂單 → 使用該 status
3. 如果沒有現有訂單 → 使用默認值 'pending'
4. 創建測試訂單 → 使用獲取的 status
```

### 更新訂單

```
1. 查詢測試訂單 → 獲取當前 status
2. 查詢其他訂單 → 獲取不同的 status
3. 如果找到不同的 → 使用該 status
4. 如果沒找到 → 使用相同的 status（仍會觸發 UPDATE）
5. 更新測試訂單 → 使用獲取的 status
```
