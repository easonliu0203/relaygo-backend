# 設置 Edge Function 環境變數

**問題**：Edge Function 返回 `success: 0, failure: 7`  
**原因**：環境變數未設置  
**預計時間**：3 分鐘

---

## 🚀 立即修復

### 步驟 1：打開環境變數設置頁面

**URL**：
```
https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/functions
```

或者：
1. 在 Supabase Dashboard 中
2. 點擊左側選單「Project Settings」（齒輪圖標）
3. 點擊「Edge Functions」

---

### 步驟 2：設置環境變數

**需要設置的變數**：

#### 1. FIREBASE_PROJECT_ID

**值**：您的 Firebase Project ID

**如何獲取**：
1. 打開 Firebase Console：https://console.firebase.google.com
2. 選擇您的專案
3. 點擊「Project Settings」（齒輪圖標）
4. 在「General」標籤中找到「Project ID」
5. 複製 Project ID

**範例**：`my-charter-app-12345`

---

#### 2. FIREBASE_API_KEY

**值**：您的 Firebase Web API Key

**如何獲取**：
1. 在 Firebase Console 的「Project Settings」中
2. 滾動到「Your apps」section
3. 找到您的 Web App
4. 複製「Web API Key」

**範例**：`AIzaSyD...` (以 AIza 開頭的長字串)

---

#### 3. SUPABASE_URL

**值**：
```
https://vlyhwegpvpnjyocqmfqc.supabase.co
```

---

#### 4. SUPABASE_SERVICE_ROLE_KEY

**值**：您的 Supabase Service Role Key

**如何獲取**：
1. 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/settings/api
2. 找到「service_role」section
3. 點擊「Reveal」
4. 複製 key

**範例**：`eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` (很長的 JWT token)

---

### 步驟 3：保存並重新部署

**在環境變數設置頁面**：
1. 點擊「Save」或「Update」
2. 環境變數會自動應用到所有 Edge Functions

**不需要重新部署**（環境變數會自動生效）

---

### 步驟 4：重新測試

**返回 Edge Function 測試頁面**：
1. 打開：https://supabase.com/dashboard/project/vlyhwegpvpnjyocqmfqc/functions
2. 點擊 `sync-to-firestore`
3. 點擊「Invoke」

**預期結果**：
```json
{
  "message": "事件處理完成",
  "total": 7,
  "success": 7,
  "failure": 0
}
```

---

## 🔍 驗證環境變數

**檢查環境變數是否已設置**：

在 Edge Function 代碼中，環境變數是這樣使用的：

```typescript
const FIREBASE_PROJECT_ID = Deno.env.get('FIREBASE_PROJECT_ID')!
const FIREBASE_API_KEY = Deno.env.get('FIREBASE_API_KEY')!
const SUPABASE_URL = Deno.env.get('SUPABASE_URL')!
const SUPABASE_SERVICE_ROLE_KEY = Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
```

如果這些變數未設置，Edge Function 會拋出錯誤。
