# 即時同步開關問題 - 診斷與修復報告

> **問題**: 即時同步開關無法控制同步行為  
> **診斷日期**: 2025-10-16  
> **狀態**: ✅ **已診斷並提供修復方案**

---

## 🔍 **問題診斷**

### 問題描述

**用戶報告**:
- 在管理後台關閉「即時同步設定」開關後
- 系統仍然執行即時同步（1-3秒延遲）
- 預期應該退回到 Cron Job 補償同步模式（每分鐘一次）

**實際行為**:
```
管理後台開關：OFF（已關閉）
實際同步行為：即時同步仍在運作（1-3秒延遲）
預期行為：Cron Job 補償同步（0-60秒延遲）
```

---

### 根本原因

**診斷結果**: ✅ **Trigger 函數沒有檢查開關狀態**

**證據**:

#### 1. 管理後台的開關控制邏輯

**文件**: `web-admin/src/app/api/admin/realtime-sync/toggle/route.ts`

**開關關閉時的操作**:
```typescript
if (!enabled) {
  // 停用即時同步
  sql = `
    -- 刪除 Trigger
    DROP TRIGGER IF EXISTS bookings_realtime_notify_trigger ON bookings;
    
    -- 更新配置狀態
    UPDATE system_settings
    SET value = jsonb_set(value, '{enabled}', 'false'::jsonb)
    WHERE key = 'realtime_sync_config';
  `;
}
```

**問題**:
- API 嘗試刪除 Trigger，但可能失敗（權限問題或 SQL 執行失敗）
- 即使配置更新為 `enabled: false`，Trigger 仍然存在並繼續執行

#### 2. Trigger 函數的當前實現

**文件**: `supabase/FINAL_FIX_401.sql`（修復前）

**當前代碼**:
```sql
CREATE OR REPLACE FUNCTION notify_edge_function_realtime()
RETURNS TRIGGER AS $$
DECLARE
  edge_function_url TEXT;
  service_role_key TEXT;
  request_id BIGINT;
BEGIN
  -- 設定 Edge Function URL
  edge_function_url := '...';
  
  -- Service Role Key
  service_role_key := 'eyJ...';
  
  -- 發送 HTTP 請求（沒有檢查開關狀態！）
  SELECT net.http_post(...) INTO request_id;
  
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

**問題**:
- ❌ **函數沒有檢查 `system_settings.realtime_sync_config.enabled` 狀態**
- ❌ **無論開關是否關閉，都會發送 HTTP 請求**

---

## 🔧 **修復方案**

### 方案概述

**修改 Trigger 函數，添加開關檢查邏輯**:
1. 在發送 HTTP 請求前，查詢 `system_settings` 表
2. 檢查 `realtime_sync_config.enabled` 狀態
3. 如果開關關閉（`enabled = false`），跳過 HTTP 請求
4. 如果開關開啟（`enabled = true`），正常發送 HTTP 請求

---

### 修復後的代碼

**文件**: `supabase/FINAL_FIX_401.sql`（已修復）

**新增的開關檢查邏輯**:
```sql
CREATE OR REPLACE FUNCTION notify_edge_function_realtime()
RETURNS TRIGGER AS $$
DECLARE
  edge_function_url TEXT;
  service_role_key TEXT;
  request_id BIGINT;
  sync_enabled BOOLEAN;  -- 新增：開關狀態變量
BEGIN
  -- ============================================
  -- 步驟 1: 檢查即時同步開關狀態（新增）
  -- ============================================
  BEGIN
    SELECT (value->>'enabled')::BOOLEAN INTO sync_enabled
    FROM system_settings
    WHERE key = 'realtime_sync_config';
    
    -- 如果找不到配置，默認為停用
    IF sync_enabled IS NULL THEN
      sync_enabled := false;
    END IF;
    
  EXCEPTION
    WHEN OTHERS THEN
      -- 如果查詢失敗，默認為停用
      sync_enabled := false;
      RAISE WARNING 'Failed to check realtime sync config: %', SQLERRM;
  END;
  
  -- 如果即時同步已停用，跳過 HTTP 請求（新增）
  IF NOT sync_enabled THEN
    RAISE NOTICE 'Realtime sync is disabled, skipping HTTP request for booking %. Will be handled by Cron Job.', NEW.id;
    RETURN NEW;
  END IF;
  
  -- ============================================
  -- 步驟 2: 設定 Edge Function URL 和 Service Role Key
  -- ============================================
  edge_function_url := '...';
  service_role_key := 'eyJ...';
  
  -- ============================================
  -- 步驟 3: 發送 HTTP 請求到 Edge Function
  -- ============================================
  SELECT net.http_post(...) INTO request_id;
  
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

**關鍵改進**:
1. ✅ 新增 `sync_enabled` 變量
2. ✅ 查詢 `system_settings.realtime_sync_config.enabled`
3. ✅ 如果開關關閉，跳過 HTTP 請求並記錄日誌
4. ✅ 異常處理：查詢失敗時默認為停用

## 📋 **驗證查詢**

### 查詢 1: 檢查開關狀態

```sql
SELECT 
  key AS "配置鍵",
  value->>'enabled' AS "開關狀態",
  CASE 
    WHEN (value->>'enabled')::BOOLEAN THEN '✅ 已啟用'
    ELSE '❌ 已停用'
  END AS "狀態說明",
  updated_at AS "最後更新時間"
FROM system_settings
WHERE key = 'realtime_sync_config';
```

---

### 查詢 2: 檢查 HTTP 請求記錄

```sql
SELECT 
  id,
  status_code,
  created AS request_time
FROM net._http_response
WHERE created > NOW() - INTERVAL '10 minutes'
ORDER BY created DESC
LIMIT 10;
```

**驗證**:
- 開關關閉後，不應該有新的 HTTP 請求記錄

---

### 查詢 3: 檢查 Outbox 事件延遲

```sql
SELECT 
  id,
  aggregate_id AS booking_id,
  created_at,
  processed_at,
  EXTRACT(EPOCH FROM (processed_at - created_at)) AS delay_seconds
FROM outbox
WHERE created_at > NOW() - INTERVAL '10 minutes'
ORDER BY created_at DESC
LIMIT 10;
```

**驗證**:
- 開關開啟：delay_seconds < 5 秒
- 開關關閉：delay_seconds 在 0-60 秒之間
