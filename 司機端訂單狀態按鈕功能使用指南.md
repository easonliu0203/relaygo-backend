# 司機端訂單狀態按鈕功能使用指南

## 功能簡介

在司機確認接單後，司機端訂單詳情頁面會顯示兩個狀態更新按鈕：
1. **出發前往載客** - 司機開始前往上車地點時點擊
2. **抵達上車地點** - 司機到達上車地點時點擊

點擊按鈕後，系統會自動：
- 更新訂單狀態
- 在聊天室發送通知訊息給客戶
- 同步狀態到 Firestore

## 部署步驟

### 1. 更新數據庫約束

確保 Supabase `bookings` 表支援新的狀態：

```bash
# 連接到 Supabase 並執行以下 SQL
cd supabase
psql -h your-supabase-host -U postgres -d postgres -f update-booking-status-constraint.sql
```

或者在 Supabase Dashboard 的 SQL Editor 中執行 `supabase/update-booking-status-constraint.sql` 文件。

### 2. 部署 Backend API

Backend API 已經包含了新的端點，無需額外部署。確保 Backend 服務正在運行：

```bash
cd backend
npm install
npm run dev
```

### 3. 部署 Flutter 應用

重新編譯並部署 Flutter 應用：

```bash
cd mobile
flutter pub get
flutter build apk  # Android
# 或
flutter build ios  # iOS
```

### 4. 驗證 Firestore 同步

確保 Supabase Edge Function 正在運行：

```bash
# 檢查 Edge Function 狀態
supabase functions list

# 如果未運行，啟動 Edge Function
supabase functions deploy sync-to-firestore
```

## 使用流程

### 司機端操作流程

1. **確認接單**
   - 公司端派單後，司機在訂單列表中看到新訂單
   - 點擊訂單進入詳情頁面
   - 點擊「確認接單」按鈕
   - 系統創建聊天室，訂單狀態變為「已配對」

2. **出發前往載客**
   - 訂單狀態為「已配對」時，顯示「出發前往載客 🚗」按鈕（藍色）
   - 司機準備出發時，點擊此按鈕
   - 確認對話框中點擊「確認出發」
   - 系統自動：
     - 更新訂單狀態為「進行中」
     - 在聊天室發送訊息：「司機已出發，正在前往上車地點 🚗」
     - 客戶收到通知

3. **抵達上車地點**
   - 訂單狀態為「進行中」時，顯示「抵達上車地點 📍」按鈕（綠色）
   - 司機到達上車地點時，點擊此按鈕
   - 確認對話框中點擊「確認到達」
   - 系統自動：
     - 更新訂單狀態（保持「進行中」）
     - 在聊天室發送訊息：「司機已到達上車地點，請準備上車 📍」
     - 客戶收到通知

### 客戶端體驗

1. **接收出發通知**
   - 司機點擊「出發前往載客」後
   - 客戶在聊天室中看到系統訊息：「司機已出發，正在前往上車地點 🚗」
   - 訂單狀態更新為「進行中」

2. **接收到達通知**
   - 司機點擊「抵達上車地點」後
   - 客戶在聊天室中看到系統訊息：「司機已到達上車地點，請準備上車 📍」
   - 客戶準備上車

## 狀態流程圖

```
訂單狀態流程（Supabase）:
matched → driver_confirmed → driver_departed → driver_arrived → trip_started → ...

訂單狀態流程（Firestore）:
pending → matched → inProgress → inProgress → inProgress → ...

按鈕顯示邏輯:
matched (Firestore) → 顯示「出發前往載客」按鈕
inProgress (Firestore) → 顯示「抵達上車地點」按鈕
```

## 測試方法

### 方法 1: 使用 Flutter 應用測試

1. 使用司機帳號登入 Flutter 應用
2. 確保有一個狀態為「已配對」的訂單
3. 進入訂單詳情頁面
4. 按照上述使用流程操作
5. 檢查聊天室是否收到系統訊息

### 方法 2: 使用 API 測試腳本

```bash
cd backend

# 修改腳本中的 BOOKING_ID 和 DRIVER_UID
nano test-driver-status-api.sh

# 執行測試
chmod +x test-driver-status-api.sh
./test-driver-status-api.sh
```

### 方法 3: 使用 curl 手動測試

```bash
# 測試司機出發
curl -X POST http://localhost:3001/api/booking-flow/bookings/{bookingId}/depart \
  -H "Content-Type: application/json" \
  -d '{"driverUid": "your-driver-firebase-uid"}'

# 測試司機到達
curl -X POST http://localhost:3001/api/booking-flow/bookings/{bookingId}/arrive \
  -H "Content-Type: application/json" \
  -d '{"driverUid": "your-driver-firebase-uid"}'
```

## 常見問題

### Q1: 按鈕沒有顯示
**A**: 檢查訂單狀態是否正確。「出發前往載客」按鈕只在訂單狀態為 `matched` 時顯示，「抵達上車地點」按鈕只在訂單狀態為 `inProgress` 時顯示。

### Q2: 點擊按鈕後顯示「無權限操作此訂單」
**A**: 確保當前登入的司機是該訂單的分配司機。檢查訂單的 `driverId` 是否與當前司機的 Firebase UID 匹配。

### Q3: 點擊按鈕後顯示「訂單狀態不正確」
**A**: 檢查訂單的 Supabase 狀態。「出發」操作需要訂單狀態為 `driver_confirmed`，「到達」操作需要訂單狀態為 `driver_departed`。

### Q4: 系統訊息沒有出現在聊天室
**A**: 檢查以下幾點：
1. Firebase Admin SDK 是否正確配置
2. Backend 日誌中是否有錯誤訊息
3. Firestore 安全規則是否允許寫入
4. 聊天室是否已創建

### Q5: Firestore 狀態沒有同步
**A**: 檢查以下幾點：
1. Supabase Edge Function 是否正在運行
2. `outbox` 表中是否有未處理的事件
3. Edge Function 日誌中是否有錯誤訊息
4. 等待 5-30 秒後再檢查（同步有延遲）

3. **Flutter 日誌**: Android Studio / Xcode 控制台
4. **Firestore 日誌**: Firebase Console → Firestore → Usage
